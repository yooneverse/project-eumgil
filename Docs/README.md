## Archive 기준

`archive` 하위 문서는 구현 완료, 최신 기준 대체, 초기 조사, 초안, 일정 관리 등 이력 보관용 문서다. 최신 정책이나 구현 기준을 판단할 때는 위 최신 기준 문서를 먼저 본다.

| 위치 | 보관 내용 |
| --- | --- |
| `Docs/PRD/archive/2026-04_requirement_sources/` | 초기 Notion 요구사항 정리, MON-01 범위 스냅샷 |
| `Docs/ERD/archive/` | `ERD_v1`~`ERD_v3` 이전 버전 |
| `Docs/기획/archive/drafts/` | 과거 MVP/확장 화면명세, 기능명세 재구성 초안 |
| `Docs/기획/archive/research/` | 초기 인터뷰, 필드트립 질문지, 검토자료 |
| `Docs/기획/archive/schedules/` | 과거 간트차트, 개발 일정 공유 문서 |

## 충돌 판단 원칙

1. 기능 정책은 기획서, 화면명세서, PRD/기능명세서 순으로 확인한다.
2. 요청/응답 필드와 서버 계약은 API 명세를 따른다.
3. 데이터 저장 책임과 테이블 구조는 `Docs/ERD/ERD_v4.md`를 따른다.
4. FE 구현 세부는 `FE/Docs/` 문서를 참고하되, 제품 정책과 충돌하면 `Docs/` 기준 문서를 우선한다.
5. archive 문서는 이력 추적용이며 최신 기준 문서를 대체하지 않는다.
