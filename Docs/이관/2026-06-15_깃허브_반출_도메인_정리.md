# GitHub 반출 도메인 정리 기록

## 배경

GitLab 반출 심사에서 교육기관 내부 서버에 직접 접근 가능한 도메인 패턴이 검출되어, GitHub 이관 전 `develop` 브랜치 기준으로 직접 접근 정보를 제거한다.

## 처리 원칙

- 내부 Git, 운영 서버, 채팅 webhook, LLM gateway 주소를 저장소에 직접 기록하지 않는다.
- 운영에 필요한 주소는 환경변수, Jenkins credential, Terraform 변수, 배포 환경의 secret으로 주입한다.
- 문서와 포팅 매뉴얼에는 실제 주소 대신 `<INTERNAL_...>` placeholder를 사용한다.
- 검증 기록 문서 자체가 재검출 원인이 되지 않도록 실제 내부 도메인 문자열은 남기지 않는다.

## 변경 유형

- AI 테스트 서버의 LLM gateway 기본 URL을 환경변수 기반으로 변경
- Jenkins pipeline과 init groovy의 Git repository URL을 credential 또는 환경변수 기반으로 변경
- Grafana/SonarQube Git 연동 기본 URL을 예시 도메인으로 변경
- S1 운영 서버 주소를 문서 placeholder 또는 예시 호스트로 변경
- 운영 브리프 테스트 데이터의 Git/LLM gateway URL을 예시 도메인으로 변경

## 검증

- 반출 대상 브랜치: `develop`
- 검증일: 2026-06-15
- 검증 대상: 현재 작업트리의 내부 직접 접근 도메인 패턴 잔존 여부
- 검증 결과: 수정 후 검색 결과를 커밋 전 재확인한다.

## 후속 주의사항

- Git history, tag, 다른 장기 브랜치까지 함께 이관할 경우 별도 history 정리 또는 sanitized export repository가 필요할 수 있다.
- 운영 환경에서 Jenkins `e102-repo-url`, `e102-llm-gateway-base-url`, `E102_REPO_URL`, S1 host 관련 변수 값을 별도로 주입해야 한다.
