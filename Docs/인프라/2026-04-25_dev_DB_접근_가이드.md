# dev DB 접근 가이드

이 문서는 백엔드 개발자가 S1 dev 서버의 PostgreSQL/PostGIS에 접근하는 방법을 정리한다.

## 원칙

- dev DB는 외부에 직접 공개하지 않는다.
- S1 서버에서도 PostgreSQL 원 포트는 `127.0.0.1:5432`에만 바인딩한다.
- 로컬 PC에서는 SSH tunnel 또는 SSM port forwarding으로만 접근한다.
- DB 접속 계정, 비밀번호, DB 이름은 저장소에 기록하지 않고 루트 `.env.dev`에만 둔다.

## 수동 DB 연결

로컬 PC에서 터널을 연다.

```bash
ssh -i K14E102T.pem -L 15432:127.0.0.1:5432 ubuntu@<INTERNAL_S1_HOST>
```

DB 클라이언트에서는 아래처럼 접속한다.

```text
Host: 127.0.0.1
Port: 15432
Database: .env.dev의 POSTGRES_DB
User: .env.dev의 DB_USERNAME
Password: .env.dev의 DB_PASSWORD
```

`15432`를 쓰는 이유는 로컬 PC에 이미 PostgreSQL이 `5432`로 떠 있을 수 있기 때문이다. 꼭 `5432`를 써야 하면 `-L 5432:127.0.0.1:5432`로 바꿔도 된다.

## 로컬 WAS에서 dev DB 사용

`make be-dev-up`은 로컬 Docker container로 백엔드를 띄우고, S1 dev DB/Redis/MinIO/GraphHopper에 붙인다.

동작 방식:

1. `K14E102T.pem`으로 S1에 SSH tunnel을 연다.
2. 로컬 포트 `15432`, `16379`, `19000`, `18989`를 각각 S1의 PostgreSQL, Redis, MinIO, GraphHopper로 포워딩한다.
3. backend container에는 `host.docker.internal` 기준 endpoint를 주입한다.
4. DB 계정과 비밀번호는 루트 `.env.dev`에서 읽는다.
5. backend container 기동 후 WAS 로그를 follow한다. `Ctrl+C`를 누르면 로그 follow만 빠져나오고 container는 유지된다.

실행:

```bash
make be-dev-up
```

전제:

- Docker Desktop이 실행 중이어야 한다.
- 루트에 `K14E102T.pem`이 있어야 한다.
- `K14E102T.pem`은 `.gitignore`의 `*.pem` 규칙으로 저장소에 올라가지 않아야 한다.

확인:

```bash
make be-dev-config
make be-dev-logs
```

`make be-dev-up`도 실행 직후 `docker logs -f`를 붙여주므로 별도 터미널을 열지 않아도 startup log를 볼 수 있다.

중지:

```bash
make be-dev-down
```

`make be-dev-down`은 backend container를 삭제하고, `make be-dev-up`이 자동으로 연 SSH tunnel도 함께 종료한다. 사용자가 직접 열어둔 tunnel은 추적하지 않으므로 종료하지 않는다.

## `.env.dev`에 필요한 값

아래 값은 각자 로컬의 `.env.dev`에 채운다. 실제 값은 저장소에 올리지 않는다.

```dotenv
SPRING_PROFILES_ACTIVE=dev
SERVER_PORT=8080
POSTGRES_DB=
POSTGRES_PORT=5432
DB_USERNAME=
DB_PASSWORD=
JPA_DDL_AUTO=update
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_SSL=false
MINIO_ROOT_USER=
MINIO_ROOT_PASSWORD=
MINIO_BUCKET=
MINIO_REGION=ap-northeast-2
AUTH_TEST_ENABLED=true
AUTH_TEST_KAKAO_JAVASCRIPT_KEY=
AUTH_TEST_NAVER_CLIENT_ID=
AUTH_TEST_GOOGLE_CLIENT_ID=
JWT_SECRET=
JWT_ISSUER=e102-dev
JWT_ACCESS_TOKEN_TTL=15m
JWT_REFRESH_TOKEN_TTL=14d
JWT_SIGNUP_TOKEN_TTL=10m
```

`make be-dev-up`은 아래 endpoint 값을 실행 시점에 자동으로 override한다.

```dotenv
DB_URL=jdbc:postgresql://host.docker.internal:15432/${POSTGRES_DB}
REDIS_HOST=host.docker.internal
REDIS_PORT=16379
MINIO_ENDPOINT=http://host.docker.internal:19000
GRAPHHOPPER_BASE_URL=http://host.docker.internal:18989
```

dev 서버의 `.env.dev`에는 서버 내부 주소를 명시한다.

```dotenv
DB_URL=jdbc:postgresql://postgres:5432/${POSTGRES_DB}
REDIS_HOST=redis
MINIO_ENDPOINT=http://minio:9000
MINIO_REGION=ap-northeast-2
GRAPHHOPPER_BASE_URL=http://graphhopper:8989
```

로컬 `make be-dev-up`에서는 위 값을 그대로 쓰지 않고 터널 주소로 override한다.

dev 서버에서는 테스트 로그인 페이지를 사용하므로 `AUTH_TEST_ENABLED=true`와 `AUTH_TEST_*` 값을 `.env.dev` 또는 Jenkins `e102-dev-env-file` credential에 포함한다.
`JWT_SECRET`은 표준 Base64 문자열이어야 하며, URL-safe Base64의 `_`, `-` 문자는 사용하지 않는다.

## `.env.prod`에 필요한 값

prod는 관리형 리소스 endpoint를 직접 채운다. 실제 값은 배포 서버의 `.env.prod` 또는 Jenkins `e102-prod-env-file` credential로만 관리한다.

```dotenv
SPRING_PROFILES_ACTIVE=prod
SERVER_PORT=8080
POSTGRES_DB=
POSTGRES_PORT=5432
DB_URL=
DB_USERNAME=
DB_PASSWORD=
JPA_DDL_AUTO=update
REDIS_HOST=
REDIS_PORT=6379
S3_ENDPOINT=  # AWS S3 기본 endpoint 사용 시 비워둔다. MinIO/호환 스토리지에서만 지정한다.
S3_BUCKET=
S3_ACCESS_KEY=
S3_SECRET_KEY=
S3_REGION=ap-northeast-2
GRAPHHOPPER_BASE_URL=
JWT_SECRET=
JWT_ISSUER=e102
JWT_ACCESS_TOKEN_TTL=15m
JWT_REFRESH_TOKEN_TTL=14d
JWT_SIGNUP_TOKEN_TTL=10m
```

prod의 `DB_URL`은 RDS endpoint 기준으로 작성한다. 예시는 아래 형식만 참고하고 실제 host, DB 이름, 계정 값은 기록하지 않는다.

```dotenv
DB_URL=jdbc:postgresql://<rds-endpoint>:5432/<database>
```

dev/prod DB는 `geometry(Point, 4326)` 컬럼을 사용하므로 대상 데이터베이스에 `postgis` extension이 활성화되어 있어야 한다.
현재 backend bootstrap은 `schema.sql`에서 `CREATE EXTENSION IF NOT EXISTS postgis;`를 먼저 수행한 뒤 스키마 보정을 진행한다.
따라서 새 DB에서 최초 기동하는 계정은 extension 생성 권한이 있어야 하며, 권한이 없으면 DBA가 미리 `postgis` extension을 활성화해야 한다.

prod에서는 테스트 로그인 페이지를 사용하지 않으므로 `AUTH_TEST_ENABLED`와 `AUTH_TEST_*` 값을 운영 compose에 전달하지 않는다.

## 포트 충돌 시

로컬에서 `15432`, `16379`, `19000`을 이미 사용 중이면 실행 전에 포트를 바꾼다.

```bash
BE_DEV_DB_LOCAL_PORT=25432 BE_DEV_REDIS_LOCAL_PORT=26379 BE_DEV_MINIO_LOCAL_PORT=29000 BE_DEV_GRAPHHOPPER_LOCAL_PORT=28989 make be-dev-up
```

터널을 직접 열어둔 상태라면 자동 터널을 끌 수 있다.

```bash
BE_DEV_TUNNEL_AUTO=false make be-dev-up
```

자동 tunnel은 `.tmp/be-dev-tunnel.pid`로 추적한다. `make be-dev-down`이 이 PID를 기준으로 종료하므로 일반 사용자는 OS 프로세스 목록에서 직접 `ssh`를 찾아 종료할 필요가 없다.
