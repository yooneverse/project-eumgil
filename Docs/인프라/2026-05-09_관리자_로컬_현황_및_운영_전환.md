# 관리자 웹 운영 전환 정리

> **작성일:** 2026-05-09  
> **작성자:** AI

---

## 1. 기준

관리자 화면은 BE 정적 리소스가 아니라 별도 관리자 프론트로 운영한다.

```text
관리자 웹: https://admin.busaneumgil.com
운영 API: https://api.busaneumgil.com
```

따라서 `/admin-web/` 정적 서빙 구조는 운영 기준에서 제외한다.

---

## 2. 현재 코드 기준

### 2.1 ADMIN

`ADMIN/`은 관리자 프론트 소스 루트다.

```text
ADMIN/src
ADMIN/package.json
ADMIN/vite.config.ts
```

운영 빌드:

```powershell
cd ADMIN
npm run build:prod
```

prod 배포에서는 `docker-compose.prod.yml`의 `admin` 서비스가 `ADMIN/Dockerfile`로 동일 빌드를 수행한다.

빌드 산출물:

```text
ADMIN/dist
```

`ADMIN/dist`는 배포 산출물이며, 일반 커밋 대상이 아니다.

### 2.2 API 호출 기준

관리자 프론트는 `VITE_BACKEND_API_URL`을 우선 사용한다.

운영 빌드 환경변수:

```text
VITE_BACKEND_API_URL=https://api.busaneumgil.com
VITE_ADMIN_KAKAO_JAVASCRIPT_KEY=${KAKAO_JAVASCRIPT_KEY}
VITE_ADMIN_NAVER_CLIENT_ID=Naver Client ID
VITE_ADMIN_GOOGLE_CLIENT_ID=Google Client ID
```

### 2.3 지도 SDK

Kakao Map SDK는 `VITE_KAKAO_MAP_KEY`를 사용한다.

```text
VITE_KAKAO_MAP_KEY=${KAKAO_JAVASCRIPT_KEY}
```

카카오 개발자 콘솔에 아래 origin이 등록되어야 한다.

```text
https://admin.busaneumgil.com
```

`Kakao Map SDK 로드 실패`는 대부분 코드 문제가 아니라 앱 키 또는 Web platform 도메인 등록 문제다.

### 2.4 인증

관리자 화면은 진입 전에 로그인 화면을 보여준다.

지원 방식:

```text
Kakao 로그인
Naver 로그인
Google 로그인
```

흐름:

```text
소셜 제공자 accessToken 발급
-> BE /auth/social-login
-> 서비스 accessToken 발급
-> BE /admin/me
-> users.role = ADMIN 이면 관리자 화면 진입
```

운영 관리자 로그인 설정은 ADMIN 빌드 환경변수 `VITE_ADMIN_*`로 주입한다.
`/auth/test-config`는 테스트용 보조 경로이며 운영 의존성으로 두지 않는다.

권한 기준:

```text
users.role = USER | ADMIN
```

가입 기본값은 `USER`다.

### 2.5 백엔드 보안

`/admin/**` API는 `ROLE_ADMIN`으로 보호된다.

```text
/admin, /admin/** -> hasRole("ADMIN")
```

`JwtAuthenticationFilter`는 accessToken의 userId로 사용자를 조회하고, DB의 `users.role`이 `ADMIN`이면 `ROLE_ADMIN` 권한을 부여한다.

토큰 자체에 role을 넣지 않으므로 DB role 변경은 다음 요청부터 반영된다.

## 3. CORS

관리자 웹과 API가 다른 도메인이므로 운영에서는 CORS가 필요하다.

운영 설정:

```text
CORS_ALLOWED_ORIGINS=https://admin.busaneumgil.com
```

운영에서 관리자 도메인이 바뀌면 `CORS_ALLOWED_ORIGINS`도 같이 바꿔야 한다.

---

## 4. OAuth 제공자 콘솔 등록값

운영 관리자 도메인을 기준으로 각 제공자 콘솔에 등록한다.

```text
origin: https://admin.busaneumgil.com
redirect/callback: https://admin.busaneumgil.com
```

제공자별 확인 항목:

```text
Kakao: Web platform domain, Kakao Login, redirect URI, Maps JavaScript SDK
Naver: Callback URL
Google: Authorized JavaScript origins
```

## 5. 관리자 기능 범위

현재 관리자 화면에서 다루는 기능:

```text
보행 네트워크 검수
편의시설/접근성 데이터 확인
사용자 도로 상태 제보 검토
```

주요 API:

```text
GET /admin/me
GET /admin/areas
GET /admin/road-network/segments
GET /admin/places/facilities
GET /admin/hazard-reports
GET /admin/hazard-reports/{reportId}
PATCH /admin/hazard-reports/{reportId}/approve
PATCH /admin/hazard-reports/{reportId}/reject
```

관리자 지도는 CSV 서버가 아니라 BE DB 조회 API를 사용한다.

```text
ADMIN -> BE /admin/** -> DB
```

## 6. 데이터 적재 기준

관리자 화면은 적재 결과를 확인하는 도구다.
DB 적재 자체를 관리자 화면에서 직접 실행하지 않는다.

적재는 별도 스크립트, 배치, 운영 명령 또는 CI/CD job으로 처리한다.

관리자 화면에서 확인할 대상:

```text
admin_areas
road_segments
road_nodes
places
place_accessibility_features
```

운영 적재 전 확인 항목:

```text
row count
geometry SRID
geometry index
orphan 데이터 여부
구/동별 조회 성능
rollback 방법
```

## 7. 운영 전환 체크리스트

- [ ] `https://admin.busaneumgil.com` 배포 대상이 준비되어 있다.
- [ ] Route53에 `admin.busaneumgil.com` A record가 S2 Elastic IP로 연결되어 있다.
- [ ] S2 nginx에 `admin.busaneumgil.com -> 127.0.0.1:3001` 라우팅이 적용되어 있다.
- [ ] `docker-compose.prod.yml`의 `admin` 서비스가 기동되어 있다.
- [ ] `VITE_BACKEND_API_URL=https://api.busaneumgil.com`로 빌드된다.
- [ ] `VITE_ADMIN_KAKAO_JAVASCRIPT_KEY`, `VITE_ADMIN_NAVER_CLIENT_ID`, `VITE_ADMIN_GOOGLE_CLIENT_ID`가 운영용 값으로 주입된다.
- [ ] `VITE_KAKAO_MAP_KEY`가 운영용 Kakao JavaScript key다.
- [ ] BE prod CORS가 `https://admin.busaneumgil.com`을 허용한다.
- [ ] Kakao/Naver/Google 콘솔에 `https://admin.busaneumgil.com`이 등록되어 있다.
- [ ] 운영 DB에 `users.role` 컬럼이 반영되어 있다.
- [ ] 최초 관리자 계정이 `ADMIN`으로 승격되어 있다.
- [ ] 운영 DB에 관리자 지도 데이터가 적재되어 있다.
- [ ] `/admin/**` API가 `ROLE_ADMIN`으로 보호된다.

## 8. 현재 결론

관리자 운영 기준은 아래 구조로 간다.

```text
https://admin.busaneumgil.com
-> https://api.busaneumgil.com/admin/**
```

BE의 `/admin-web/` 정적 서빙은 운영 기준에 맞지 않으므로 제거한다.
