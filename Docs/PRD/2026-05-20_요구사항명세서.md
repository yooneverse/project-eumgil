## 1. 문서 목적

본 문서는 평가자가 부산이음길 앱의 요구사항 범위와 최신 구현 상태를 한 번에 확인할 수 있도록 정리한 제출용 요구사항명세서다.

- `구현 완료`: FE 화면/상태/route 또는 API 연동 코드가 현재 구현되어 제출 시연 범위에 포함된다.
- `부분 구현`: 핵심 화면이나 흐름은 구현되어 있으나, 서버 데이터 품질, 외부 연동, 예외 정책 또는 상세 UX 확인이 남아 있다.
- `확인 필요`: FE에서 소비하거나 호출하는 지점은 있으나, 백엔드/AI/인프라 완료 여부를 해당 파트에서 확인해야 한다.
- `후순위`: 제출 MVP의 직접 시연 범위가 아니거나 향후 개선 항목이다.

## 3. 제출 MVP 기능 범위

제출 시연에서 기능으로 설명할 수 있는 범위는 아래와 같다.

| 기능군 | 제출 설명 기준 |
| --- | --- |
| 인증/온보딩 | 소셜 로그인 이후 사용자 유형을 선택하고, 보행약자/저시력자에 따라 다른 온보딩과 홈 화면으로 진입한다. |
| 지도/시설 | 현재 위치 기반 지도에서 접근성 시설과 승인 제보 정보를 확인하고, 시설 상세에서 북마크와 길찾기로 이어진다. |
| 검색/목적지 | 텍스트 또는 음성으로 목적지를 검색하고, 결과를 선택해 경로 탐색으로 연결한다. |
| 경로/안내 | 도보/대중교통 경로를 비교하고, 경로 상세와 길안내 화면으로 이동한다. 도보는 안전/최단 기준, 대중교통은 추천/최소 환승/최소 도보 기준을 제공한다. |
| 저시력자 UI | 저시력자 전용 홈, 음성 입력, 카테고리, 북마크, 경로 브리핑, 길안내, 도착, 마이페이지 화면을 제공한다. |
| 북마크 | 장소/경로 북마크를 탭으로 분리하고, 정렬, 편집 삭제, 경로 재탐색을 제공한다. |
| 제보 | 장애물 유형 선택, 위치 조정, 사진/설명 입력, 제출 완료, 제보 이력 확인을 제공한다. |
| 마이페이지 | 프로필 요약, 사용자 유형 변경, 앱 도움말/약관/개인정보, 텍스트 크기, 로그아웃, 회원탈퇴를 제공한다. |

## 5. 백엔드/AI/인프라 확인 요청

FE 문서와 시연 정합성을 위해 아래 항목은 담당 파트 확인이 필요하다.

1. 장소 검색 결과와 내부 접근성 데이터 매칭 기준, 반경/이름/주소 매칭 우선순위
2. 도보 경로의 안전/최단 기준과 실제 GraphHopper 가중치 적용 여부
3. 대중교통 경로의 추천/최소 환승/최소 도보 기준과 저상버스 데이터 최신성
4. 두리발 전화 연결 및 저상버스 예약 후보의 외부 연동 성공/실패 정책
5. 제보 승인 상태가 지도 마커와 위험도에 반영되는 기준
6. `voice/analyze`의 STT/LLM/TTS 처리 범위와 실패 응답 정책
7. PostGIS/OSM/공공데이터 전처리 완료 범위
8. 운영 배포/모니터링 자료의 제출 포함 여부
