본 문서는 2026-05-20 제출 전 최신 FE 구현 상태를 기준으로 기능 기준선을 다시 정리한 문서다. 기능 우선순위, 인증/온보딩 정책, 서버/로컬 데이터 책임은 본 문서와 최신 요구사항명세서를 기준으로 판단하며, 과거 FE 분석/정합성 문서는 구현 참고 자료로만 사용한다. 충돌 시에는 요구사항명세서 -> 기능명세서 -> 화면명세서 -> API 명세 -> FE 보조 문서 순으로 해석한다.

## 0. 정렬 기준

| 항목 | 최신 기준 |
| --- | --- |
| 앱 진입 gate | 앱 시작 gate는 `pending signup -> ONB-001`, `세션 없음 -> AUTH-001`, `세션 있음 + profile 미완료 -> AUTH-002`, `온보딩 완료 + LOW_VISION -> LV-HOM-01`, `온보딩 완료 + MOBILITY_IMPAIRED -> MAP-01` 순으로 분기한다. |
| 로그인 정책 | `AUTH-001`은 필수 로그인 화면이며 카카오/네이버/구글 소셜 로그인만 제공한다. 운영 기준 `비로그인으로 계속`, guest mode, 비로그인 핵심 기능 사용은 제거한다. |
| 온보딩 정책 | current runtime 기준 보행약자 플로우는 `ONB-001 -> ONB-003 -> ONB-004 -> OS 위치 권한 시스템 팝업 -> ONB-005 -> MAP-01`이고, 저시력자 플로우는 `ONB-001 -> LV-ONB-01~05(TermsGuide) -> OS 권한 요청 -> LV-HOM-01`이다. `ONB-002 low_vision_followup` route는 기본 회원가입 플로우가 아니다. 회원가입 완료(`POST /auth/signup`)는 튜토리얼 종료 이후가 아니라 권한 요청 전후 gate 흐름에서 처리된다. |
| 사용자 용어 | 온보딩 1차 분기 용어는 `저시력자`, `보행약자`로 통일한다. 보행약자 2차 선택지는 `전동휠체어 사용자`, `수동휠체어 사용자`, `기타 보행약자`다. |
| 약관/권한 구분 | `개인위치정보 수집·이용 동의`는 `ONB-004`의 법적 동의 항목이고, `OS 위치 권한 허용`은 `ONB-004` 다음 단계의 시스템 팝업이다. |
| 하단 탭 | 보행약자 기본 하단 탭은 `홈 / 북마크 / 제보 / 마이페이지` 4탭이다. 저시력자 전용 UI는 별도 `LV-*` 화면군에서 `홈 / 북마크 / 카테고리 / 마이페이지`를 사용한다. |
| 지도 필터 | 빠른 필터는 `장애인 화장실`, `엘리베이터`, `전동보장구 충전 가능` 접근성 속성이다. 더보기는 장소 카테고리 7개를 노출한다. |
| 경로 구조 | `ROUTE-01`은 경로 지도/탐색 결과, `ROUTE-02`는 경로 상세, `NAV-01`은 길안내 진행 화면이다. |
| 경로 옵션 | `ROUTE-01`은 도보/대중교통 탭을 제공한다. 도보는 `안전한 길`, `최단 경로`를 사용하고, 대중교통은 `추천 경로`, `최소 환승`, `최소 도보`를 사용한다. |
| 북마크 명칭 | 사용자 노출 명칭은 `북마크`로 통일한다. `BMK-01`은 장소 탭, `BMK-02`는 경로 탭이다. |
| 제보 구조 | 제보는 `REPORT-01`~`REPORT-04` 4단계 플로우다. |
| 데이터 책임 | 서버 데이터가 기준이며 로컬은 `cache/offline/pending` 보조 역할로만 사용한다. `local-first`를 production 핵심 정책으로 서술하지 않는다. |

## 1. 인증 및 온보딩

| 화면 ID | 핵심 기능 | 주요 전환 규칙 | 데이터/정책 |
| --- | --- | --- | --- |
| `AUTH-001` | 소셜 로그인으로 기존 회원 세션 또는 신규 사용자 `signupToken`을 받는다. | 기존 회원은 사용자 유형에 따라 홈으로 이동하고, 신규 사용자는 `ONB-001`로 이동한다. | 카카오/네이버/구글만 제공한다. guest mode와 비로그인 우회는 제공하지 않는다. |
| `AUTH-002` | 로그인 후 프로필 완료 여부가 false인 사용자의 닉네임과 기본 사용자 유형을 확정한다. | 완료 후 gate 결과에 따라 `ONB-001`, `LV-HOM-01`, `MAP-01`로 이동한다. | current runtime의 `auth/profile_setup` gate다. |
| `ONB-001` | 신규 사용자의 사용자 유형 1차 선택을 받는다. | `저시력자` 선택 시 `LV-ONB-01`, `보행약자` 선택 시 `ONB-003`으로 이동한다. | 소셜 인증 후 받은 `signupToken`이 있을 때 진입한다. |
| `ONB-002` | 과거 저시력 후속 단계 route를 보관한다. | current signup flow 기본 경로에서는 사용하지 않는다. | `onboarding/low_vision_followup` route는 남아 있으나 기본 진입은 `LV-ONB-01`이다. |
| `ONB-003` | 보행약자 2차 선택을 받는다. | 사용자 유형 선택 후 `다음`으로 `ONB-004`로 이동한다. | 선택지는 `전동휠체어 사용자`, `수동휠체어 사용자`, `기타 보행약자`만 사용한다. `장애등급`은 이 단계에 포함하지 않는다. 서버에는 `selectedMobilitySubtype` 한 개 값으로 저장한다. |
| `ONB-004` | 보행약자 기본 플로우의 필수 약관/동의를 수집하고 전문 확인 동선을 제공한다. | 필수 동의 완료 후 `다음` 선택 시 `OS 위치 권한 시스템 팝업`을 띄우고, 응답 이후 `ONB-005`로 이동한다. | 저시력자 기본 플로우는 `LV-ONB-01`~`05` 순차 동의로 대체한다. 서버에는 별도 약관 동의 필드를 저장하지 않고 가입 완료 조건으로만 확인한다. |
| `ONB-005` | 위치 권한 응답 이후 보행약자 튜토리얼을 제공하고 온보딩을 완료한다. | 튜토리얼 완료 후 `MAP-01`로 이동한다. | current runtime의 route는 `tutorial/onboarding`이며 저시력자 기본 플로우는 이 단계를 거치지 않는다. |

### 1.1 온보딩 공통 규칙

| 항목 | 기준 |
| --- | --- |
| 단계 수 | `ONB-001`~`ONB-005` 5단계로 고정한다. |
| 용어 | 온보딩 1차 분기는 `저시력자`, `보행약자`로 통일하고, 보행약자 2차 선택은 `전동휠체어 사용자`, `수동휠체어 사용자`, `기타 보행약자`로 통일한다. |
| 약관/권한 | `ONB-004`의 법적 동의와 그 다음 단계의 `OS 위치 권한 시스템 팝업`은 분리해 서술한다. |
| 상태 표현 | 선택, 오류, 비활성, 완료 상태는 색상만이 아니라 텍스트와 semantics를 함께 제공한다. |
| 재진입 | 사용자 유형 변경은 신규 가입용 `ONB-001`이 아니라 profile edit branch(`onboarding/profile_user_type_primary`)를 사용한다. 보행약자 2차 선택 수정은 `PATCH /users/me/user-type`으로 `selectedMobilitySubtype`에 저장한다. |

### 1.2 저시력자 전용 음성 중심 UI

| 화면 ID | 역할 | 주요 전환 |
| --- | --- | --- |
| `LV-ONB-01` | 저시력자 약관 동의 | `LV-ONB-02` |
| `LV-ONB-02` | 저시력자 민감정보 동의 | `LV-ONB-03` |
| `LV-ONB-03` | 저시력자 개인위치정보 동의 | `LV-ONB-04` |
| `LV-ONB-04` | 저시력자 만 14세 이상 확인 | `LV-ONB-05` |
| `LV-ONB-05` | 저시력자 개인정보 처리방침 확인 | OS 위치 권한 시스템 팝업 -> `LV-HOM-01` |
| `LV-HOM-01` | 저시력자 홈 | `LV-SEA-01`, `LV-BMK-01`, `LV-CAT-01`, `LV-MYP-01` |
| `LV-SEA-01` | 저시력자 음성 입력 진행 | `LV-RES-01`, `LV-HOM-01` |
| `LV-RES-01` | 저시력자 장소 결과 목록 | `LV-NAV-01`, `LV-HOM-01` |
| `LV-CAT-01` | 저시력자 카테고리 | `LV-RES-01`, `LV-HOM-01` |
| `LV-BMK-01` | 저시력자 북마크 | `LV-BRF-01`, `LV-NAV-01`, `LV-HOM-01` |
| `LV-BRF-01` | 저시력자 경로 브리핑 | `LV-NAV-01`, `LV-BMK-01` |
| `LV-NAV-01` | 저시력자 길안내 진행 | `LV-ARR-01`, `LV-HOM-01` |
| `LV-ARR-01` | 저시력자 도착 완료 | `LV-HOM-01` |
| `LV-MYP-01` | 저시력자 마이페이지 | `ONB-001`, `AUTH-001`, 앱 정보 상세 |

| 규칙 | 기준 |
| --- | --- |
| 별도 화면 원칙 | 저시력자 UI는 공통 `MAP-01`, `DEST-02`, `ROUTE-02`, `NAV-01`, `BMK-01`, `MYP-01`의 모드 예외가 아니라 별도 `LV-*` 화면군으로 다룬다. |
| 동의 방식 | `LV-ONB-01`~`LV-ONB-05`는 `onboarding/terms_guide/{step}` route로 구현되며, 각 화면 중앙 대형 버튼 더블탭으로 다음 화면으로 이동한다. |
| 위치 권한 시점 | `LV-ONB-05` 완료 후 `OS 위치 권한 시스템 팝업`을 띄운다. 법적 동의와 시스템 권한은 분리 유지한다. |
| 튜토리얼 연결 | 저시력자 기본 플로우는 OS 위치 권한 응답 후 `LV-HOM-01`로 바로 진입한다. `ONB-005`는 보행약자 튜토리얼로만 사용한다. |

## 2. 지도 홈, 시설, 목적지 설정

| 화면 ID | 핵심 기능 | 주요 전환 규칙 | 데이터/정책 |
| --- | --- | --- | --- |
| `MAP-01` | 지도 홈에서 검색, 시설 탐색, 최근 목적지, 하단 4탭 진입점을 제공한다. | 검색바는 `DEST-01`, 시설 마커는 `FAC-01`, 승인 제보 마커는 승인 제보 바텀시트, 북마크 탭은 `BMK-01`, 제보 탭은 `REPORT-01`, 마이페이지 탭은 `MYP-01`로 이동한다. | 홈 탭 활성 상태다. 위치 권한이 없으면 기능 제한 안내를 제공한다. 승인된 공개 제보만 지도 마커로 노출한다. |
| `FAC-01` | 시설 상세 바텀시트에서 시설 정보, 접근성 라벨, 북마크, 길찾기를 제공한다. | `길찾기` 선택 시 `DEST-03` 확인 후 `ROUTE-01`로 연결한다. | 지도 또는 북마크 장소 상세에서 진입한다. 장소 식별자는 내부 `placeId`를 기준으로 하며, 카카오 검색 결과는 `provider=KAKAO`, `providerPlaceId`로 조회/생성한다. |
| `DEST-01` | 목적지 검색과 최근 방문지 목록을 제공한다. | 검색 실행 시 `DEST-02`, 최근 방문지 선택 시 `DEST-03`, `전체보기` 선택 시 `BMK-01`로 이동한다. | 하단 탭은 유지한다. 마이크 버튼은 `POST /voice/analyze` 기반 음성 장소 검색 진입점이다. |
| `DEST-02` | 검색 결과를 카드 목록으로 보여주고 후보를 고르게 한다. | 결과 카드 선택 시 항상 `DEST-03`으로 이동한다. | 검색 결과에서 곧바로 `ROUTE-01`에 진입하지 않는다. |
| `DEST-03` | 지도와 목적지 바텀시트로 도착지를 최종 확인한다. | `도착지로 설정` 선택 시 현재 구현은 `ROUTE-01`로 이동한다. | 장소 식별자, 좌표, 주소를 일관되게 유지해야 한다. |
| `DEST-04` | 첫 사용 시 현재 이동 상태 설문을 제공하는 후순위 후보 화면이다. | 제출용 현재 구현에는 포함하지 않는다. | 표시 조건, 저장 위치, 온보딩 사용자 유형과의 우선순위가 미정이므로 향후 개선 항목으로 둔다. |

### 2.1 지도 및 검색 공통 규칙

| 항목 | 기준 |
| --- | --- |
| 접근성 필터 | `MAP-01` 빠른 필터는 `장애인 화장실`, `엘리베이터`, `전동보장구 충전 가능`이다. 더보기는 음식·카페, 관광지, 숙박, 의료·보건, 복지·돌봄, 공공기관, 기타 카테고리를 제공한다. |
| 최근 목적지 | 별도 전체 목록 화면을 만들지 않고 `BMK-01`로 연결한다. |
| STT/음성 분석 | FE가 STT로 텍스트를 만들고 `POST /voice/analyze`를 호출한다. 보행약자는 추출된 `placeName`으로 바로 `GET /places/search`를 호출하고, 저시력자는 TTS 확인 후 `confirmed=true`일 때만 검색한다. |
| 장소 데이터 | 검색 결과와 시설 상세는 장소 식별자, 좌표, 주소를 일관되게 내려줘야 한다. |

### 2.2 저시력자 전용 별도 화면군 지도/검색 규칙

| 항목 | 기준 |
| --- | --- |
| 홈 화면 | 저시력자 전용 지도 홈은 `MAP-01`이 아니라 `LV-HOM-01`이다. |
| 홈 기본 CTA | `LV-HOM-01`은 `음성 입력`, `현재 위치` 2개 대형 버튼을 우선 노출한다. |
| 하단 탭 | `LV-HOM-01`은 `홈 / 북마크 / 카테고리 / 마이페이지`를 사용한다. |
| 카테고리 구성 | `LV-CAT-01`은 `화장실`, `음식점`, `숙박시설`, `병원` 4개 카드로 구성한다. |
| 결과 화면 | 검색 결과와 카테고리 결과는 모두 `LV-RES-01`을 사용한다. 첫 화면에서는 2개 카드만 크게 보여주고 스크롤로 추가 결과를 본다. |
| 결과 카드 CTA | `LV-RES-01` 카드마다 `북마크`, `길찾기`를 제공한다. |
| 길찾기 경로 | `LV-RES-01`의 `길찾기`는 `LV-NAV-01` 즉시 진입을 기준으로 한다. |

## 3. 경로 탐색, 길안내, 도착, 평가

| 화면 ID | 핵심 기능 | 주요 전환 규칙 | 데이터/정책 |
| --- | --- | --- | --- |

## 8. MVP 제외 또는 후순위

- `SCR-001`~`SCR-003`, `ONB-01`~`ONB-03` 같은 구 온보딩 구조
- 운영 기준 `비로그인으로 계속`, guest mode, 비로그인 지도/경로/북마크/제보 진입
- 3탭 또는 5탭 후보 구조
- 저시력자 전용 후속 화면을 별도 `VIS` 메인 플로우로 대체해 서술하는 정책
- 사용자 노출 명칭으로 `저장`, `Saved`, `자주 가는 길`을 기본 화면명처럼 사용하는 정책
- 북마크 전체 선택 일괄 처리와 고급 정렬 조건
- 별도 제보 처리 상태 전용 화면
- 고객센터/1:1 문의 실제 연결
- 길안내 중도 종료 정책 확정
- `local-first`나 로컬 source of truth를 production 핵심 정책으로 서술하는 방식

## 9. 미확정 항목

| 영역 | 내용 |
| --- | --- |
| 로그인 UX 순서 | 기존 회원은 `authSession -> profileCompleted gate -> selectedPrimaryUserType 홈 분기`로 이동한다. 신규 사용자는 소셜 인증 후 `signupToken`으로 `ONB-001 사용자 유형 1차 선택`에 진입하고, 가입 완료는 권한 요청 전후 gate 흐름에서 처리된다. |
| 권한 | `ONB-004`에서 사용자가 `다음`을 누른 뒤 다음 단계에서 `OS 위치 권한 시스템 팝업`을 띄우는 것으로 고정한다. |
| 저시력자 후속 화면 | `ONB-002`는 legacy route이며, current runtime 기본 플로우는 `LV-ONB-01`~`LV-ONB-05` 순차 동의를 사용한다. legacy route 정리 여부는 추가 확인이 필요하다. |
| 저시력자 전용 상세 | `LV-MYP-01`의 앱 정보 상세를 기존 `MYP-03` 재사용으로 볼지 별도 `LV-*`로 분리할지는 추가 확인이 필요하다. |
| 목적지 설문 | `DEST-04`는 현재 구현에 포함하지 않는 후순위 후보 화면이다. 노출 조건, 저장 위치, 온보딩 사용자 유형과의 우선순위가 미정이다. |
| 길안내 종료 | `NAV-01` 중도 종료 후 `MAP-01`, `ROUTE-01`, 이전 화면 중 어디로 이동할지 미정이다. |
| 경로 평가 | 평가 실패 처리와 중복 평가 허용 여부는 미정이다. |
| 대중교통/이동지원 | 대중교통 경로, 저상버스 예약 후보, 두리발 연결 UI는 FE에 구현되어 있다. 실제 외부 연동 성공 기준, 데이터 최신성, 실패 정책은 백엔드 확인이 필요하다. |
| 프로필 매핑 | 개인정보는 노출하지 않는다. 소셜 이름/닉네임, 1차 사용자 유형, 보행약자 세부 유형 표시 문구만 UX 확정이 필요하다. |
| 활동 배지 | `제보왕`, `도움왕`, `참여왕` 산정 기준이 미정이다. |
| 알림 | 푸시/진동 설정은 서버 저장 범위에서 제외한다. 앱 내부 설정 노출 여부만 UX 확정이 필요하다. |
| 로그아웃/탈퇴 | 회원탈퇴 재인증 방식은 미정이다. 탈퇴 시 북마크 데이터는 삭제하고, 제보/평가 데이터는 운영 이력 보존을 위해 유지한다. 경로 저장/안내 세션 처리 범위는 백엔드 계정 정책에 맞춰 확인한다. |
