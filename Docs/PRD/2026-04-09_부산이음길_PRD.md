## 1. 제품 개요

### 1.1 한 줄 정의

부산이음길은 보행약자가 부산 지역에서 목적지까지 이동할 때 접근성 정보, 경로 비교, 장애물 제보를 하나의 흐름에서 확인할 수 있도록 돕는 무장애 길안내 서비스다.

### 1.2 해결하려는 문제

- 기존 길찾기 서비스는 일반 보행자 기준이어서 경사, 계단, 단차, 엘리베이터, 공사/통제 같은 이동 장벽을 경로 선택 기준으로 충분히 반영하지 못한다.
- 보행약자는 처음 가는 목적지에서 접근 가능한 시설과 실제 이동 위험 요소를 동시에 파악하기 어렵다.
- 현장 장애물 정보는 빠르게 변하지만 사용자 제보와 경로 탐색 흐름이 분리되어 있으면 반영 속도가 떨어진다.

### 1.3 제품 목표

| 목표 | 설명 |
| --- | --- |
| 접근 가능한 길찾기 | `안전한 길`, `최단거리` 2개 경로 옵션으로 빠르게 경로를 비교하게 한다. |
| 일관된 진입 경험 | 필수 로그인 후 `ONB-001` 진입, 온보딩 완료 뒤 `저시력자 -> LV-HOM-01`, `보행약자 -> MAP-01`로 분기하는 흐름을 유지한다. |
| 제보와 길찾기 연결 | `REPORT-01`~`REPORT-04` 제보 플로우와 `MYP-02` 제보 내역을 일관되게 연결한다. |
| 문서/제품 정합성 | 화면 ID, 용어, CTA, 데이터 책임, MVP 범위를 문서 전반에서 동일하게 유지한다. |

## 2. 타겟 사용자와 용어

| 구분 | 기준 |
| --- | --- |
| 온보딩 1차 사용자 용어 | `저시력자`, `보행약자` |
| 보행약자 2차 선택 | `전동휠체어 사용자`, `수동휠체어 사용자`, `기타 보행약자` |
| 포함 예시 | 전동/수동 휠체어 사용자, 보행 보조기구 사용자, 고령자, 유아차 동반 보호자, 일시적 이동 불편자 |
| 후속 접근성 확장 | 저시력자 핵심 UI는 별도 `LV-*` 화면군으로 MVP 기준선에 포함한다. STT/LLM/TTS 고도화, 자동 브리핑 강화만 후속 확장 후보로 둔다. |

`장애등급`은 온보딩 필수 단계와 서버 스키마에서 제거한다. 세부 이동 특성은 서버에 `selectedMobilitySubtype` 한 개 값으로 저장한다.

## 3. 제품 원칙

| 항목 | 원칙 |
| --- | --- |
| 앱 진입 gate | 인증 세션이 없으면 `AUTH-001`로 이동한다. 기존 회원은 `authSession -> selectedPrimaryUserType 홈 분기`로 이동하고, 신규 사용자는 소셜 인증 후 `signupToken`으로 온보딩을 완료한 뒤 회원가입을 생성한다. |
| 로그인 정책 | 카카오/네이버/구글 소셜 로그인만 제공한다. 운영 기준 `비로그인으로 계속`, guest mode, 비로그인 핵심 기능 사용은 제거한다. |
| 온보딩 정책 | 온보딩은 `ONB-001 사용자 유형 1차 선택` -> `ONB-002 저시력자 전용 후속 화면` 또는 `ONB-003 보행약자 2차 선택` -> `ONB-004 약관/동의` -> `OS 위치 권한 시스템 팝업` -> `ONB-005 튜토리얼` 5단계다. 1차 사용자 유형은 서버 `selectedPrimaryUserType`에 저장하고, 보행약자 2차 선택값은 보행약자일 때만 서버 `selectedMobilitySubtype` 한 개 값으로 저장한다. 장애등급 필수 입력 구조는 사용하지 않는다. |
| 약관/권한 구분 | `개인위치정보 수집·이용 동의`는 법적 동의 항목이고, `OS 위치 권한 허용`은 다음 단계의 시스템 팝업이다. |
| 화면 구조 | 보행약자 기본 하단 탭은 `홈 / 북마크 / 제보 / 마이페이지` 4탭이다. 저시력자 전용 UI는 별도 화면군 `LV-*`로 분리하고 `홈 / 북마크 / 카테고리 / 마이페이지`를 사용한다. |
| 지도 필터 정책 | 상단 빠른 필터는 장애인 화장실, 엘리베이터, 전동보장구 충전 가능 접근성 속성이다. 더보기는 장소 카테고리 7개를 노출한다. |
| 경로 구조 | `ROUTE-01`은 경로 지도/탐색 결과, `ROUTE-02`는 경로 상세, `NAV-01`은 길안내 진행 화면으로 분리한다. |
| 경로 옵션 | 사용자 노출 경로 옵션은 `안전한 길`, `최단거리` 2개만 사용한다. |
| 북마크 정책 | 사용자 노출 명칭은 `북마크`다. `BMK-01`은 장소 탭, `BMK-02`는 경로 탭이다. |
| 제보 정책 | 제보는 `REPORT-01`~`REPORT-04` 4단계 플로우다. 제보는 사용자 계정과 연결하며 내 제보 내역에서 처리 상태를 확인할 수 있다. 지도에는 승인된 공개 제보만 마커로 노출한다. 사진은 선택이며 최대 5장이다. |
| 데이터 책임 | 서버 데이터가 기준이며 로컬은 `cache/offline/pending` 보조 역할로만 사용한다. |
| 미확정 처리 | 화면, 정책, API, enum, 저장 위치가 불명확하면 추정하지 않고 `미확정`, `TBD`, `BE 확인 필요`로 남긴다. |

## 4. MVP 범위

### 4.1 포함 범위

| 영역 | 화면 ID | 포함 내용 |
| --- | --- | --- |
| 인증 | `AUTH-001` | 필수 소셜 로그인 |
| 온보딩 | `ONB-001`~`ONB-005` | 사용자 유형 1차 선택, 저시력자 전용 후속 화면 또는 보행약자 2차 선택, 약관/동의, OS 위치 권한 시스템 팝업 이후 튜토리얼 |
| 지도 홈 | `MAP-01`, `FAC-01` | 검색바, STT 진입 버튼, 접근성 필터, 시설 마커/상세, 최근 목적지, 하단 4탭을 포함한 보행약자 공통 홈 화면이다. |
| 저시력자 전용 화면군 | `LV-ONB-01`~`LV-MYP-01` | 5개 약관 안내, `음성 입력`/`현재 위치` 홈, 카테고리 4카드, 2카드 검색/북마크 결과, 경로 브리핑, 길안내, 도착지 저장/완료, 3버튼 마이페이지를 포함한다. |
| 목적지 설정 | `DEST-01`~`DEST-04` | 검색, 검색 결과, 지도/바텀시트 확인, 사용자 상태 설문 |
| 경로/길안내 | `ROUTE-01`, `ROUTE-02`, `NAV-01` | 경로 비교, 경로 상세, 실시간 길안내 |
| 도착/평가 | `ARR-01`, `ARR-02` | 도착 완료, 방금 안내받은 경로 별점 평가, 경로 저장 |
| 북마크 | `BMK-01`, `BMK-02` | 장소/경로 북마크 조회, 삭제, 저장 경로 재탐색 연결 |
| 제보 | `REPORT-01`~`REPORT-04` | 유형 선택, 위치 확인, 상세 입력, 접수 완료 |
| 마이페이지 | `MYP-01`~`MYP-03` | 프로필 요약, 제보 내역, 앱 정보 및 도움말, 로그아웃/회원탈퇴 동선 |

### 4.2 제외 또는 후순위

- `비로그인으로 계속`, guest mode, 데모 모드 기반 메인 흐름
- `SCR-001`~`SCR-003`, `ONB-01`~`ONB-03` 같은 구 온보딩 구조
- 3탭 또는 5탭 후보 구조
- 사용자 노출 화면명으로 `저장`, `Saved`, `자주 가는 길`을 기본 명칭처럼 사용하는 정책
- 저시력자 핵심 UI를 후순위 확장이나 레거시 참고 흐름으로 축소 서술하는 정책
- 북마크 정렬, 전체 선택, 다중 선택
- 별도 제보 처리 상태 전용 화면
- 고객센터/1:1 문의 실제 연결
- `local-first`나 로컬 source of truth를 production 핵심 정책으로 서술하는 방식

## 5. 핵심 사용자 흐름

### 5.1 앱 진입

```text
앱 실행
  -> authSession 확인
  -> 세션 없음: AUTH-001
  -> 신규 사용자 signupToken 있음: ONB-001 또는 미완료 온보딩 단계
  -> 세션 있음 + selectedPrimaryUserType=LOW_VISION: LV-HOM-01
  -> 세션 있음 + selectedPrimaryUserType=MOBILITY_IMPAIRED: MAP-01
```

### 5.2 목적지 탐색과 길안내

```text
MAP-01 -> DEST-01 -> DEST-02 -> DEST-03 -> DEST-04(필요 시) -> ROUTE-01 -> ROUTE-02(선택) -> NAV-01 -> ARR-01 -> ARR-02
```

| 단계 | 기준 |
| --- | --- |
| 검색 결과 선택 | `DEST-02`에서 선택해도 반드시 `DEST-03` 목적지 확인을 거친다. |
| 사용자 상태 설문 | `DEST-04` 노출 조건은 미정이다. |
| 경로 비교 | `ROUTE-01`에서는 `안전한 길`, `최단거리` 2개만 비교한다. |
| 길안내 시작 | `ROUTE-01` 또는 `ROUTE-02`의 `길 안내 시작`으로 `NAV-01`에 진입한다. |
| 도착/평가 | `ARR-01` 진입 즉시 `ARR-02` 평가 모달을 자동 표시한다. 평가는 별점만 저장한다. |

### 5.3 북마크

```text
MAP-01 -> BMK-01 <-> BMK-02
```

| 단계 | 기준 |
| --- | --- |
| 장소 북마크 | 카드 본문은 `FAC-01`, `길찾기`는 `DEST-03` 확인 후 `ROUTE-01`로 연결한다. |
| 경로 북마크 | 안내 종료 후 저장된 route snapshot으로 선택 당시 경로 상세를 다시 보여주고, 저장된 출발지/도착지/옵션으로 최신 경로 재탐색도 가능하게 한다. |

### 5.4 제보

```text
MAP-01 -> REPORT-01 -> REPORT-02 -> REPORT-03 -> REPORT-04 -> MYP-02
```

| 단계 | 기준 |
| --- | --- |
| 유형 선택 | 6개 제보 유형 중 하나를 선택한다. |
| 위치 확인 | pin drag로 위치를 확정한다. |
| 상세 입력 | 사진은 갤러리/카메라 모두 허용한다. 선택 입력이며 최대 5장이다. |
| 완료 | `제보 내역 확인하기`는 `MYP-02`로 연결한다. |

### 5.5 저시력자 전용 별도 화면군

| 화면 ID | 화면명 | 역할 | 주요 이동 |
| --- | --- | --- | --- |
| `LV-ONB-01` | 저시력자 약관 동의 | 1번째 약관 안내 화면 | `LV-ONB-02` |
| `LV-ONB-02` | 저시력자 민감정보 동의 | 2번째 약관 안내 화면 | `LV-ONB-03` |
| `LV-ONB-03` | 저시력자 개인위치정보 동의 | 3번째 약관 안내 화면 | `LV-ONB-04` |
| `LV-ONB-04` | 저시력자 만 14세 이상 확인 | 4번째 약관 안내 화면 | `LV-ONB-05` |
| `LV-ONB-05` | 저시력자 개인정보 처리방침 확인 | 5번째 약관 안내 화면 | OS 위치 권한 시스템 팝업, `ONB-005` |
| `LV-HOM-01` | 저시력자 홈 | `음성 입력`, `현재 위치`, 하단 `홈/북마크/카테고리/마이페이지` 진입 | `LV-SEA-01`, `LV-BMK-01`, `LV-CAT-01`, `LV-MYP-01` |
| `LV-SEA-01` | 저시력자 음성 입력 진행 | 음성 인식 대기/입력 진행 | `LV-RES-01`, `LV-HOM-01` |
| `LV-RES-01` | 저시력자 장소 결과 목록 | 검색/카테고리 공통 2카드 결과 목록 | `LV-NAV-01`, `LV-HOM-01` |
| `LV-CAT-01` | 저시력자 카테고리 | `화장실 / 음식점 / 숙박시설 / 병원` 4카드 | `LV-RES-01`, `LV-HOM-01` |
| `LV-BMK-01` | 저시력자 북마크 | 2카드 북마크 목록, `북마크(해제)` / `길찾기` | `LV-BRF-01`, `LV-NAV-01`, `LV-HOM-01` |
| `LV-BRF-01` | 저시력자 경로 브리핑 | 경로 단계 요약, 음성 재생, 대형 `시작/중지` | `LV-NAV-01`, `LV-BMK-01` |
| `LV-NAV-01` | 저시력자 길안내 진행 | `남은 거리`, `남은 시간`, `현재 위치`, `안내 종료` 중심 안내 | `LV-ARR-01`, `LV-HOM-01` |
| `LV-ARR-01` | 저시력자 도착 완료 | `도착지 저장`, `완료` | `LV-HOM-01` |
| `LV-MYP-01` | 저시력자 마이페이지 | `모드변경`, `로그아웃`, `앱정보` 3버튼 | `ONB-001`, `AUTH-001`, 앱 정보 상세 |

| 규칙 | 기준 |
| --- | --- |
| 별도 화면 원칙 | 저시력자 UI는 공통 `MAP-01`, `DEST-02`, `ROUTE-02`, `NAV-01`, `BMK-01`, `MYP-01`의 모드 예외가 아니라 별도 화면군으로 관리한다. |
| 동의 제스처 | `LV-ONB-01`~`LV-ONB-05`는 각 화면 중앙 대형 버튼 더블탭으로 다음 화면으로 이동한다. |
| 법적 동의와 시스템 권한 | `개인위치정보 수집·이용 동의`는 `LV-ONB-03`의 법적 동의 항목이고, 그 다음 단계의 OS 팝업은 별개다. |
| 결과 화면 | 검색 결과와 카테고리 결과는 모두 `LV-RES-01`을 사용한다. |
| 경로 브리핑 진입 | 북마크 카드의 빈 영역 더블탭은 `LV-BRF-01`로 이동한다. |
| 길찾기 진입 | `LV-RES-01`, `LV-BMK-01`의 `길찾기`는 기본 추천 경로로 `LV-NAV-01`에 즉시 진입한다. |

## 6. 기능 요구사항 요약

| 기능 영역 | 사용자 가치 | 핵심 요구사항 |
| --- | --- | --- |
| 인증 | 로그인된 사용자 기준으로 개인화와 저장 기능을 일관되게 제공한다. | `AUTH-001` 필수 로그인, 세션 유지/재발급/로그아웃 처리 |
| 온보딩 | 사용자의 이동 특성을 초기에 정리해 경로 추천 기준을 만든다. | `ONB-001`~`ONB-005` 5단계, 사용자 유형 1차 선택, 저시력자 전용 후속 화면 또는 보행약자 2차 선택, 약관/동의, 튜토리얼 |
| 지도 홈 | 탐색과 주요 진입점을 하나의 홈 화면에서 제공한다. | `MAP-01` 검색, 필터, 최근 목적지, 시설 상세, 하단 4탭 |
| 목적지 설정 | 검색 결과를 바로 경로로 연결하지 않고 목적지를 확인한다. | `DEST-01`~`DEST-04`, 검색 결과 후 `DEST-03` 확인 필수 |
| 경로/길안내 | 사용자가 비교 가능한 경로 후보와 실시간 길안내를 제공한다. | `ROUTE-01`, `ROUTE-02`, `NAV-01`, 2개 경로 옵션, 재탐색 후보 |
| 도착/평가 | 도착 직후 만족도와 경로 저장을 자연스럽게 수집한다. | `ARR-01`, `ARR-02`, 별점 평가, 경로 저장 |
| 북마크 | 자주 찾는 장소와 경로를 빠르게 다시 사용하게 한다. | `BMK-01`, `BMK-02`, 장소/경로 분리, 삭제만 허용 |
| 제보 | 장애물 정보를 구조화된 흐름으로 수집한다. | `REPORT-01`~`REPORT-04`, 계정 연결, 내 제보 상태 표시, 승인 공개 제보 지도 마커 |
| 마이페이지 | 계정과 사용자 상태 변경, 제보 확인, 도움말을 제공한다. 저시력자 전용 별도 화면군에서는 `LV-MYP-01` 3버튼 구조를 사용한다. | `MYP-01`~`MYP-03`, `LV-MYP-01`, 사용자 유형 변경, 로그아웃/탈퇴 동선 |

## 7. 데이터 및 연동 책임

| 데이터 영역 | 기준 데이터 책임 | 로컬 역할 | 비고 |
| --- | --- | --- | --- |
| 인증 세션 | 서버 인증 응답 기준 | secure storage/token cache | 로그아웃 시 앱 세션 토큰을 삭제한다. |
| 회원가입 완료 상태 | 서버 `users` row 존재 여부 기준 | 온보딩 재개용 보조 상태 | 단계별 완료 상태와 별도 완료 boolean은 서버에 저장하지 않는다. |
| 사용자 유형 | 서버 사용자 설정 기준 | 현재 세션 선택 상태 보조 | `selectedPrimaryUserType`은 저시력자/보행약자 홈 분기값이다. `LOW_VISION`이면 `selectedMobilitySubtype=null`, `MOBILITY_IMPAIRED`이면 보행약자 2차 선택을 `users.selectedMobilitySubtype`에 `POWER_WHEELCHAIR`, `MANUAL_WHEELCHAIR`, `OTHER_MOBILITY` 중 하나로 저장한다. |
| 장소/경로 응답 | 서버 및 외부 연동 응답 기준 | 현재 세션 캐시 | 식별자, 좌표, 주소 일관성이 필요하다 |
| 북마크 장소/경로 | 서버 데이터 기준 | `cache/offline/pending` | 장소 북마크와 경로 북마크를 분리한다. 경로 북마크는 안내 종료 후 route session의 좌표/옵션/snapshot을 복사 저장한다. |
| 제보 draft | 제출 전 로컬 임시 상태 | draft/pending | 제출 이후에는 서버 제보 이력이 기준이다 |
| 제보 이력 | 서버 사용자 제보 데이터 기준 | 목록 캐시 | 내 제보 목록/상세에서 처리 상태 표시 |
| 경로 평가 | 서버 `route_ratings` 기준 | 제출 전 UI 임시 상태 | 방금 안내받은 경로에 별점만 저장한다. |

## 8. 성공 지표와 품질 기준

| 구분 | 기준 |
| --- | --- |
| 핵심 흐름 완성도 | `보행약자: AUTH-001 -> ONB-001 -> ONB-003 -> ONB-004 -> ONB-005 -> MAP-01 -> DEST-01~04 -> ROUTE-01 -> NAV-01 -> ARR-01/02`, `저시력자: AUTH-001 -> ONB-001 -> ONB-002 -> LV-ONB-01~05 -> ONB-005 -> LV-HOM-01 -> LV-RES-01/LV-BMK-01 -> LV-NAV-01 -> LV-ARR-01` 흐름이 end-to-end로 동작해야 한다. |
| 탐색/제보 완성도 | 장소 북마크, 경로 북마크, 제보 4단계, 제보 내역 조회가 서로 끊기지 않아야 한다. |
| 접근성 | 핵심 화면에서 TalkBack 라벨, 명확한 포커스 순서, 색상 외 상태 전달을 보장한다. |
| 문서 정합성 | 화면 ID, 용어, 탭 구조, 데이터 책임, 미확정 항목이 화면명세서/기능명세서/PRD에서 동일해야 한다. |

## 9. 미확정 항목

| 영역 | 내용 |
| --- | --- |
| 로그인 UX 순서 | 기존 회원은 `authSession -> selectedPrimaryUserType 홈 분기`로 이동한다. 신규 사용자는 소셜 인증 후 `signupToken`으로 `ONB-001 사용자 유형 1차 선택`에 진입하고, 온보딩 완료 시 `POST /auth/signup` 성공 후 홈으로 이동한다. |
| 권한 | `ONB-004`에서 사용자가 `다음`을 누른 뒤 다음 단계에서 `OS 위치 권한 시스템 팝업`을 띄운다. |
| 저시력자 후속 화면 | `ONB-002`의 상세 UI와 저장 스키마는 추가 확인이 필요하다. 현재는 화면 존재와 흐름 연결을 기준선으로 사용한다. |
| 저시력자 전용 UI 범위 | 저시력자 전용 UI는 별도 `LV-*` 화면군으로 분리한다. `LV-MYP-01`의 앱 정보 상세를 기존 `MYP-03` 재사용으로 볼지 별도 `LV-*`로 분리할지는 추가 확인이 필요하다. |
| 목적지 설문 | `DEST-04` 노출 조건과 저장 우선순위는 미정이다. |
| 길안내 종료 | `NAV-01` 중도 종료 후 이동 정책은 미정이다. |
| 경로 평가 | 평가 실패 처리와 중복 평가 허용 여부는 미정이다. |
| 프로필 매핑 | 개인정보는 노출하지 않는다. 소셜 이름/닉네임, 1차 사용자 유형, 보행약자 세부 유형 표시 문구만 UX 확정이 필요하다. |
| 활동 배지 | `제보왕`, `도움왕`, `참여왕` 산정 기준은 미정이다. |
| 알림 | 푸시/진동 설정은 서버 저장 범위에서 제외한다. 앱 내부 설정 노출 여부만 UX 확정이 필요하다. |
| 로그아웃/탈퇴 | 회원탈퇴 재인증 방식은 미정이다. 탈퇴 시 북마크 데이터는 삭제하고 제보/평가 데이터는 운영 이력 보존을 위해 유지한다. 경로 저장/안내 세션 처리 범위는 백엔드 계정 정책에 맞춰 확인한다. |
