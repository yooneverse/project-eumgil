## 커밋 컨벤션

### 커밋 메시지 형식

```text
{gitmoji} {Type}[S14P31E102-{이슈번호}]: {내용}
```

**예시:**
```bash
✨ Feat[S14P31E102-31]: docker-compose.dev.yml 생성
🐛 Fix[S14P31E102-32]: RDS 커넥션 타임아웃 수정
♻️ Refactor[S14P31E102-33]: Celery 태스크 구조 개선
🔧 Chore[S14P31E102-41]: .gitignore 업데이트
📝 Docs[S14P31E102-42]: API 명세서 작성
```

커밋 메시지에는 타입별 gitmoji와 Jira 전체 키를 함께 작성합니다. Jira 키는 `S14P31E102-{이슈번호}` 형식으로 직접 작성하며, `[#31]`처럼 숫자만 쓰는 축약형은 사용하지 않습니다.

예시 명령:

```bash
git commit -m "📝 Docs[S14P31E102-271]: 경로 API 명세 수정 및 ERD_v4 작성"
```

### 타입(Type) 분류

| gitmoji | Type | 용도 |
| :---: | :--- | :--- |
| `✨` | `Feat` | 새 기능 추가 |
| `🐛` | `Fix` | 버그 수정 |
| `🚑` | `Hotfix` | 긴급 수정 |
| `♻️` | `Refactor` | 코드 리팩토링 |
| `🔧` | `Chore` | 빌드 설정, 의존성 업데이트 등 기타 잡무 |
| `✅` | `Test` | 테스트 코드 |
| `📝` | `Docs` | 문서 작성 및 수정 |

## 브랜치 컨벤션

브랜치 생성은 `git br` 단축 명령을 활용합니다.

```text
feat/{설명}-{이슈번호}      (예: feat/server-init-31)
fix/{설명}-{이슈번호}       (예: fix/rds-timeout-32)
be/feat/{설명}-{이슈번호}   (예: be/feat/login-31)
fe/fix/{설명}-{이슈번호}    (예: fe/fix/header-32)
```

**예시 명령어:**
```bash
git br be/feat/login-31
```

👇 **실제 브랜치 생성 결과:**
```text
be/feat/login-S14P31E102-31
```

⚠️ **주의사항:** 브랜치 규칙의 **마지막은 반드시 `-숫자` (이슈번호)** 여야 합니다.

❌ **비허용 생성 예시 (이슈번호 뒤에 문자가 붙으면 안 됨):**
```bash
git br be/feat/login-31-ryuwon
git br be/feat/login-S14P31E102-31-ryuwon
```

## MR 컨벤션

- **제목:** Jira 이슈 키를 제목에 반드시 포함해 주세요. (예: `S14P31E102-32 Prod AWS 서비스 연결`)
- **본문:** `Closes S14P31E102-32` 혹은 `Resolves S14P31E102-32` 와 같이 명시해 두면, 머지(Merge) 처리 시 해당 Jira 이슈가 자동으로 `완료(Done)` 상태로 넘어갑니다.
