# 📁 docs 문서 작성 및 관리 규칙

> **작성일:** 2026-04-07  
> **작성자:** 김응서

`docs/` 폴더 내의 문서를 일관성 있게 유지하고 쉽게 찾기 위한 최소한의 규칙입니다.

---

## 📂 폴더 구조

문서는 용도에 맞는 하위 폴더에 배치해 주세요.

- **컨벤션/** : 팀 공통 규칙 (지라, 깃, 문서 규칙 등)
- **기획/** : 기획 산출물, 와이어프레임 등
- **회의록/** : 회의 기록, 팀 미팅 스크린샷 및 PDF 등
- **PRD/** : 제품 요구사항 정의서 (기능 스펙)
- **ARD/** : 아키텍처 설계 기록, DB ERD 등 기술 설계
- **API/** : API 명세서
- **인프라/** : CI/CD, Nginx 설정, 배포 가이드 등
- **트러블슈팅/** : 에러 원인 및 해결 기록
- **기타/** : 분류하기 애매한 문서를 임시 보관

---

## 1️⃣ 파일명 규칙

```text
YYYY-MM-DD_문서_제목.md
```

- **날짜**: 최초 작성일 기준 (`YYYY-MM-DD` 형식)
- **제목**: 띄어쓰기 대신 `_`(언더스코어) 사용
- **확장자**: 가급적 마크다운(`.md`)을 사용합니다. PDF 등 다른 자료에도 파일명 규칙을 적용해 주세요.

> **예시:** `2026-04-07_지라_컨벤션.md`, `2026-04-10_결제_API_명세.md`

---

## 2️⃣ 문서 헤더 (필수 기재)

모든 `.md` 문서의 **최상단**에는 아래 양식을 복사하여 작성 정보 및 수정 이력을 남깁니다.

```markdown
# 📋 문서 제목

> **작성일:** YYYY-MM-DD  
> **작성자:** 본인 이름
> **최종 수정일:** YYYY-MM-DD (내용 업데이트 시 추가)
```

---

## 3️⃣ 줄바꿈 규칙 통일 (중요 🔥)

Windows와 Mac 간 줄바꿈 문자 차이로 충돌이 나는 것을 방지하기 위해 파일은 **LF(`\n`)** 로 통일합니다.

**📌 PC 설정 (필수)**
```bash
# 커밋 시 CRLF → LF 자동 변환되도록 Git 속성 설정
git config --global core.autocrlf input
```
