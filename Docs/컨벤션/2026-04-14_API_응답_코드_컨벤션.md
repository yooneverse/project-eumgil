# API 응답 코드 컨벤션

## 목적

백엔드와 프론트엔드가 API 응답 상태를 같은 기준으로 해석하기 위한 애플리케이션 응답 코드 규칙을 정의한다.

HTTP status code는 실제 HTTP 응답 상태로 유지하고, 응답 body의 `status`에는 서비스 내부에서 사용하는 애플리케이션 응답 코드를 넣는다.

## 응답 예시

성공 응답:

```json
{
  "status": "S2000",
  "data": {},
  "message": "정상 처리되었습니다."
}
```

실패 응답:

```json
{
  "status": "C4000",
  "message": "잘못된 입력입니다."
}
```

## 코드 형식

```text
{prefix}{httpStatus}{sequence}
```

| 구성 요소 | 설명 |
| --- | --- |
| prefix | 성공, 공통 오류, 인증, 도메인, 외부 연동 등 오류 분류 접두어 |
| httpStatus | 실제 HTTP status code와 같은 3자리 숫자 |
| sequence | 같은 prefix와 HTTP status 안에서 중복되지 않게 붙이는 순번 |

예시:

- `U4040`: 사용자 없음
- `U4041`: 사용자 프로필 없음
- `PL4040`: 장소 없음
- `PL4041`: 좌표 주소 없음

## Prefix 기준

| Prefix | 의미 | 예시 |
| --- | --- | --- |
| S | Success | `S2000` |
| C | Common client error | `C4000`, `C4040`, `C4090` |
| A | Auth | `A4010`, `A4030` |
| U | User | `U4040`, `U4090` |
| BM | Bookmark | `BM4040`, `BM4090` |
| FR | Favorite Route | `FR4040`, `FR4030` |
| PL | Place | `PL4040` |
| RT | Route | `RT4000`, `RT4040` |
| RL | Route Log | `RL4000` |
| HR | Hazard Report | `HR4000`, `HR4090` |
| PT | Public Transit | `PT5020` |
| OP | Operation | `OP4010`, `OP4090` |
| E | External API | `E5020` |
| I | Internal server error | `I5000` |

## 공통 코드

| HTTP Status | status | 의미 |
| --- | --- | --- |
| 200 | S2000 | 정상 처리 |
| 201 | S2010 | 리소스 생성 완료 |
| 400 | C4000 | 공통 요청 검증 실패 |
| 401 | A4010 | 인증 필요 |
| 403 | A4030 | 권한 없음 |
| 404 | C4040 | 공통 조회 대상 없음 |
| 409 | C4090 | 공통 중복 요청 또는 상태 충돌 |
| 500 | I5000 | 서버 내부 예외 |
| 502 | E5020 | 외부 API 호출 실패 |

`204 No Content`는 응답 body를 보내지 않는다. 따라서 body의 `status` 값도 내려주지 않는다.

## API 명세 기반 코드 후보

> 아래 코드는 현재 API 명세 기준 초안이다. API 명세가 변경되면 이 표도 함께 수정한다.
> `A4010`, `A4030`, `I5000`, `E5020`처럼 여러 API에 공통으로 적용되는 코드는 API별 후보 표에 반복하지 않고 공통 코드 표를 따른다.

## 작성 규칙

- 새 실패 케이스는 백엔드 도메인의 `{Domain}ErrorCode` enum에 먼저 추가한다.
- 동일한 `status` 값을 서로 다른 에러에 재사용하지 않는다.
- HTTP status code와 응답 body의 `status` 안에 포함된 숫자는 맞춘다.
- `204 No Content`는 응답 body를 내려주지 않는다.
- 프론트엔드는 사용자 메시지 분기나 예외 케이스 처리가 필요할 때 `status` 값을 기준으로 판단한다.
- 단순 화면 노출 문구는 `message`를 우선 사용한다.
