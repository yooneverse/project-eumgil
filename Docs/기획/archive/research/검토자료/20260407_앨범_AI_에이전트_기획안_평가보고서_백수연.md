# 앨범 AI 에이전트 기획안 평가 보고서

> **한 줄 정의:** 갤럭시 사용자의 대용량 사진 앨범을 자연어 대화로 검색·분류·정리하는 온디바이스 앨범 관리 서비스  
> **최종 수정일:** 2026-04-07

## 평가 전제

- 이번 평가는 `아이디어 호감도`가 아니라 `심사 기준상 얼마나 점수를 받을 수 있는가`를 기준으로 다시 산정했다.
- 따라서 `이미 시장에 있는 기능`, `구현 근거가 없는 주장`, `팀 프로젝트에서 현실적으로 소화하기 어려운 범위`는 강하게 감점했다.
- 현재는 기획 단계이므로 `기능 정상 동작`, `UI/UX 우수성`은 실제 구현이 없으면 높은 점수를 줄 수 없다고 봤다.

### 재평가 기준

- `80점 이상`: 차별화가 분명하고, 구현 범위와 검증 계획, 역할 배분까지 상당히 구체적인 기획
- `70점대`: 문제 정의는 좋지만 차별화나 실현 가능성에 빈틈이 있는 기획
- `60점대`: 아이디어는 납득되나, 경쟁 서비스 대비 우위와 실행 계획이 약한 기획

이 기준으로 보면 이 프로젝트는 `좋은 문제 인식이 있는 아이디어`이지만, 현재 문서만으로는 `상위권 기획`으로 보긴 어렵다.

## 냉정하게 봤을 때 강점

### 1. 문제 자체는 실제로 존재함

- 사진 1만 장 이상 사용자의 검색 피로와 정리 피로는 실제 문제다.
- `찾기`, `정리`, `저장공간 관리`, `프라이버시`를 한 서비스로 묶은 점은 사용성 측면에서 가치가 있다.
- 즉, `문제 선정`은 좋다.

이 항목은 높게 볼 수 있다. 점수가 낮아진 핵심 원인은 문제 선정이 아니라 `차별화`와 `실현 가능성`이다.

### 2. 타겟과 목표는 명확함

- 갤럭시 사용자
- 사진 1만 장 이상 보유자
- 여행·육아·일상 기록 사용자

이 정도면 타겟 정의는 충분히 선명하다. 발표에서 전달력은 좋다.

### 3. 가치 제안은 직관적임

- 자연어 검색
- 인물 폴더 생성
- 중복 정리
- 베스트샷 추천

사용자는 기능을 바로 이해할 수 있다. 설명 난이도는 낮고 데모도 만들기 쉽다.

## 점수를 낮게 본 이유

### 1. 독창성은 생각보다 약함

이 기획의 핵심 기능 다수는 이미 대형 플랫폼이 제공하고 있다.

- 삼성은 2025년 4월 30일 공식 뉴스룸에서 `Galaxy S25 Gallery Search`가 자연어 기반 사진 검색과 on-device 최적화를 지원한다고 설명했다.
- 구글 포토는 공식 도움말에서 사람, 장소, 사물, 날짜 조합의 자유 텍스트 검색을 안내하고 있다.
- 애플 Photos 역시 `People & Pets` 분류와 로컬 인덱싱을 제공한다.

즉 아래 기능은 독창성 근거가 약하다.

- 자연어 사진 검색
- 인물 찾기/분류
- 유사 사진 탐색
- 저장공간 정리

이 프로젝트의 새로움은 `대화형 실행 경험`에 있는데, 현재 문서만 보면 이것도 아직 `개념 설명` 수준이다.  
냉정하게 말하면 지금 상태는 `새로운 제품`이라기보다 `기존 갤러리 기능들을 AI 에이전트 UX로 재포장한 기획`에 더 가깝다.

### 2. 기술적으로 말은 되지만, 팀 프로젝트 범위로는 무겁다

기획상 가장 어려운 부분은 `온디바이스`다.

- 1만 장 이상 사진에 대한 초기 인덱싱
- 얼굴 임베딩 생성과 인물 매칭
- 유사 사진 군집화
- 자연어 질의 해석
- 베스트샷 점수화

이걸 모두 로컬에서 돌리려면 성능 최적화가 핵심인데, 현재 문서에는 아래가 빠져 있다.

- 어떤 모델을 쓸지
- 어떤 기기 스펙을 기준으로 할지
- 인덱싱 시간이 얼마나 걸리는지
- 배터리/발열을 어떻게 통제할지
- 백그라운드 처리 정책이 무엇인지

즉, `AI를 온디바이스로 돌린다`는 문장은 강하지만, 심사 기준에서는 오히려 `검증되지 않은 고난도 약속`으로 보일 수 있다.

### 3. 실제 제품화 가능성도 높게 보기 어렵다

이 서비스가 `삼성 갤러리와 자연스럽게 합쳐진 제품`처럼 보이려면 OEM 수준 통합이 필요하다.  
그런데 팀 프로젝트가 만들 수 있는 건 보통 `별도 앱`이다.

별도 앱이 되면 현실적으로 아래 문제가 생긴다.

- 사용자가 전체 사진 접근 권한을 허용해야 함
- 기존 갤러리 UX와 분리될 가능성이 큼
- 삼성 기본 앨범의 얼굴 분류/메타데이터를 그대로 활용하기 어려울 수 있음
- 정리/삭제/이동 기능에서 실수 위험과 권한 UX가 커짐

즉, 이 아이디어는 `삼성 시스템 앱`으로는 그럴듯하지만, `학생 팀프로젝트 산출물`로는 제품 완성도가 떨어질 가능성이 높다.

### 4. UI/UX도 아직 높게 줄 수 없음

채팅형 인터페이스는 보기에는 멋있지만, 실제 앨범 관리에서는 오히려 위험하다.

- 검색은 대화형으로 편할 수 있다.
- 하지만 삭제, 이동, 폴더 생성, 병합은 실수 비용이 크다.
- 사용자는 결과를 한눈에 검토할 수 있는 `그리드`, `필터`, `일괄 선택`, `되돌리기`를 원한다.

즉, 이 서비스는 본질적으로 `채팅 앱`이 아니라 `앨범 관리 툴`이다.  
그런데 현재 기획은 채팅 UX의 장점은 강조했지만, 관리 툴로서 필요한 안전 UX는 충분히 설계되지 않았다.

### 5. 역할 배분이 사실상 비어 있음

팀 프로젝트 평가에서 이 항목은 크게 본다. 그런데 현재 문서에는 아래가 없다.

- 누가 Android 앱을 맡는지
- 누가 AI 파이프라인을 맡는지
- 누가 데이터셋/테스트를 맡는지
- 누가 UX 설계와 검증을 맡는지

이 상태면 `아이디어 설명서`는 되지만 `실행 계획서`는 아니다.

## 심사 기준별 상세 평가

### 독창성 — 55점

주요 기능이 이미 삼성, 구글, 애플의 앨범/포토 서비스에 상당 부분 존재한다.  
현재 문서만 보면 차별점은 `대화형 UI`와 `기능 묶음` 정도인데, 이것만으로 높은 독창성 점수를 주긴 어렵다.

### 아이디어 가치 — 84점

사용자 문제는 분명하고, 해결했을 때 효용도 크다.  
특히 육아·여행·기록형 사용자에게는 검색 시간 절감과 정리 자동화 가치가 높다.  
다만 `가치가 크다`와 `차별적이다`는 별개다. 이 항목만 높다.

### 기술 구체성 — 60점

기술 키워드는 있으나 구현 설계가 부족하다.

- 모델 선택 근거 없음
- 데이터 처리 파이프라인 없음
- 성능 목표 없음
- 오류 상황 대응 없음

기술 수준이 낮다기보다 `기획 문서상 증명되지 않았다`고 보는 편이 맞다.

### 실제 제품화 가능성 — 58점

가장 큰 감점 항목이다.

- 기존 갤러리와의 통합 문제
- Android 권한 UX 문제
- 로컬 추론 성능 문제
- 삭제/정리 기능의 신뢰성 문제

특히 `삼성 갤러리 기본 기능을 대체하거나 그 위에 깊게 얹는 수준`은 팀 프로젝트 범위를 넘을 가능성이 높다.

### 주제 명확성 — 90점

누구를 위한 서비스인지, 왜 필요한지는 분명하다.  
이 항목은 높은 점수를 줄 수 있다.

### 목표 구체성 — 82점

목표도 비교적 분명하다. 다만 `어디까지를 MVP로 볼 것인가`가 정리되지 않아 조금 감점했다.

### 기획 구체성 — 66점

기능 설명은 있지만, 기획 문서로서 중요한 아래 요소가 약하다.

- 사용자 플로우
- 핵심 화면 구조
- 예외 상황 처리
- 성공 지표
- 우선순위

즉, 기획의 폭은 있지만 깊이는 아직 부족하다.

### 역할 배분 명확성 — 30점

현재 상태로는 매우 낮게 봐야 한다.  
역할 분담이 없다시피 하므로 이 항목에서 높은 점수를 주면 오히려 기준이 무너진다.

### 창의성 발현도 — 57점

`AI를 앨범 관리에 붙인다`는 방향은 무난하다.  
하지만 기존 기능을 새 UX로 묶은 수준에 가깝고, 완전히 새로운 상호작용 모델이나 고유 데이터 자산이 보이지 않는다.

### UI/UX 사용성 계획 — 55점

현재는 채팅형 진입 경험만 있고, 실제 관리 UX 설계가 부족하다.

- 어떤 화면에서 결과를 검토하는지
- 삭제/이동 전 확인은 어떻게 하는지
- 오탐 시 어떻게 수정하는지
- 다량 선택은 어떻게 하는지

이 부분이 빠진 상태에서 사용성 우수 판정을 주긴 어렵다.

## 평가자 코멘트

이 프로젝트는 `사용자 문제를 잘 잡은 기획`이지, 아직 `독창적이고 실행력 높은 기획`은 아니다.

좋게 보이는 이유는 분명하다.

- 누구나 바로 이해하는 문제
- AI가 잘 어울리는 도메인
- 시연하면 그럴듯해 보이는 화면

하지만 냉정하게 보면 약점도 분명하다.

- 이미 있는 기능이 많다.
- 온디바이스 구현은 생각보다 어렵다.
- 별도 앱으로 만들면 통합 경험이 약해질 가능성이 크다.
- 팀 프로젝트 기준 역할과 검증 계획이 부족하다.

즉, 현재 점수대는 `아이디어는 좋지만 아직 수상권 기획으로 보기엔 근거가 약한 상태`가 맞다.

## 점수를 올리려면 필요한 것

### 1. 차별화를 다시 정의해야 함

아래처럼 바꾸는 편이 낫다.

> 우리는 사진 검색 앱이 아니라, `사용자 승인 기반으로 앨범 관리 작업을 실제 수행하는 온디바이스 에이전트`를 만든다.

그리고 기능도 줄여야 한다.

- 자연어 검색
- 인물 폴더 생성
- 유사 사진 추천 정리

이 3개만 명확히 해도 훨씬 낫다.

### 2. 구현 범위를 줄여야 함

현재는 기능이 많아서 오히려 실현 가능성이 떨어져 보인다.

- `완전 자동 삭제`는 제외
- `베스트샷 추천`은 후순위
- `갤럭시 전체 앨범 대체`가 아니라 `보조 정리 앱`으로 정의

이렇게 줄여야 제품화 가능성 점수가 올라간다.

### 3. 역할 배분표가 반드시 필요함

예를 들면 아래 정도는 있어야 한다.

| 역할 | 담당 범위 |
|------|-----------|
| PM/UX | 시나리오, 와이어프레임, 사용성 검증 |
| Android | 앨범 접근, UI, 결과 화면, 액션 플로우 |
| AI | 임베딩, 얼굴 매칭, 유사도 검색 |
| QA/Data | 테스트셋, 성능 측정, 오탐 검증 |

### 4. 성능 지표를 넣어야 함

- 1만 장 기준 인덱싱 시간
- 검색 응답 시간
- 인물 매칭 정확도
- 유사 사진 추천 precision
- 사용자 승인률
- 삭제 취소율

이게 들어가야 `기획`이 아니라 `검증 가능한 프로젝트`가 된다.

---

## 최종 총평

냉정하게 보면 현재의 앨범 AI 에이전트 기획은 `문제 선정은 좋지만, 차별성·구현성·실행 계획이 약한 중간 수준 기획`이다.  
강점은 분명하지만, 지금 문서 기준으로는 다른 AI 아이디어들보다 특별히 높은 점수를 줄 근거가 부족하다.

현재 상태에서의 적정 평가는 `66점 전후`가 맞다.  
점수를 올리려면 `기능 축소`, `차별화 재정의`, `역할 분담`, `성능 검증 계획`이 먼저 보강되어야 한다.
