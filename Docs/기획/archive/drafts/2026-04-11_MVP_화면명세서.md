본 문서는 2026-04-28 화면명세서와 2026-04-29 기능명세서에서 현재 MVP 범위만 추린 보조 명세다. 기능 또는 흐름이 충돌하면 상위 기준인 화면명세서, 기능명세서, API 명세를 우선 적용하고, FE 문서는 구현 참고 자료로만 사용한다.

## 1. MVP 범위 원칙

| 항목 | MVP 기준 |
| --- | --- |
| 앱 진입 | 인증 세션이 없으면 `AUTH-001`로 이동한다. 기존 회원은 `authSession -> selectedPrimaryUserType 홈 분기`로 이동하고, 신규 사용자는 소셜 인증 후 `signupToken`으로 온보딩을 완료한 뒤 회원가입을 생성한다. |
| 로그인 | `AUTH-001`은 필수 로그인 화면이며 카카오/네이버/구글 소셜 로그인만 제공한다. 운영 기준 `비로그인으로 계속`, guest mode는 제공하지 않는다. |
| 온보딩 | `ONB-001 사용자 유형 1차 선택`, `ONB-002 저시력자 전용 후속 화면`, `ONB-003 보행약자 2차 선택`, `ONB-004 약관/동의`, `ONB-005 튜토리얼` 5단계다. |
| 사용자 유형 | 1차 분기는 `저시력자`, `보행약자`로 통일하고, 보행약자 2차 선택은 `전동휠체어 사용자`, `수동휠체어 사용자`, `기타 보행약자`로 정리한다. |
| 장애등급 | 온보딩 필수 단계와 서버 스키마에서 제거한다. |
| 권한/약관 | `개인위치정보 수집·이용 동의`는 법적 동의 항목이고, `OS 위치 권한`은 `ONB-004` 다음 단계의 시스템 팝업이다. 마이크 권한은 온보딩에서 요청하지 않는다. 필수 약관 동의는 회원가입 완료 조건으로 확인하고 서버 `users` 별도 필드로 저장하지 않는다. |
| 하단 탭 | 보행약자 기본 탭은 `홈/북마크/제보/마이페이지` 4탭이다. 저시력자 전용 화면군은 `홈/북마크/카테고리/마이페이지`를 사용한다. |
| 지도 필터 | 빠른 필터는 `장애인 화장실`, `엘리베이터`, `전동보장구 충전 가능` 접근성 속성이다. 더보기는 장소 카테고리 7개를 보여준다. |
| 경로 | `ROUTE-01`의 대중교통 탭은 MVP에 포함하되 실제 대중교통 후보 조회는 후순위다. 경로 옵션은 `안전한 길`, `최단거리` 2개만 사용한다. |
| 제보 | 제보는 사용자 계정과 연결하고, 내 제보 목록/상세에서 처리 상태를 표시한다. 지도에는 승인된 공개 제보만 마커로 노출한다. 사진은 선택이며 최대 5장이다. |
| 데이터 책임 | 서버 데이터가 기준이며 로컬은 `cache/offline/pending` 보조 역할로만 사용한다. |
| BE 연동 | 확정된 API 상세는 `Docs/API/2026-04-12_API_전체_목록.md`와 도메인별 상세 명세를 따른다. |

## 3. 핵심 플로우

### 3.1 온보딩

| 순서 | 화면 | 핵심 동작 |
| --- | --- | --- |
| 1 | ONB-001 | `저시력자` 또는 `보행약자`를 선택한다. |
| 2 | ONB-002 또는 ONB-003 | `저시력자`는 전용 후속 화면으로, `보행약자`는 `전동휠체어 사용자`, `수동휠체어 사용자`, `기타 보행약자` 중 선택 화면으로 이동한다. |
| 3 | ONB-004 | 필수 약관/동의를 받고 전문은 노션 웹뷰로 확인한다. |
| 4 | OS 위치 권한 시스템 팝업 | `ONB-004`의 `다음` 이후 위치 권한 응답을 받는다. |
| 5 | ONB-005 | 튜토리얼 완료 후 `selectedPrimaryUserType=LOW_VISION`이면 `LV-HOM-01`, `MOBILITY_IMPAIRED`이면 `MAP-01`로 이동 |

### 3.2 지도 홈과 목적지 설정

| 출발 | 조건 | 도착 |
| --- | --- | --- |
| MAP-01 | 검색바 선택 | DEST-01 |
| MAP-01 | 마이크 아이콘 선택 | 마이크 권한 사용 시점 요청 후 음성 분석 기반 장소 검색 진입 |
| MAP-01 | 최근 목적지 전체보기 | BMK-01 |
| MAP-01 | 미세먼지 chevron | 날씨 외부 정보 패널 |
| DEST-01 | 검색 실행 | DEST-02 |
| DEST-02 | 검색 결과 카드 선택 | DEST-03 |
| DEST-03 | 도착지로 설정 + 설문 필요 | DEST-04 |
| DEST-03 | 도착지로 설정 + 설문 불필요 | ROUTE-01 |
| DEST-04 | 사용자 상태 선택 | ROUTE-01 |

> 검색 결과 카드 선택 후에는 반드시 `DEST-03` 지도 바텀시트 확인을 거친다. 사용자 상태 설문 노출 조건은 추가 확인 필요다.

### 3.3 경로와 길안내

| 화면 | MVP 동작 |
| --- | --- |
| ROUTE-01 | 도보/대중교통 탭을 제공한다. 대중교통 탭은 MVP에 포함한다. |
| ROUTE-01 | 경로 옵션은 `안전한 길`, `최단거리` 2개 카드만 세로로 나열한다. |
| ROUTE-01 | 옵션 카드 본문 선택은 선택 상태 변경이고, 우측 상단 화살표가 `ROUTE-02` 상세 진입이다. |
| ROUTE-01 | 길 안내 시작 선택 시 `NAV-01`로 이동한다. |
| NAV-01 | 경로 이탈 상태는 재검색/재탐색으로 정리한다. |
| NAV-01 | 도착 완료 시 `ARR-01`로 이동한다. 중도 종료 정책은 미정이다. |
| ARR-01 | 진입 즉시 `ARR-02` 평가 모달을 자동 표시한다. |
| ARR-02 | 우측 상단 X 버튼으로 평가를 건너뛸 수 있다. 평가는 별점만 저장한다. |

### 3.4 북마크

| 화면 | MVP 동작 |
| --- | --- |
| BMK-01 | 장소 북마크 탭. 카드 본문 선택 시 장소 상세로 이동한다. |
| BMK-01 | 장소 `길찾기`는 목적지 확인 후 `ROUTE-01`로 연결한다. |
| BMK-02 | 경로 북마크 탭. 저장된 출발지/도착지/옵션으로 다시 경로를 탐색한다. |
| BMK-02 | 경로 `길찾기`는 저장된 입력값으로 재탐색 후 길안내에 진입한다. |
| BMK-01/BMK-02 | 편집 모드는 삭제 기능만 제공한다. 정렬, 전체 선택, 다중 선택은 제공하지 않는다. |

### 3.5 제보

| 화면 | MVP 동작 |

| MYP-03 | 이용 가이드는 처음 튜토리얼 화면을 앱 내부 화면으로 재사용할 수 있다. |
| MYP-03 | 고객센터/1:1 문의는 `준비중인 기능입니다.` 메시지를 표시한다. |
| MYP-03 | 회원탈퇴는 재인증 필요, 북마크 데이터 삭제, 제보/평가 데이터 유지, 처리 후 `AUTH-001` 이동으로 정리한다. 경로 저장/안내 세션 처리 범위는 BE/기획 확인 필요다. |

## 4. MVP 미정 / BE 확인 필요

| 영역 | 내용 |
| --- | --- |
| 로그인 UX 순서 | 기존 회원은 `authSession -> selectedPrimaryUserType 홈 분기`로 이동한다. 신규 사용자는 소셜 인증 후 `signupToken`으로 `ONB-001 사용자 유형 1차 선택`에 진입한다. |
| 권한 | `ONB-004`에서 사용자가 `다음`을 누른 뒤 다음 단계에서 `OS 위치 권한 시스템 팝업`을 띄운다. |
| 저시력자 후속 화면 | `ONB-002`의 상세 UI와 저장 스키마, 그리고 `LV-MYP-01`의 앱 정보 상세를 기존 화면 재사용으로 볼지 여부는 추가 확인이 필요하다. |
| 접근성 필터 | 빠른 필터는 `장애인 화장실`, `엘리베이터`, `전동보장구 충전 가능`이고, 더보기는 장소 카테고리 7개다. |
| 사용자 상태 설문 | 노출 조건과 상태 저장 위치 추가 확인 필요 |
| STT | 처리 주체와 서버 연동 방식 BE 확인 필요 |
| 약관 | 노션 URL 제공 방식 BE 또는 기획 확인 필요 |
| 대중교통 | 후보 조회, 저상버스/역 접근성 검증 범위, 외부 API 연동 범위 BE 확인 필요 |
| 재탐색 | 현재 위치 기준 재검색/재탐색 계약 BE 확인 필요 |

| 활동/계정 | 활동 배지 산정, 회원탈퇴 재인증 방식은 BE/기획 확인 필요 |

## 5. MVP 제외 또는 후순위

- `SCR-001`~`SCR-003`, `ONB-01`~`ONB-03` 같은 구 온보딩 구조는 사용하지 않는다.
- 운영 기준 `비로그인으로 계속`, 게스트 모드, 로그인 없이 지도/경로/북마크/제보 진입은 제거한다.
- 보행약자 기본 탭 구조는 `홈/북마크/제보/마이페이지` 4탭으로 유지한다. 저시력자 UI는 별도 `LV-*` 화면군에서 `홈/북마크/카테고리/마이페이지`를 사용한다.
- 푸시/진동 알림 설정은 서버 API 범위에서 제외하고 앱 내부 설정으로 둔다.
- 별도 최근 목적지 전체 목록 화면은 제공하지 않는다.
- 별도 제보 처리 상태 전용 화면은 제공하지 않는다. 제보 처리 상태는 `MYP-02` 목록/상세 안에서 확인한다.
- 북마크 정렬, 전체 선택, 다중 선택은 제공하지 않는다.
- 길안내 중도 종료 정책은 이번 MVP에서 확정하지 않는다.
- 고객센터/1:1 문의 실제 연결은 제공하지 않고 `준비중인 기능입니다.` 메시지만 표시한다.
- `local-first`나 로컬 source of truth 표현은 production 핵심 정책으로 사용하지 않는다.
- `VIS-01`~`VIS-04` 같은 과거 레거시 명칭은 정식 화면 ID로 복귀시키지 않는다. 저시력자 전용 UI는 별도 화면군 `LV-*`로 정리한다.
