## 1. 문서 성격

2026-04-28 화면명세서와 2026-04-29 기능명세서가 백엔드 변경 사항을 반영해 최신 제품 기준선이 되었으므로, 이 문서는 그 상위 문서에 포함되지 않은 후순위 또는 미정 확장 항목만 정리한다. 확장 후보가 상위 문서와 충돌하면 상위 문서를 우선한다.

이 문서는 최신 화면명세서와 직접 충돌하지 않도록 후순위 또는 미정 확장 항목만 정리한다. 최신 MVP 핵심 화면 ID는 `AUTH`, `ONB`, `LV`, `MAP`, `DEST`, `ROUTE`, `NAV`, `ARR`, `BMK`, `REPORT`, `MYP` 체계를 따른다. 이 문서는 핵심 기준 문서를 대체하지 않으며, 확장 후보가 최신 정책을 뒤집는 근거가 되어서는 안 된다.

| 원칙 | 내용 |
| --- | --- |
| 사용자 용어 | 상위 용어는 `보행약자`로 통일한다. |
| 앱 진입 | 로그인은 핵심 gate로 이동했으므로 확장 항목이 아니다. |
| 하단 탭 | 보행약자 공통 화면은 `홈/북마크/제보/마이페이지`, 저시력자 전용 화면군은 `홈/북마크/카테고리/마이페이지` 탭 정책을 따른다. |
| 북마크 | 사용자 노출명은 `북마크`로만 표기한다. |
| 데이터 책임 | 서버 데이터가 기준이며 로컬은 `cache/offline/pending` 보조 역할로만 사용한다. |
| BE/API | API 상세는 이 문서에서 확정하지 않고 `BE 확인 필요`로 남긴다. |

## 2. 핵심 화면명세서로 이관된 항목

| 항목 | 최신 기준 |
| --- | --- |
| 로그인 | `AUTH-001` |
| 온보딩 | `ONB-001`~`ONB-005` |
| 저시력자 전용 화면군 | `LV-ONB-01`~`LV-ONB-05`, `LV-HOM-01`, `LV-SEA-01`, `LV-RES-01`, `LV-CAT-01`, `LV-BMK-01`, `LV-BRF-01`, `LV-NAV-01`, `LV-ARR-01`, `LV-MYP-01` |
| 지도 홈 | `MAP-01` |
| 목적지 설정 | `DEST-01`~`DEST-04` |
| 경로 탐색/상세/안내 | `ROUTE-01`, `ROUTE-02`, `NAV-01` |
| 도착/평가 | `ARR-01`, `ARR-02` |
| 북마크 | `BMK-01`, `BMK-02` |
| 제보 | `REPORT-01`~`REPORT-04` |
| 마이페이지 | `MYP-01`~`MYP-03` |

## 3. 후순위 또는 미정 확장 항목

### 3.1 알림

| 항목 | 내용 |
| --- | --- |
| 상태 | MVP 제외 또는 후순위/미정 |
| 화면 | 별도 알림함, 공지함, 메시지 inbox 화면은 만들지 않는다. |
| 정책 | 푸시/진동 토글을 노출하더라도 앱 내부 설정으로 관리한다. |
| BE 기준 | 푸시/진동 알림 설정은 서버 저장 범위에서 제외한다. |

### 3.2 길안내 중도 종료

| 항목 | 내용 |
| --- | --- |
| 상태 | 미정 |
| 최신 기준 | 도착 완료는 `ARR-01`로 이동한다. |
| 미정 | 중도 종료 시 `MAP-01` 복귀, `ROUTE-01` 복귀, 종료 확인 다이얼로그 표시 여부는 추후 확정한다. |

### 3.3 경로 이탈 재검색/재탐색

| 항목 | 내용 |
| --- | --- |
| 상태 | 후순위 보강 가능 |
| 최신 기준 | 길안내 중 경로 이탈 문구는 재검색/재탐색으로 정리한다. |
| BE 확인 필요 | 현재 위치 기준 재검색/재탐색 요청/응답 계약은 BE 확인 필요다. |

### 3.4 접근성 필터 더보기

| 항목 | 내용 |
| --- | --- |
| 상태 | 미정 |
| 적용 화면 | `MAP-01`, `DEST-02`, `FAC-01`, `BMK-01` |
| 미정 | `더보기` 안에 포함할 필터 항목과 정렬 기준은 추가 확인 필요다. |

### 3.5 저시력자 고도화 후속 확장

| 항목 | 내용 |
| --- | --- |
| 상태 | 후속 고도화 후보 |
| 범위 | 핵심 저시력자 화면군 `LV-ONB-01`~`LV-MYP-01`는 이미 MVP 기준선에 포함한다. 이 문서는 STT/LLM/TTS 고도화, 자동 브리핑 강화, 자연어 목적지 탐색 같은 후속 기능만 다룬다. |
| 현재 기준 | 저시력자 핵심 UI는 `LV-*` 별도 화면군으로 확정되어 핵심 화면명세서에서 관리한다. 이 문서의 확장 항목은 그 기준선을 대체하지 않는다. |
| 권한 | 마이크 권한은 STT 장소 검색 사용 시점에 요청한다. |
| 연동 기준 | 목적지 음성 분석을 사용한다. 보행약자는 장소명 추출 후 즉시 검색하고, 저시력자는 TTS 확인 후 검색한다. |

### 3.6 고객센터 / 1:1 문의

| 항목 | 내용 |
| --- | --- |
| 상태 | 후순위 |
| 적용 화면 | `MYP-03` |
| MVP 동작 | 선택 시 `준비중인 기능입니다.` 메시지를 표시한다. |
| BE 연동 | 현재 별도 BE 연동은 필요하지 않다. |

### 3.7 회원탈퇴 세부 정책

| 항목 | 내용 |
| --- | --- |
| 적용 화면 | `MYP-03` |
| 최신 기준 | 재인증 필요, 제보/평가 데이터 유지, 북마크 데이터 삭제, 처리 후 `AUTH-001` 이동 |
| BE 확인 필요 | 재인증 방식, 삭제/유지 처리 범위, 실패 처리 방식은 BE 확인 필요다. |

## 4. 임의 생성 금지

- 별도 비로그인 우회 진입 화면
- 별도 guest/demo 기반 메인 진입 화면
- 별도 최근 목적지 전체 목록 화면
- 별도 제보 처리 상태 화면
- 별도 시설 목록 전체 화면
- 별도 알림함/공지함/메시지 inbox 화면
- 별도 3탭/5탭 구조 탐색 화면
- 북마크 정렬, 전체 선택, 다중 선택 화면
- 고객센터/1:1 문의 실제 상담 화면
