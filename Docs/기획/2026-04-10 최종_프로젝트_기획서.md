## 1. 프로젝트 개요

### 프로젝트 한 줄 정의

부산이음길은 보행약자가 부산 지역에서 목적지까지 이동할 때 접근성 정보, 경로 후보, 장애물 제보를 함께 확인할 수 있는 배리어프리 길안내 서비스다.

### 해결하려는 문제

보행약자는 처음 가는 장소로 이동할 때 경사, 계단, 단차, 엘리베이터 부재, 공사/통제 등 이동 장벽을 사전에 파악하기 어렵다. 기존 길찾기 서비스는 접근성 시설과 현장 장애물을 경로 선택 기준으로 충분히 반영하지 않는다.

### 프로젝트 목적

- 접근성 정보를 포함한 목적지 탐색과 경로 비교를 제공한다.
- `안전한 길`, `최단거리` 2개 경로 옵션으로 이동 결정을 단순화한다.
- 시민 제보를 사용자 계정과 연결해 현장 장애물 정보를 축적한다.
- FE 최신 문서 기준의 화면 ID, 하단 탭, 북마크, 제보, 마이페이지 정책을 일관되게 유지한다.

## 2. 서비스 대상 및 사용자 정의

| 구분 | 정의 |
| --- | --- |
| 온보딩 1차 사용자 용어 | `저시력자`, `보행약자` |
| 보행약자 2차 선택 | `전동휠체어 사용자`, `수동휠체어 사용자`, `기타 보행약자` |
| 후속 접근성 확장 | 저시력자 핵심 UI는 별도 화면군 `LV-*`로 기준선에 포함한다. STT/LLM/TTS 고도화, 자동 브리핑 강화 같은 음성 기능 고도화만 후속 확장 후보로 둔다. |

> 1차 사용자 유형은 `selectedPrimaryUserType`, 보행약자 세부 유형은 `selectedMobilitySubtype`으로 저장한다. 저시력자는 `selectedMobilitySubtype`을 받지 않는다.

## 3. 서비스 범위

### 3.1 MVP 포함 범위

| 영역 | 포함 내용 |
| --- | --- |
| 인증/온보딩 | 로그인 gate, 온보딩 진입 gate |
| 온보딩 | `ONB-001 사용자 유형 1차 선택`, `ONB-002 저시력자 전용 후속 화면`, `ONB-003 보행약자 2차 선택`, `ONB-004 약관/동의`, `LV-ONB-01`~`LV-ONB-05` 저시력자 5단계 약관 안내, `ONB-005 튜토리얼` |
| 권한/약관 | `개인위치정보 수집·이용 동의`는 약관/동의 화면의 법적 동의 항목이고, `OS 위치 권한`은 `ONB-004` 다음 단계의 시스템 팝업이다. 마이크 권한은 STT 사용 시점 요청이다. |
| 지도 홈 | 검색바, STT 장소 검색 버튼, 접근성 필터, 날씨/미세먼지, 최근 목적지, 하단 4탭 |
| 저시력자 전용 화면군 | `LV-HOM-01`, `LV-SEA-01`, `LV-RES-01`, `LV-CAT-01`, `LV-BMK-01`, `LV-BRF-01`, `LV-NAV-01`, `LV-ARR-01`, `LV-MYP-01` |
| 목적지 설정 | `DEST-01`~`DEST-04` 검색/결과/지도 확인/사용자 상태 설문 |
| 경로 | `ROUTE-01` 도보/대중교통 탭, `ROUTE-02` 경로 상세, `NAV-01` 길안내 |
| 도착/평가 | `ARR-01` 도착 완료, `ARR-02` 평가 모달 자동 표시 |
| 북마크 | `BMK-01` 장소 탭, `BMK-02` 경로 탭 |
| 제보 | `REPORT-01`~`REPORT-04` 단계형 제보 |
| 마이페이지 | `MYP-01` 홈, `MYP-02` 제보 내역, `MYP-03` 앱 정보 및 도움말 |

### 3.2 MVP 제외 또는 후순위

- 알림함, 공지함, 메시지 inbox 화면
- 별도 최근 목적지 전체 목록 화면
- 별도 제보 처리 상태 전용 화면
- 북마크 정렬, 전체 선택, 다중 선택
- 고객센터/1:1 문의 실제 상담 연결
- 길안내 중도 종료 정책
- STT/LLM/TTS 기반 저시력자 고도화 음성 기능

## 4. 핵심 사용자 흐름

### 4.1 앱 진입

```text
AUTH-001
  -> ONB-001 사용자 유형 1차 선택
  -> 저시력자: ONB-002 저시력자 전용 후속 화면
  -> 저시력자: LV-ONB-01 -> LV-ONB-02 -> LV-ONB-03 -> LV-ONB-04 -> LV-ONB-05
  -> 보행약자: ONB-003 보행약자 2차 선택
  -> 보행약자: ONB-004 약관/동의
  -> 저시력자/보행약자: OS 위치 권한 시스템 팝업
  -> ONB-005 튜토리얼
  -> 저시력자: LV-HOM-01
  -> 보행약자: MAP-01
```

- 앱 시작 gate는 기존 회원 기준 `authSession -> selectedPrimaryUserType 홈 분기`로 분기한다. 신규 사용자는 소셜 인증 후 `signupToken`으로 온보딩을 완료하고 `POST /auth/signup` 성공 후 `저시력자`는 `LV-HOM-01`, `보행약자`는 `MAP-01`로 이동한다.
- 로그인 제공자 범위와 세션 만료 안내 방식은 추가 확인 필요다.

### 4.2 목적지 검색과 경로 안내

```text
MAP-01 -> DEST-01 -> DEST-02 -> DEST-03 -> DEST-04(필요 시) -> ROUTE-01 -> NAV-01 -> ARR-01
```

- 검색 결과 카드 선택 후에는 항상 `DEST-03` 지도 바텀시트 확인을 거친다.
- 사용자 상태 설문 노출 조건은 추가 확인 필요다.
- `ROUTE-01`에서는 도보/대중교통 탭을 제공하며 대중교통 탭은 MVP에 포함한다.
- 경로 이탈 상태는 재검색/재탐색으로 정리한다.
- 도착 완료 시 `ARR-01`로 이동하고 진입 즉시 `ARR-02` 평가 모달을 자동 표시한다.

### 4.3 북마크

```text
MAP-01 -> BMK-01 <-> BMK-02
```

- 최근 목적지 전체보기는 `BMK-01`로 연결한다.
- 장소 카드 본문 선택 시 장소 상세로 이동한다.
- 경로 카드 본문 선택 시 `ROUTE-02` 상세로 이동한다.
- 경로 북마크의 `길찾기`는 `NAV-01`로 즉시 진입한다.
- 편집 모드는 삭제 기능만 제공한다.

### 4.4 제보

```text
MAP-01 -> REPORT-01 -> REPORT-02 -> REPORT-03 -> REPORT-04 -> MYP-02
```

- 제보 유형은 `계단·단차 있음`, `점자블록 문제`, `인도 없음`, `경사로 문제`, `인도폭 문제`, `기타 장애물` 6개다.
- 위치는 pin drag로 확정한다.
- 사진 첨부는 갤러리와 카메라를 모두 허용한다.
- 사진은 presigned upload 방식으로 먼저 업로드하고, 제보 저장 요청에는 업로드 결과 `imageObjectKeys`를 전달한다.
- 제보는 사용자 계정과 연결한다.
- 내 제보 내역에서는 처리 상태를 확인할 수 있고, 지도에는 승인된 공개 제보만 마커로 노출한다.

## 5. 기능 정의

| 기능 | 화면 | 기준 |
| --- | --- | --- |
| 로그인 | `AUTH-001` | 소셜 로그인만 앱 진입 gate로 둔다. |
| 온보딩 | `ONB-001`~`ONB-005`, `LV-ONB-01`~`LV-ONB-05` | 사용자 유형 1차 선택, 저시력자 전용 후속 화면 또는 보행약자 2차 선택, 저시력자 5단계 약관 안내, 약관/동의, OS 위치 권한 이후 튜토리얼을 포함한다. |
| 지도 홈 | `MAP-01` | 하단 탭은 `홈/북마크/제보/마이페이지` 4탭이다. |
| 저시력자 전용 화면군 | `LV-HOM-01`~`LV-MYP-01` | 음성 입력, 현재 위치, 2카드 결과/북마크, 카테고리 4카드, 경로 브리핑, 도착지 저장/완료, 3버튼 마이페이지를 포함한다. |
| 시설 상세 | `FAC-01` | 지도 마커 또는 북마크 장소에서 접근한다. |
| 목적지 설정 | `DEST-01`~`DEST-04` | 검색 결과는 지도 바텀시트 확인 후 경로로 연결한다. |
| 경로 탐색 | `ROUTE-01`, `ROUTE-02` | `안전한 길`, `최단거리` 2개 옵션만 사용한다. |
| 길안내 | `NAV-01` | 도착 완료는 `ARR-01`, 중도 종료 정책은 미정이다. |
| 평가 | `ARR-02` | X 버튼으로 건너뛸 수 있고 평가 저장 API는 신규 예정/추후 구현이다. |
| 북마크 | `BMK-01`, `BMK-02` | 장소/경로 탭으로 분리한다. |
| 제보 | `REPORT-01`~`REPORT-04` | 완료 CTA는 `MYP-02` 목록으로 연결한다. |
| 마이페이지 | `MYP-01`~`MYP-03` | 사용자 유형 변경은 `ONB-001`부터 다시 시작한다. |

## 6. 정책 및 운영 규칙

| 항목 | 정책 |
| --- | --- |
| 데이터 책임 | 서버 데이터가 기준이며 로컬은 `cache/offline/pending` 보조 역할로만 사용한다. |
| 마이크 권한 | 온보딩에서 요청하지 않고 STT 장소 검색 사용 시점에 요청한다. |
| 약관 전문 | 노션 웹뷰로 표시한다. |
| 접근성 필터 | `더보기` 포함 항목은 미정이다. |
| 미세먼지 chevron | 날씨 외부 정보 패널로 이동한다. |
| 마이페이지 프로필 | 현재 사용자 유형 요약 데이터 매핑은 미정이다. |
| 활동 배지 | `제보왕`, `도움왕`, `참여왕` 조건은 미정이다. |
| 알림 | MVP 제외 또는 후순위/미정이다. |
| 로그아웃 | 온보딩/프로필 상태 유지 여부, 확인 다이얼로그, 토큰 삭제 범위는 미정이다. |
| 회원탈퇴 | 재인증 필요, 제보/평가 데이터 유지, 북마크 데이터 삭제, 처리 후 `AUTH-001`로 이동한다. |
| 이용 가이드 | 처음 튜토리얼 화면을 앱 내부 화면으로 재사용할 수 있다. |
| 고객센터/1:1 문의 | `준비중인 기능입니다.` 메시지를 표시한다. |

## 7. 데이터 및 BE 확인 필요

> 이 문서는 API URL, 필드명, enum 코드, 인증 정책을 확정하지 않는다.

| 영역 | 확인 필요 내용 |
| --- | --- |
| 로그인 UX 순서 | 기존 회원은 `authSession -> selectedPrimaryUserType 홈 분기`로 이동한다. 신규 사용자는 소셜 인증 후 `signupToken`으로 `ONB-001 사용자 유형 1차 선택`에 진입하고, 온보딩 완료 시 `POST /auth/signup` 성공 후 홈으로 이동한다. |
| 저시력자 후속 화면 | `ONB-002`의 상세 UI와 저장 스키마, `LV-MYP-01`의 앱 정보 상세 재사용 범위는 추가 확인이 필요하다. |
| 권한 | `ONB-004`에서 사용자가 `다음`을 누른 뒤 다음 단계에서 `OS 위치 권한 시스템 팝업`을 띄운다. |
| 사용자/프로필 | 1차 사용자 유형 2개 값, 보행약자 세부 유형 3개 값, 프로필 표시 문구 매핑 |
| STT | 처리 주체와 서버 연동 방식 |
| 약관 | 노션 URL 제공 방식 |
| 목적지/경로 | 장소 식별자/좌표/주소, 경로 옵션 식별자, 대중교통 후보 조회 |
| 재검색/재탐색 | 현재 위치 기준 요청/응답 계약 |
| 평가 | 신규 평가 저장 API 상세 |
| 북마크 | 장소/경로 북마크 식별자, 경로 북마크 즉시 길안내 payload |
| 제보 | 제보 유형 enum, 좌표 저장, presigned image upload와 `imageObjectKeys`, 사용자 계정 연결, 내 제보 상태 표시, 승인 공개 제보 지도 마커 |
| 계정 | 로그아웃 정책, 회원탈퇴 재인증과 데이터 처리 범위 |

## 8. 비기능 요구사항

| 구분 | 요구사항 |
| --- | --- |
| 접근성 | 주요 화면에 TalkBack 접근성 라벨과 명확한 포커스 순서를 제공한다. |
| 터치 영역 | 주요 CTA와 선택 카드는 충분한 터치 영역을 확보한다. |
| 상태 전달 | 색상만으로 상태를 전달하지 않고 텍스트, 아이콘, 배지를 함께 사용한다. |
| 실패 처리 | 권한 거부, 검색 실패, 제보 제출 실패, 북마크 삭제 실패 등은 사용자가 다음 행동을 알 수 있게 문구를 제공한다. |
| 개인정보 | 제보, 평가, 북마크, 프로필 데이터는 로그인 사용자 기준으로 다루며 삭제/유지 정책은 문서화한다. |
