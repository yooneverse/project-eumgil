## 1. 전역 정책

### 1.1 앱 진입 Gate

```text
앱 실행
  -> pending signup 있음: ONB-001 또는 미완료 온보딩 단계
  -> authSession 없음: AUTH-001
  -> authSession 있음 + profileCompleted=false: AUTH-002
  -> authSession 있음 + onboarding 완료 + selectedPrimaryUserType=LOW_VISION: LV-HOM-01
  -> authSession 있음 + onboarding 완료 + selectedPrimaryUserType=MOBILITY_IMPAIRED: MAP-01
```

| 분기 | 조건 | 이동 화면 | 비고 |
| --- | --- | --- | --- |
| 신규 사용자 | `pending signup` 존재 | `ONB-001` 또는 마지막 미완료 단계 | 사용자 유형 등 초기 설정은 온보딩에서 수집한다. |
| 인증 없음 | 저장된 세션 없음 또는 refresh 실패 | `AUTH-001` | 로그인 제공자 범위와 세션 만료 안내는 추가 확인 필요 |
| 프로필 미완료 | 인증 세션 있음 + `profileCompleted=false` | `AUTH-002` | `auth/profile_setup` gate |
| 진입 완료 | 인증 세션 있음 + 온보딩 완료 + `저시력자` | `LV-HOM-01` | 저시력자 전용 별도 홈 |
| 진입 완료 | 인증 세션 있음 + 온보딩 완료 + `보행약자` | `MAP-01` | 홈 탭 활성 |

`AUTH-001`은 운영 기준 필수 로그인 화면이며 `비로그인으로 계속`, guest mode, 로그인 없이 지도/경로/북마크/제보 진입은 허용하지 않는다. current runtime에는 로그인 직후 `AUTH-002 profile_setup` gate가 추가로 존재한다.

### 1.2 사용자 용어

| 구분 | 기준 |
| --- | --- |
| 온보딩 1차 사용자 용어 | `저시력자`, `보행약자` |
| 보행약자 2차 선택 | `전동휠체어 사용자`, `수동휠체어 사용자`, `기타 보행약자` |
| 장애등급 | 온보딩 필수 단계와 서버 스키마에서 제거한다. |
| 금지/주의 | 과거 문서의 구형 접근성 용어 기반 메인 온보딩, `VIS-*` 메인 분기, `SCR-001`~`SCR-003`, `ONB-01`~`ONB-03` 구조는 최신 MVP 필수 단계로 보지 않는다. 후속 접근성 확장 후보로만 분리한다. |
| 서버 매핑 | 1차 분기값은 `users.selectedPrimaryUserType`에 `LOW_VISION` 또는 `MOBILITY_IMPAIRED`로 저장한다. `LOW_VISION`은 `selectedMobilitySubtype`을 받지 않고, `MOBILITY_IMPAIRED`일 때만 보행약자 2차 선택을 `selectedMobilitySubtype` 한 개 값으로 저장한다. |

## 4. 저시력자 관련 레거시 확장 후보

현재 MVP 핵심 온보딩에서 저시력자 분기는 `ONB-001 -> LV-ONB-01 -> LV-ONB-02 -> LV-ONB-03 -> LV-ONB-04 -> LV-ONB-05 -> OS 위치 권한 시스템 팝업 -> LV-HOM-01`로 처리한다. `VIS-01`~`VIS-04`는 현재 메인 라우트 ID가 아니라 과거 목업 기반 레거시 참고다.

| 화면 후보 | 목적 | 상태 |
| --- | --- | --- |
| `VIS-01` 대화형 목적지 검색 | 음성 중심 목적지 입력 | mockup-only / 후속 확장 |
| `VIS-02` 목적지 확인 | 음성 인식 결과 확인 | mockup-only / 후속 확장 |
| `VIS-03` 경로 브리핑 | 경로 요약 음성 브리핑 | mockup-only / 후속 확장 |
| `VIS-04` 저시력자 안내 진행 | 큰 글씨/음성 중심 길안내 | mockup-only / 후속 확장 |

| 적용 원칙 |
| --- |
| 현재 온보딩 1차 분기의 `저시력자` 흐름은 `LV-ONB-01`~`LV-ONB-05`가 담당한다. |
| `VIS-*`를 현재 MVP 메인 온보딩 route ID처럼 사용하지 않는다. |
| 저시력자 음성 검색은 장소명 확인 대화를 거친 뒤 장소 검색으로 연결한다. 큰 글씨, 음성 중심 길안내 UI는 별도 접근성 확장 정책에 맞춰 설계한다. |

## 5. 데이터 및 상태 정의 초안

| 데이터 | 적용 화면 | 상태 |
| --- | --- | --- |
| `authSession` | 앱 시작, `AUTH-001` | 서버 인증 세션이 기준이다. 로컬은 secure storage/token cache 역할만 수행한다. |
| `signupToken` | 신규 사용자 온보딩 | 소셜 인증 후 회원가입 완료 전까지 쓰는 임시 토큰이다. 회원가입 완료 요청 성공 후 폐기한다. |
| `selectedPrimaryUserType` | `ONB-001`, `MYP-01` | `LOW_VISION`, `MOBILITY_IMPAIRED` 1차 분기 값이다. 서버의 `users.selectedPrimaryUserType`이 기준이다. |
| `lowVisionFollowUpCompleted` | `LV-ONB-01`~`LV-ONB-05`, legacy `ONB-002` | 저시력자 순차 동의 완료를 위해 사용하는 상태다. legacy route와의 정리 방식은 추가 확인이 필요하다. |
| `selectedMobilitySubtype` | `ONB-003`, `DEST-04`, `MYP-01` | 사용자 노출 값은 `전동휠체어 사용자`, `수동휠체어 사용자`, `기타 보행약자`다. `selectedPrimaryUserType=MOBILITY_IMPAIRED`일 때만 서버에 한 개 값으로 저장하고, `LOW_VISION`이면 `null`이다. |
| 필수 약관 동의 상태 | `ONB-004` | 회원가입 완료 요청의 검증 조건이다. 서버 `users` 별도 필드로 저장하지 않는다. |
| `locationPermissionStatus` | `ONB-004`, `ONB-005`, `MAP-01` | `ONB-004` 다음 단계의 `OS 위치 권한 시스템 팝업` 응답 결과다. |
| `placeDestination` | `DEST-03`, `ROUTE-01` | 서버/검색 응답의 장소 식별자, 좌표, 주소가 기준이다. 로컬은 현재 세션 handoff만 담당한다. |
| `routeOptions` / `selectedRouteOption` | `ROUTE-01`, `ROUTE-02`, `NAV-01` | 서버 경로 응답이 기준이다. 로컬은 현재 경로 진행 상태와 임시 선택 상태만 유지한다. |
| `bookmarkedPlaces` | `FAC-01`, `BMK-01` | 서버 북마크 장소 데이터가 기준이다. 로컬은 `cache/offline/pending` 보조 역할이다. |
| `bookmarkedRoutes` | `ARR-02`, `BMK-02` | 서버 북마크 경로 데이터가 기준이다. 로컬은 `cache/offline/pending` 보조 역할이다. |
| `reportDraft` | `REPORT-01`~`REPORT-03` | 작성 중 임시 상태만 로컬에 유지한다. 제출 이후에는 서버 제보 데이터가 기준이다. |
| `reportHistory` | `REPORT-04`, `MYP-02` | 사용자 계정과 연결된 서버 제보 이력이 기준이다. |
| `routeRating` | `ARR-02` | 방금 안내받은 경로에 대한 별점만 서버에 저장한다. |
| `mypageProfileSummary` | `MYP-01` | 소셜 이름/닉네임, 교통약자 유형, 활동 배지 표시를 조합한다. 개인정보 항목은 노출하지 않는다. |

## 6. 미확정 항목

| 영역 | 확인 필요 내용 |
| --- | --- |
| 로그인 UX 순서 | 기존 회원은 `authSession -> selectedPrimaryUserType 홈 분기`로 이동한다. 신규 사용자는 소셜 인증 후 `signupToken`으로 `ONB-001`에 진입하고, 온보딩 완료 시 회원가입 완료 요청 성공 후 홈으로 이동한다. |

| 경로 | 대중교통 후보 범위, 상세 step payload, 경로 옵션 식별자, 길안내 중도 종료 후 이동 정책은 미정이다. |
| 평가 | 평가 실패 처리와 중복 평가 허용 여부는 미정이다. 실제 저장 연동 기준은 API 명세를 따른다. |
| 프로필 | 활동 배지, `이용권 사용자` 표시 문구, 1차 사용자 유형 요약 데이터 매핑은 미정이다. 개인정보는 노출하지 않는다. |
| 활동 배지 | `제보왕`, `도움왕`, `참여왕` 산정 기준과 계산 주체는 미정이다. |
| 알림 | 푸시/진동 알림 설정은 서버 API 범위에서 제외한다. 앱 내부 설정 노출 여부만 UX에서 결정한다. |
| 로그아웃/탈퇴 | 로그아웃 시 온보딩/프로필 상태 유지 여부, 토큰 삭제 범위, 확인 다이얼로그, 회원탈퇴 재인증 방식은 `BE 확인 필요`다. |

## 7. MVP 제외 또는 후순위

- 알림함, 공지함, 메시지 inbox 화면은 MVP에서 생성하지 않는다.
- `비로그인으로 계속` 또는 게스트 핵심 기능 진입은 production에 포함하지 않는다.
- `SCR-001`~`SCR-003`, `ONB-01`~`ONB-03` 같은 구 온보딩 구조는 사용하지 않는다.
- 보행약자 공통 화면의 기본 탭 구조는 `홈/북마크/제보/마이페이지` 4탭으로 유지한다. 저시력자 전용 UI는 공통 탭의 예외가 아니라 별도 `LV-*` 화면군에서 `홈/북마크/카테고리/마이페이지`를 사용한다.
- 별도 최근 목적지 전체 목록 화면은 만들지 않고 북마크 화면으로 연결한다.
- 사용자 노출 용어로 `저장`, `Saved`, `게스트 모드`, `비로그인 핵심 기능`을 사용하지 않는다.
- 북마크 정렬, 전체 선택, 다중 선택은 제공하지 않는다.
- 별도 제보 처리 상태 전용 화면은 제공하지 않는다. 제보 처리 상태는 `MYP-02` 목록/상세 안에서 확인한다.
- 고객센터/1:1 문의 실제 연결은 제공하지 않고 `준비중인 기능입니다.` 메시지만 표시한다.
- 길안내 중도 종료 정책은 이번 MVP에서 확정하지 않는다.
- `local-first`나 로컬 source of truth 표현은 production 핵심 정책으로 사용하지 않는다.
- `VIS-01`~`VIS-04`는 저시력자 메인 온보딩이 아니라 후속 접근성 확장 후보로 둔다.
