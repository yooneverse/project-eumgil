## 2. 텍스트 장소 검색

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `GET` |
| URL | `/places/search` |
| 인증 | 필요 |
| 설명 | 확정된 검색어로 카카오 장소를 검색하고, 내부 `places`와 매칭되는 결과에만 서비스 카테고리와 접근성 정보를 함께 제공한다. 음성 흐름에서는 `POST /voice/analyze`가 추출/확정한 `placeName`을 `keyword`로 전달한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |
| Content-Type | Y | `application/json` |

#### Query Parameter

| 변수명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| keyword | string | Y | 검색어. 음성 흐름에서는 `POST /voice/analyze`에서 확정된 `placeName` |
| lat | double | N | 기준 위도 |
| lng | double | N | 기준 경도 |
| radius | int | N | 검색 반경. meter 단위 |
| cursor | string | N | 다음 목록 조회용 cursor. 첫 조회 시 생략 |
| size | int | N | 페이지 크기 |

#### Body

> 없음

### Response

- HTTP Status: `200 OK`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| places | array | N | 검색 결과 목록 |
| places[].placeId | Long | Y | 내부 장소 ID. 매칭 실패 시 `null` |
| places[].provider | string | N | 장소 원천. 카카오 검색 결과는 `KAKAO` |
| places[].providerPlaceId | string | N | 카카오 장소 ID |
| places[].name | string | N | 장소명 |
| places[].category | string | Y | 내부 장소와 매칭된 경우의 우리 서비스 장소 카테고리. 후보값은 아래 `PlaceCategory`를 따른다. 매칭 실패 시 `null` |
| places[].address | string | Y | 주소 |
| places[].distanceMeter | int | Y | 기준 좌표로부터 거리 |
| places[].point | object | N | 좌표 |
| places[].accessibilityFeatures | array | N | 접근성 속성 목록. 내부 장소 매칭 실패 또는 접근성 정보가 없으면 빈 배열 `[]` |
| places[].accessibilityFeatures[].featureType | string | N | 접근성 속성 유형. 후보값은 아래 `AccessibilityFeatureType`을 따른다. |
| places[].accessibilityFeatures[].isAvailable | boolean | N | 제공 여부 |
| places[].matched | boolean | N | 내부 장소 매칭 여부 |
| nextCursor | string | Y | 다음 목록 조회용 cursor. 다음 결과가 없으면 `null` |
| size | int | N | 페이지 크기 |
| totalElements | Long | N | 전체 데이터 수 |
| hasNext | boolean | N | 다음 목록 존재 여부 |

#### Success Example

```json
{
  "status": "S2000",
  "data": {
    "places": [
      {
        "placeId": 10,
        "provider": "KAKAO",
        "providerPlaceId": "123456789",
        "name": "부산시민공원",
        "category": "TOURIST_SPOT",
        "address": "부산광역시 부산진구 시민공원로 73",
        "distanceMeter": 350,
        "point": {
          "lat": 35.1686,
          "lng": 129.0576
        },
        "accessibilityFeatures": [
          {
            "featureType": "accessibleEntrance",
            "isAvailable": true
          }
        ],
        "matched": true
      }
    ],
    "nextCursor": null,
    "size": 10,
    "totalElements": 1,
    "hasNext": false
  },
  "message": "정상 처리되었습니다."
}
```

### Error

| HTTP Status | status | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | PL4000 | 장소 조회 요청값이 올바르지 않습니다. | `lat`, `lng`, `radius`, `cursor`, `size` 형식 오류 또는 좌표/반경 조합 오류 |
| 400 | PL4001 | 장소 검색어를 입력해주세요. | `keyword` 누락 또는 빈 문자열 |
| 502 | PL5020 | 장소 검색 외부 API 호출에 실패했습니다. | 카카오 API 호출 실패 |

## 3. 좌표 주소 변환

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `GET` |
| URL | `/places/reverse-geocode` |
| 인증 | 필요 |
| 설명 | 확정된 좌표를 카카오 Local API 좌표 주소 변환으로 전달해 도로명 주소와 지번 주소를 반환한다. 도로명 주소가 없는 좌표는 지번 주소를 표시 주소로 사용한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |
| Content-Type | Y | `application/json` |

#### Query Parameter

| 변수명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| lat | double | Y | 위도 |
| lng | double | Y | 경도 |

#### Body

> 없음

### Response

- HTTP Status: `200 OK`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| displayAddress | string | N | 화면 표시용 주소. 도로명 주소가 있으면 도로명 주소, 없으면 지번 주소 |
| roadAddress | string | Y | 도로명 주소. 카카오 응답에 없으면 `null` |
| address | string | Y | 지번 주소 |
| region1DepthName | string | Y | 시/도 단위 지역명 |
| region2DepthName | string | Y | 시/군/구 단위 지역명 |
| region3DepthName | string | Y | 읍/면/동 단위 지역명 |

#### Success Example

```json
{
  "status": "S2000",
  "data": {
    "displayAddress": "부산 부산진구 시민공원로 73",
    "roadAddress": "부산 부산진구 시민공원로 73",
    "address": "부산 부산진구 범전동 200",
    "region1DepthName": "부산",
    "region2DepthName": "부산진구",
    "region3DepthName": "범전동"
  },
  "message": "정상 처리되었습니다."
}
```

### Error

| HTTP Status | status | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | PL4000 | 장소 조회 요청값이 올바르지 않습니다. | `lat`, `lng` 누락, 형식 오류, 좌표 범위 오류 |
| 404 | PL4041 | 좌표에 해당하는 주소를 찾을 수 없습니다. | 카카오 응답에 주소 결과가 없음 |
| 502 | PL5021 | 좌표 주소 변환 외부 API 호출에 실패했습니다. | 카카오 API 호출 실패 |

## 4. 지도 클릭 상세와의 역할 구분

| 항목 | 기준 |
| --- | --- |
| 지도 클릭 상세 전용 API | `POST /places/detail` |
| `GET /places/search` 역할 | 사용자가 텍스트로 입력하거나 음성 분석으로 확정된 검색어를 검색하는 API |
| `GET /places/reverse-geocode` 역할 | 좌표를 표시 주소로 변환하는 단일 유틸리티 API |
| 클릭 상세에서 주소/카테고리 조합 | 백엔드가 `POST /places/detail` 내부에서 Kakao 검색/역지오코딩을 조합해 응답 |
| provider category 정책 | `GET /places/search`는 내부 장소 매칭용 `category`만 다루고, 외부 원본 카테고리 문자열은 `POST /places/detail.providerCategory`에서만 다룬다. |

## 5. 음성 검색 연동 정책

| 항목 | 기준 |
| --- | --- |
| 음성 분석 | `POST /voice/analyze`가 STT 텍스트에서 의도와 장소명을 추출한다. |
| 장소 검색 | `GET /places/search`는 자연어 문장을 해석하지 않고 확정된 `keyword`만 검색한다. |
| 보행약자 | `voice/analyze`에서 `placeName`이 추출되면 FE가 바로 `GET /places/search?keyword={placeName}`를 호출한다. |
| 저시력자 | `voice/analyze`에서 `confirmationMessage`를 받은 뒤 FE가 TTS로 확인하고, `confirmed=true`가 된 경우에만 `GET /places/search?keyword={placeName}`를 호출한다. |
| DB 매칭 | BE는 카카오 검색 결과의 장소 ID를 `places.providerPlaceId`와 비교해 내부 장소와 매칭한다. |

## 6. 공통 값 집합

### PlaceCategory

| 값 | 표시명 | 설명 |
| --- | --- | --- |
| `FOOD_CAFE` | 음식·카페 | 음식점, 카페, 제과점 등 식음 목적지 |
| `TOURIST_SPOT` | 관광지 | 관광지, 공원, 해변 등 방문 목적지 |
| `ACCOMMODATION` | 숙박 | 관광숙박, 일반숙박, 생활숙박 |
| `HEALTHCARE` | 의료·보건 | 병원, 의원, 치과, 한의원, 보건소, 종합병원 |
| `WELFARE` | 복지·돌봄 | 노인/장애인/아동/사회복지시설, 경로당, 요양시설 |
| `PUBLIC_OFFICE` | 공공기관 | 주민센터, 지자체 청사, 공단, 우체국, 파출소, 지구대 |
| `ETC` | 기타 편의시설 | 보행약자 편의시설 원천 근거는 있으나 대표 서비스 카테고리로 단정하기 어려운 장소 |

### AccessibilityFeatureType

| 값 | 표시명 | 설명 |
| --- | --- | --- |
| `accessibleEntrance` | 접근 가능 출입구 | 주출입구 접근 가능, 무단차 진입, 경사로형 접근로를 통합 |
| `elevator` | 엘리베이터 | 장소 내부 또는 인접 엘리베이터 |
| `accessibleToilet` | 장애인 화장실 | 장애인 이용 가능 화장실 |
| `accessibleParking` | 장애인 주차장 | 장애인 전용 주차구역 |
| `chargingStation` | 전동보장구 충전 가능 | 장소 내부 또는 독립 충전소의 충전 기능 |
| `accessibleRoom` | 장애인 객실 | 숙박시설 내 장애인 이용 가능 객실 |
| `guidanceFacility` | 안내 설비 | 점자/촉지/음성/유도 안내 설비 |

### 카테고리 정책

- `BUS_STATION`, `ELEVATOR`, `BARRIER_FREE_FACILITY`는 장소 도메인 최종 카테고리로 사용하지 않는다.
- 카카오 검색 결과가 내부 `places.providerPlaceId`와 매칭되지 않으면 접근성 정보는 빈 배열로 반환한다.
- 내부 장소로 매칭된 결과의 `category`는 위 `PlaceCategory` 값만 사용한다.
- 카카오 `category_name`은 `GET /places/search` 응답과 내부 `places` DB 컬럼으로 보존하지 않는다. 검색/필터의 공식 카테고리는 내부 장소와 매칭된 `category`다.
- 외부 POI 하단 시트용 원본 카테고리 문자열이 필요하면 `POST /places/detail.providerCategory`를 사용한다.
- `TOILET`과 `CHARGING_STATION`은 장소 카테고리로 사용하지 않는다.
- 장애인 화장실은 `featureType=accessibleToilet`, 전동보장구 충전 가능 여부는 `featureType=chargingStation`으로 조회한다.
- 전동보장구 충전소 원천 장소는 `category=ETC`로 저장하고 `chargingStation=true`를 반드시 가진다.
- `BARRIER_FREE_FACILITY` 원천 데이터는 실제 시설 성격에 따라 `FOOD_CAFE`, `TOURIST_SPOT`, `ACCOMMODATION`, `HEALTHCARE`, `WELFARE`, `PUBLIC_OFFICE`, `ETC` 중 하나로 분류한다.
