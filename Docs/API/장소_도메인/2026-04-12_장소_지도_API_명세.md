## 1. API 목록

| 우선순위 | API | 설명 |
| --- | --- | --- |
| P1 | `GET /places` | 주변 시설 마커 조회, 장소 카테고리/접근성 필터 |
| P1 | `GET /places/{placeId}` | 내부 시설 상세 조회, 접근성 정보 조회 |
| P1 | `POST /places/detail` | 지도 클릭 상세 조회 |

## 2. 주변 시설 마커 조회

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `GET` |
| URL | `/places` |
| 인증 | 필요 |
| 설명 | 현재 위치 기준으로 주변 시설 마커 목록을 조회한다. 로그인 사용자의 북마크 여부를 함께 반환한다. 더보기는 장소 카테고리 필터로, 상단 빠른 버튼은 접근성 속성 필터로 처리한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |
| Content-Type | Y | `application/json` |

#### Query Parameter

| 변수명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| lat | double | Y | 기준 위도 |
| lng | double | Y | 기준 경도 |
| radius | int | N | 검색 반경. meter 단위. 생략 시 `1000`, 최대 `3000` |
| category | string | N | 장소 카테고리. 다중 선택 시 comma-separated 값 사용. 후보값은 아래 `PlaceCategory`를 따른다. |
| featureType | string | N | 접근성 속성 필터. 다중 선택 시 comma-separated 값 사용. 후보값은 아래 `AccessibilityFeatureType`을 따른다. |

#### Body

> 없음

### Response

- HTTP Status: `200 OK`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| places | array | N | 장소 목록. 최대 200개 |
| places[].placeId | Long | N | 장소 ID |
| places[].name | string | N | 장소명 |
| places[].category | string | N | 장소 카테고리. 후보값은 아래 `PlaceCategory`를 따른다. |
| places[].address | string | Y | 주소 |
| places[].point | object | N | 좌표 |
| places[].point.lat | number | N | 위도 |
| places[].point.lng | number | N | 경도 |
| places[].accessibilityFeatures | array | N | 마커/목록 라벨에 사용할 접근성 속성 목록 |
| places[].accessibilityFeatures[].featureType | string | N | 접근성 속성 유형 |
| places[].accessibilityFeatures[].isAvailable | boolean | N | 제공 여부 |
| places[].isBookmarked | boolean | N | 현재 로그인 사용자의 북마크 여부 |

#### Success Example

```json
{
  "status": "S2000",
  "data": {
    "places": [
      {
        "placeId": 10,
        "name": "부산시민공원",
        "category": "TOURIST_SPOT",
        "address": "부산광역시 부산진구 시민공원로 73",
        "point": {
          "lat": 35.1686,
          "lng": 129.0576
        },
        "accessibilityFeatures": [
          {
            "featureType": "accessibleToilet",
            "isAvailable": true
          },
          {
            "featureType": "elevator",
            "isAvailable": true
          }
        ],
        "isBookmarked": false
      }
    ]
  },
  "message": "정상 처리되었습니다."
}
```

### Error

| HTTP Status | status | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | PL4000 | 장소 조회 요청값이 올바르지 않습니다. | `lat`, `lng`, `radius`, `category`, `featureType` 형식 오류 또는 최대 반경 초과 |

## 3. 내부 시설 상세 조회

> 없음

| HTTP Status | status | message | 발생 조건 |

### 비고

- 운영시간은 DB에 저장하지 않는다.
- 카카오 API에서 운영시간을 제공하거나 별도 외부 데이터가 있는 경우에만 응답 확장 후보로 둔다.

## 4. 지도 클릭 상세 조회

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `POST` |
| URL | `/places/detail` |
| 인증 | 필요 |
| 설명 | 지도에서 외부 건물/POI 또는 도로/주소를 클릭했을 때 상세 정보를 조회한다. 프론트엔드는 좌표와 클릭 메타데이터를 백엔드로 전달하고, 백엔드는 필요 시 Kakao Local REST API를 호출해 정규화된 상세 응답을 반환한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |
| Content-Type | Y | `application/json` |

#### Path Variable

> 없음

#### Query Parameter

> 없음

#### Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| lat | double | Y | 클릭한 위도 |
| lng | double | Y | 클릭한 경도 |
| clickType | string | Y | 클릭 대상 유형. `POI`, `ADDRESS` 중 하나 |
| provider | string | N | 외부 지도 제공자. 현재는 `KAKAO`만 사용 |
| providerPlaceId | string | N | 제공자 장소 ID. POI 클릭 시 있으면 전달 |
| nameHint | string | N | 지도 SDK나 하단 시트에서 확보한 장소명 힌트. POI 상세 검색, 지하철역 판별, 외부 POI 표시명에 사용한다. 값이 없어도 좌표 주소 fallback이 가능하면 200 응답을 반환할 수 있다. |

#### Request Example

```json
{
  "lat": 35.1686,
  "lng": 129.0576,
  "clickType": "POI",
  "provider": "KAKAO",
  "providerPlaceId": "123456789",
  "nameHint": "부산시민공원"
}
```

- HTTP Status: `200 OK`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| bookmarkTargetId | string | N | 북마크 저장/삭제 기준으로 쓰는 서버 생성 opaque 식별자 |
| detailType | string | N | 상세 결과 유형. `INTERNAL_PLACE`, `EXTERNAL_POI`, `EXTERNAL_ADDRESS` 중 하나 |
| placeId | Long | Y | 내부 장소와 매칭된 경우의 장소 ID |
| provider | string | Y | 외부 원천. 내부 장소만으로 해석된 경우 `null` 가능 |
| providerPlaceId | string | Y | 외부 제공자 장소 ID |
| name | string | N | 화면 제목으로 사용할 장소명. 도로/주소 클릭은 표시 주소를 제목으로 사용 가능 |
| category | string | Y | 내부 장소와 매칭된 경우의 우리 서비스 장소 카테고리. 후보값은 아래 `PlaceCategory`를 따른다. |
| providerCategory | string | Y | 외부 제공자의 원본 카테고리 문자열. 내부 장소와 매칭되더라도 원본 POI 카테고리 표시가 필요하면 반환 가능 |
| phone | string | Y | 외부 제공자가 반환한 전화번호. 내부 장소, 주소 fallback, 버스정류장 등 전화번호가 없는 대상은 `null` |
| address | string | Y | 표시 주소 |
| point | object | N | 좌표 |
| point.lat | number | N | 위도 |
| point.lng | number | N | 경도 |
| accessibilityFeatures | array | N | 내부 장소와 매칭된 경우의 접근성 속성 목록. 미매칭 외부 대상은 빈 배열 `[]` |
| accessibilityFeatures[].featureType | string | N | 접근성 속성 유형 |
| accessibilityFeatures[].isAvailable | boolean | N | 제공 여부 |
| isBookmarked | boolean | N | 현재 로그인 사용자의 북마크 여부 |

### 처리 규칙

- 이 API는 외부 클릭 상세의 단일 진입점이며, 프론트엔드는 카카오 REST API를 직접 호출하지 않는다.
- `providerPlaceId`가 내부 `places.providerPlaceId`와 매칭되면 `nameHint` 없이도 내부 장소를 canonical source로 우선 사용한다.
- `clickType=POI`인 미매칭 대상은 우선 `nameHint`, `lat`, `lng`를 바탕으로 Kakao keyword 검색 후보를 좁혀 상세를 구성한다.
- POI keyword 검색이 실패해도 바로 404로 보지 않고 아래 fallback을 순서대로 시도한다.
  1. 지하철역 형태의 `nameHint`이면 Kakao category `SW8` 검색으로 지하철역 후보를 찾는다.
  2. 좌표 주소 변환으로 도로명주소와 `buildingName`을 얻는다.
  3. `buildingName`이 있으면 건물명 keyword 검색을 수행하고, 좌표 주소 변환의 도로명주소와 같은 후보만 채택한다.
  4. 위 후보가 없으면 좌표 주소 변환 결과를 `EXTERNAL_ADDRESS`로 반환한다.
- `providerPlaceId`가 `BS`로 시작하는 POI는 버스정류장으로 처리한다. 이 경우 Kakao keyword 검색을 생략하고, 클릭된 정류장을 `EXTERNAL_POI`로 반환한다. `providerPlaceId`는 `BS...` 값을 유지하고 `providerCategory`는 `교통,수송 > 버스정류장`을 반환한다. 주소는 좌표 주소 변환 결과가 있으면 보조 정보로만 붙인다.
- `clickType=ADDRESS`인 대상은 Kakao 좌표 주소 변환 결과를 사용하며 `providerCategory`는 `null`이다.
- 이 API는 조회 전용이며 DB에 외부 장소 마스터 데이터를 생성하지 않는다.

## 5. 공통 값 집합

### PlaceCategory

| 값 | 표시명 | 설명 |
| --- | --- | --- |
| `FOOD_CAFE` | 음식·카페 | 음식점, 카페, 제과점 등 식음 목적지 |
| `TOURIST_SPOT` | 관광지 | 관광지, 공원, 해변 등 방문 목적지 |
| `ACCOMMODATION` | 숙박 | 관광숙박, 일반숙박, 생활숙박 |
| `HEALTHCARE` | 의료·보건 | 병원, 의원, 치과, 한의원, 보건소, 종합병원 |
| `WELFARE` | 복지·돌봄 | 노인/장애인/아동/사회복지시설, 경로당, 요양시설 |
| `PUBLIC_OFFICE` | 공공기관 | 주민센터, 지자체 청사, 공단, 우체국, 파출소, 지구대 |
| `ETC` | 기타 편의시설 | 보행약자 편의시설 원천 근거는 있으나 대표 서비스 카테고리로 단정하기 어려운 장소 |

### AccessibilityFeatureType

| 값 | 표시명 | 설명 |
| --- | --- | --- |
| `accessibleEntrance` | 접근 가능 출입구 | 주출입구 접근 가능, 무단차 진입, 경사로형 접근로를 통합 |
| `elevator` | 엘리베이터 | 장소 내부 또는 인접 엘리베이터 |
| `accessibleToilet` | 장애인 화장실 | 장애인 이용 가능 화장실 |
| `accessibleParking` | 장애인 주차장 | 장애인 전용 주차구역 |
| `chargingStation` | 전동보장구 충전 가능 | 장소 내부 또는 독립 충전소의 충전 기능 |
| `accessibleRoom` | 장애인 객실 | 숙박시설 내 장애인 이용 가능 객실 |
| `guidanceFacility` | 안내 설비 | 점자/촉지/음성/유도 안내 설비 |

### 카테고리 정책

- `BUS_STATION`, `ELEVATOR`, `BARRIER_FREE_FACILITY`는 장소 도메인 최종 카테고리로 사용하지 않는다.
- 버스 정류소는 대중교통 도메인에서, 도시철도 엘리베이터는 `subway_station_elevators` 데이터에서 관리한다.
- 카카오 `category_name`과 공공데이터 원천 카테고리명은 내부 `places` DB 컬럼과 `GET /places`, `GET /places/{placeId}` 응답에는 보존하지 않는다.
- 다만 `POST /places/detail`의 `providerCategory`는 외부 POI 하단 시트 표시와 외부 북마크 snapshot 저장을 위해 원본 문자열을 그대로 반환할 수 있다.
- `TOILET`과 `CHARGING_STATION`은 장소 카테고리로 사용하지 않는다.
- 장애인 화장실은 `featureType=accessibleToilet`, 전동보장구 충전 가능 여부는 `featureType=chargingStation`으로 조회한다.
- 전동보장구 충전소 원천 장소는 `category=ETC`로 저장하고 `chargingStation=true`를 반드시 가진다.
- `BARRIER_FREE_FACILITY` 원천 데이터는 실제 시설 성격에 따라 `FOOD_CAFE`, `TOURIST_SPOT`, `ACCOMMODATION`, `HEALTHCARE`, `WELFARE`, `PUBLIC_OFFICE`, `ETC` 중 하나로 분류한다.
- 지도 홈 상단 빠른 필터는 `accessibleToilet`, `elevator`, `chargingStation` 세 가지를 우선 노출한다.
- 더보기에는 `FOOD_CAFE`, `TOURIST_SPOT`, `ACCOMMODATION`, `HEALTHCARE`, `WELFARE`, `PUBLIC_OFFICE`, `ETC` 장소 카테고리를 노출한다.
