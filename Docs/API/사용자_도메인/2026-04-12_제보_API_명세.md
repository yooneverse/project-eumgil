## 1. API 목록

| 우선순위 | API | 설명 |
| --- | --- | --- |
| P1 | `POST /hazard-reports/images/presigned-upload` | 제보 이미지 업로드 URL 발급 |
| P1 | `POST /hazard-reports` | 도로 상태 제보 등록 |
| P1 | `GET /hazard-reports/me` | 내가 등록한 제보 목록 조회 |

> 도로 상태 제보는 로그인 사용자 계정과 연결한다. 사용자 화면에는 처리 상태를 노출하지 않지만 서버는 운영 검토를 위해 상태를 관리한다.

## 2. 제보 이미지 업로드 URL 발급

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `POST` |
| URL | `/hazard-reports/images/presigned-upload` |
| 인증 | 필요 |
| 설명 | 로그인 사용자가 제보 사진을 S3/MinIO에 직접 업로드할 수 있도록 presigned PUT URL을 발급한다. BE 서버는 이미지 binary를 직접 받지 않는다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |
| Content-Type | Y | `application/json` |

#### Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| fileName | string | Y | 사용자가 선택한 원본 파일명. 서버 저장 경로에는 사용하지 않고 검증/추적용으로만 받는다. |
| contentType | string | Y | 업로드할 이미지 MIME type. 허용값: `image/jpeg`, `image/png`, `image/webp`, `image/heic`, `image/heif` |
| contentLength | number | Y | 업로드할 파일 크기(byte). 기본 최대 10MB |

#### Request Example

```json
{
  "fileName": "sidewalk.png",
  "contentType": "image/png",
  "contentLength": 524288
}
```

### Response

- HTTP Status: `201 Created`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| uploadUrl | string | N | FE가 이미지 binary를 `PUT`으로 업로드할 presigned URL. 기본 만료 시간은 10분 |
| objectKey | string | N | 저장 객체 key. 업로드 성공 후 `POST /hazard-reports`의 `imageObjectKeys`에 넣는 안정 저장값. `hazard-reports/{userId}/{yyyyMMdd}/{uuid}.{ext}` 형식 |
| expiresAt | string | N | `uploadUrl` 만료 시각 |

#### Success Example

```json
{
  "status": "S2010",
  "data": {
    "uploadUrl": "https://busan-eumgil-s3.s3.ap-northeast-2.amazonaws.com/hazard-reports/...",
    "objectKey": "hazard-reports/{userId}/20260515/{uuid}.png",
    "expiresAt": "2026-05-15T01:00:00Z"
  },
  "message": "생성되었습니다."
}
```

### FE Upload Rule

1. FE는 사진 선택 시 이 API를 먼저 호출한다.
2. FE는 응답의 `uploadUrl`로 이미지 binary를 `PUT` 업로드한다.
3. `PUT` 요청의 `Content-Type`은 presigned URL 발급 요청의 `contentType`과 동일해야 한다.
4. `PUT` 성공 후 응답의 `objectKey`를 outbox에 보존하고, 제보 제출 시 `imageObjectKeys` 배열에 포함한다.
5. 일부 사진 업로드가 실패하면 성공한 사진의 `objectKey`만 제출할 수 있다. 실패한 사진은 FE에서 개별 재시도한다.
6. 조회 API의 `imageUrls`/`representativeImageUrl`은 조회 시점에 새로 발급되는 presigned GET URL이다. FE는 이 값을 영구 저장하지 않고 만료 시 목록/상세를 재조회한다.

### S3/MinIO 운영 조건

- 운영 S3 버킷은 private으로 유지한다. DB에는 공개 URL이 아니라 `objectKey`만 저장한다.
- BE는 업로드 시 presigned PUT URL을, 조회 시 presigned GET URL을 발급한다.
- `S3_ACCESS_KEY`/`S3_SECRET_KEY`가 가리키는 IAM principal에는 `hazard-reports/*` prefix에 대한 `s3:PutObject`, `s3:GetObject` 권한이 필요하다.
- FE에는 AWS key를 전달하지 않는다.
- 브라우저 직접 업로드와 presigned 조회를 위해 S3 CORS는 서비스 origin에서 `PUT`, `GET`, `HEAD`와 `Content-Type` 요청 헤더를 허용해야 한다.
- 로컬 MinIO presigned URL은 FE/브라우저에서 접근 가능한 host로 서명되어야 하므로 compose 기본값은 `MINIO_PUBLIC_ENDPOINT` 또는 `http://localhost:{MINIO_API_PORT}`를 `S3_ENDPOINT`로 사용한다.

### Error

| HTTP Status | status | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | C4000 | 잘못된 입력입니다. | 필수 필드 누락, 파일 크기 초과, 허용되지 않은 이미지 MIME type |
| 401 | A4010 | 인증이 필요합니다. | 토큰 누락/만료 |
| 503 | HR5030 | 제보 이미지 업로드 URL을 발급할 수 없습니다. | S3/MinIO presigned URL 발급 실패 |

## 3. 도로 상태 제보 등록

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `POST` |
| URL | `/hazard-reports` |
| 인증 | 필요 |
| 설명 | 계단/단차, 점자블록, 인도 부재, 경사로, 인도폭, 기타 장애물을 로그인 사용자 제보로 등록한다. 이미지 binary는 업로드 URL 발급 API로 S3/MinIO에 먼저 업로드하고, 이 API에는 `imageObjectKeys` 배열만 전달한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |
| Content-Type | Y | `application/json` |
| Idempotency-Key | N | 재시도 중복 방지 키. 사용자별로 24시간 보관하며 최대 255자 |

#### Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| reportType | string | Y | 제보 유형. 후보값은 아래 `ReportType`을 따른다. |
| description | string | N | 설명 |
| reportPoint | object | Y | 제보 위치 |
| reportPoint.lat | number | Y | 위도 |
| reportPoint.lng | number | Y | 경도 |
| imageObjectKeys | array | N | 제보 이미지 object key 목록. 선택 입력, 최대 5장 |

#### Request Example

```json
{
  "reportType": "SIDEWALK_MISSING",
  "description": "지도에는 인도로 나오지만 실제로는 보행 가능한 인도가 없습니다.",
  "reportPoint": {
    "lat": 35.1686,
    "lng": 129.0576
  },
  "imageObjectKeys": [
    "hazard-reports/{userId}/20260515/{uuid}.png"
  ]
}
```

### Response

- HTTP Status: `201 Created`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| reportId | Long | N | 제보 ID |

#### Success Example

```json
{
  "status": "S2010",
  "data": {
    "reportId": 1
  },
  "message": "생성되었습니다."
}
```

### Error

| HTTP Status | status | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | C4000 | 잘못된 입력입니다. | 필수 필드 누락, enum 파싱 실패, 좌표 범위 오류, 이미지 5장 초과 |
| 400 | HR4000 | 제보 요청값이 올바르지 않습니다. | 서비스 내부 생성 규칙 위반, `Idempotency-Key` 255자 초과 |
| 401 | A4010 | 인증이 필요합니다. | 토큰 누락/만료 |
| 409 | HR4091 | 같은 Idempotency-Key로 다른 제보 요청이 이미 처리되었습니다. | 같은 사용자와 같은 `Idempotency-Key`로 요청 본문이 다른 재시도 |

### 처리 규칙

- 신규 제보는 서버 내부적으로 `PENDING` 상태로 생성한다.
- 사용자 API 응답에는 처리 상태를 내려주지 않는다.
- 제보 사진은 선택 입력이며 최대 5장이다.
- `imageObjectKeys`에는 `POST /hazard-reports/images/presigned-upload`로 발급된 현재 사용자 소유 object key만 허용한다.
- `imageObjectKeys`에는 URL scheme, query string 또는 fragment를 포함할 수 없다.
- 제보 위치의 기준값은 `reportPoint`다. 서버는 생성 시 Kakao 역지오코딩 표시 주소를 nullable snapshot으로 저장한다.
- `Idempotency-Key`가 없으면 기존과 동일하게 매 요청을 신규 생성으로 처리한다.
- 같은 사용자와 같은 `Idempotency-Key`, 같은 요청 본문이 24시간 내 재전송되면 신규 row를 만들지 않고 기존 `reportId`를 반환한다.
- 같은 사용자와 같은 `Idempotency-Key`, 다른 요청 본문이 24시간 내 전송되면 `409 HR4091`로 거부한다.
- 만료된 멱등성 메타데이터는 주기적으로 정리되어 24시간 이후 같은 키 재사용을 허용한다.
- 등록된 제보는 지도/장소 정보에 바로 반영하지 않고 운영 검토 후 반영한다.

## 4. 내가 등록한 제보 목록 조회

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `GET` |
| URL | `/hazard-reports/me` |
| 인증 | 필요 |
| 설명 | 마이페이지에서 사용자가 등록한 제보 목록을 최신순으로 조회한다. 대표 사진만 내려준다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |

#### Query Parameter

| 변수명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| cursor | Long | N | 마지막으로 조회한 제보 ID. 첫 조회 시 생략 |
| size | int | N | 조회 개수. 기본값 `10`, 허용 범위 `1`~`100` |

#### Body

> 없음

### Response

- HTTP Status: `200 OK`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| content | array | N | 제보 목록 |
| content[].reportId | Long | N | 제보 ID |
| content[].reportType | string | N | 제보 유형 |
| content[].address | string | Y | 제보 위치 역지오코딩 표시 주소. 주소 보강 실패 또는 기존 데이터는 `null` |
| content[].description | string | Y | 설명 preview. 원문이 80자를 초과하면 80자 뒤 `...`를 붙임 |
| content[].reportPoint | object | N | 제보 좌표 |
| content[].createdAt | string | N | 등록 일시 |
| content[].representativeImageUrl | string | Y | 대표 사진 presigned GET URL. 사진이 없으면 `null`, 만료되면 목록을 재조회 |
| size | int | N | 페이지 크기 |
| nextCursor | Long | Y | 다음 조회에 사용할 cursor. 다음 데이터가 없으면 `null` |
| hasNext | boolean | N | 다음 페이지 존재 여부 |

#### Success Example

```json
{
  "status": "S2000",
  "data": {
    "content": [
      {
        "reportId": 1,
        "reportType": "SIDEWALK_MISSING",
        "address": "부산 부산진구 시민공원로 73",
        "description": "지도에는 인도로 나오지만 실제로는 보행 가능한 인도가 없습니다.",
        "reportPoint": {
          "lat": 35.1686,
          "lng": 129.0576
        },
        "createdAt": "2026-04-28T17:00:00",
        "representativeImageUrl": "https://busan-eumgil-s3.s3.ap-northeast-2.amazonaws.com/hazard-reports/{userId}/20260515/{uuid}.png?X-Amz-Algorithm=AWS4-HMAC-SHA256..."
      }
    ],
    "size": 10,
    "nextCursor": null,
    "hasNext": false
  },
  "message": "정상 처리되었습니다."
}
```

### Error

| HTTP Status | status | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | C4000 | 잘못된 입력입니다. | `cursor`, `size` 값이 허용 범위를 벗어남 |
| 401 | A4010 | 인증이 필요합니다. | 토큰 누락/만료 |

## 6. 공통 값 집합

### ReportType

| 값 | 화면 제목 | 보조 설명 |
| --- | --- | --- |
| `STAIRS_STEP` | 계단·단차 있음 | 안내와 달리 계단이나 단차가 있어요 |
| `BRAILLE_BLOCK` | 점자블록 문제 | 손상, 미설치, 잘못된 설치 |
| `SIDEWALK_MISSING` | 인도 없음 | 안내와 달리 보행 가능한 인도가 없어요 |
| `RAMP` | 경사로 문제 | 경사로 이용이 어렵거나 위험해요 |
| `SIDEWALK_WIDTH` | 인도폭 문제 | 인도가 좁아 통행이 어려워요 |
| `OTHER_OBSTACLE` | 기타 장애물 | 위 항목에 없는 보행 불편 상황 |
