# 📋 경로 북마크 API 명세

> **작성일:** 2026-04-12  
> **작성자:** 김응서  
> **최종 수정일:** 2026-05-06

---

## 1. API 목록

| 우선순위 | API | 설명 |
| --- | --- | --- |
| P1 | `GET /favorite-routes` | 경로 북마크 목록 조회 |
| P1 | `GET /favorite-routes/{favRouteId}` | 경로 북마크 상세 조회 |
| P1 | `POST /favorite-routes` | 경로 북마크 저장 |
| P1 | `PATCH /favorite-routes/{favRouteId}` | 경로 북마크 수정 |
| P1 | `DELETE /favorite-routes/{favRouteId}` | 경로 북마크 삭제 |

> 경로 북마크 서버 API는 로그인 사용자의 서버 저장 데이터만 처리한다. 운영 기준 비로그인 경로 북마크 저장은 제공하지 않는다.
> 안내 종료 후 저장하는 경로 북마크는 `route_sessions`의 좌표와 route snapshot을 복사해 장기 저장한다. `favorite_routes`는 `route_sessions`를 직접 참조하지 않는다.

## 2. 경로 북마크 목록 조회

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `GET` |
| URL | `/favorite-routes` |
| 인증 | 필요 |
| 설명 | 로그인 사용자가 서버에 저장한 경로 북마크 목록을 cursor 기반 무한스크롤로 조회한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |

#### Query Parameter

| 변수명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| cursor | Long | N | 다음 목록 조회 기준 cursor. 첫 조회에서는 전달하지 않는다. |
| size | int | N | 페이지 크기 |

#### Body

> 없음

### Response

- HTTP Status: `200 OK`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| content | array | N | 경로 북마크 목록 |
| content[].favRouteId | Long | N | 경로 북마크 ID |
| content[].routeName | string | N | 경로명 |
| content[].startLabel | string | N | 출발지명 |
| content[].endLabel | string | N | 도착지명 |
| content[].startPoint | object | N | 출발지 좌표 |
| content[].endPoint | object | N | 도착지 좌표 |
| content[].transportMode | string | N | 이동 수단. `WALK`, `PUBLIC_TRANSIT` |
| content[].routeOption | string | N | 대표 경로 옵션. `SAFE`, `SHORTEST`, `RECOMMENDED`, `MIN_TRANSFER`, `MIN_WALK` |
| size | int | N | 페이지 크기 |
| nextCursor | Long | Y | 다음 조회에 사용할 cursor. 다음 페이지가 없으면 `null` |
| hasNext | boolean | N | 다음 페이지 존재 여부 |

#### Success Example

```json
{
  "status": "S2000",
  "data": {
    "content": [
      {
        "favRouteId": 1,
        "routeName": "집에서 병원",
        "startLabel": "부산시민공원",
        "endLabel": "부산역",
        "startPoint": {
          "lat": 35.1686,
          "lng": 129.0576
        },
        "endPoint": {
          "lat": 35.1152,
          "lng": 129.0422
        },
        "transportMode": "WALK",
        "routeOption": "SAFE"
      }
    ],
    "size": 10,
    "nextCursor": 1,
    "hasNext": true
  },
  "message": "정상 처리되었습니다."
}
```

#### 비고

- 목록 정렬은 최신 저장순을 기본으로 한다.
- 첫 조회는 `cursor` 없이 요청한다.
- 다음 조회는 직전 응답의 `nextCursor`를 `cursor`로 전달한다.
- 무한스크롤 응답에서는 전체 개수와 전체 페이지 수를 제공하지 않는다.
- `routeName`은 사용자가 직접 입력한 이름이 아니라 `startLabel`과 `endLabel`을 기준으로 서버가 자동 생성한 표시명이며, 최대 511자다.
- 목록 응답에는 route snapshot을 포함하지 않는다. 저장 경로 상세는 `GET /favorite-routes/{favRouteId}`로 조회한다.

### Error

| HTTP Status | status | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | C4000 | 잘못된 입력입니다. | cursor/size 형식 또는 범위 오류 |
| 401 | A4010 | 인증이 필요합니다. | 토큰 누락/만료 |

## 4. 경로 북마크 저장

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `POST` |
| URL | `/favorite-routes` |
| 인증 | 필요 |
| 설명 | 안내 종료 후 로그인 사용자가 방금 안내받은 route session을 경로 북마크로 저장한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |
| Content-Type | Y | `application/json` |

#### Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| routeId | string | Y | 안내 종료한 route session의 대표 경로 ID |
| startLabel | string | Y | 출발지 표시명. 장소명, 도로명주소, 지번주소 순으로 프론트가 결정 |
| endLabel | string | Y | 도착지 표시명. 장소명, 도로명주소, 지번주소 순으로 프론트가 결정 |

#### Request Example

```json
{
  "routeId": "walk_rt_safe_001",
  "startLabel": "부산시민공원",
  "endLabel": "부산역"
}
```

### Response

- HTTP Status: `201 Created`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| favRouteId | Long | N | 경로 북마크 ID |

#### Success Example

```json
{
  "status": "S2010",
  "data": {
    "favRouteId": 1
  },
  "message": "생성되었습니다."
}
```

### Error

| HTTP Status | status | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | C4000 | 잘못된 입력입니다. | routeId/startLabel/endLabel 누락 또는 형식 오류 |
| 400 | FR4000 | 경로 북마크 요청값이 올바르지 않습니다. | 경로 북마크 생성 규칙 위반 |
| 401 | A4010 | 인증이 필요합니다. | 토큰 누락/만료 |
| 403 | FR4030 | 경로 북마크에 대한 권한이 없습니다. | 다른 사용자의 route session 저장 시도 |
| 404 | FR4041 | 선택한 경로 정보를 찾을 수 없습니다. | route session 없음 또는 snapshot 복구 불가 |

## 5. 경로 북마크 수정

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `PATCH` |
| URL | `/favorite-routes/{favRouteId}` |
| 인증 | 필요 |
| 설명 | 로그인 사용자의 서버 저장 경로 북마크 표시명을 수정한다. 저장된 좌표, 옵션, route snapshot은 변경하지 않는다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |
| Content-Type | Y | `application/json` |

#### Path Variable

| 변수명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| favRouteId | Long | Y | 경로 북마크 ID |

#### Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| startLabel | string | N | 출발지 표시명 |
| endLabel | string | N | 도착지 표시명 |

#### Request Example

```json
{
  "startLabel": "부산시민공원",
  "endLabel": "부산역"
}
```

### Response

- HTTP Status: `200 OK`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| favRouteId | Long | N | 경로 북마크 ID |

#### Success Example

```json
{
  "status": "S2000",
  "data": {
    "favRouteId": 1
  },
  "message": "정상 처리되었습니다."
}
```

### Error

| HTTP Status | status | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | C4000 | 잘못된 입력입니다. | 요청 body 또는 path variable 형식 오류 |
| 400 | FR4001 | 경로 북마크 수정 요청값이 올바르지 않습니다. | 수정 필드가 없거나 경로 북마크 수정 규칙 위반 |
| 401 | A4010 | 인증이 필요합니다. | 토큰 누락/만료 |
| 403 | FR4030 | 경로 북마크에 대한 권한이 없습니다. | 다른 사용자의 경로 북마크 수정 시도 |
| 404 | FR4040 | 경로 북마크를 찾을 수 없습니다. | 경로 북마크가 존재하지 않음 |

### 처리 규칙

- 저장 시 `routeId + userId` 기준으로 `route_sessions`를 조회하고 소유권을 검증한다.
- `startPoint`, `endPoint`, `transportMode`, `routeOption`, `routeSnapshotJson`은 route session에서 복사한다.
- `startLabel`, `endLabel`은 프론트가 표시명 우선순위에 따라 전달한다.
- 표시명 우선순위는 장소명, 도로명주소, 지번주소 순이다.
- 수정 API는 표시명만 변경한다. 저장된 좌표, 옵션, route snapshot은 변경하지 않는다.
- `routeName`은 `startLabel`과 `endLabel` 기준으로 자동 생성하며, 최대 511자다.
