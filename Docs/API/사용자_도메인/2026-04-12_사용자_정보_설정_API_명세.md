# 📋 사용자 정보/설정 API 명세

> **작성일:** 2026-04-12  
> **작성자:** 김응서  
> **최종 수정일:** 2026-05-07

---

## 1. API 목록

| 우선순위 | API | 설명 |
| --- | --- | --- |
| P1 | `GET /users/me` | 내 정보 조회 |
| P2 | `PATCH /users/me/user-type` | 사용자 유형 수정 |

---

## 2. 내 정보 조회

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `GET` |
| URL | `/users/me` |
| 인증 | 필요 |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |

#### Query Parameter

> 없음

#### Body

> 없음

### Response

- HTTP Status: `200 OK`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| userId | string(UUID) | N | 사용자 ID |
| socialProvider | string | N | 소셜 제공자 |
| selectedPrimaryUserType | string | N | 1차 사용자 유형 |
| selectedMobilitySubtype | string | Y | 보행약자 세부 유형. 저시력자는 `null` |

#### 비고

- `users` row는 가입 완료 사용자만 저장하므로 `selectedPrimaryUserType`은 항상 값이 있다.
- `selectedPrimaryUserType` 후보값은 `LOW_VISION`, `MOBILITY_IMPAIRED`다.
- `selectedPrimaryUserType=LOW_VISION`이면 `selectedMobilitySubtype=null`이다.
- `selectedPrimaryUserType=MOBILITY_IMPAIRED`이면 `selectedMobilitySubtype`은 한 개만 저장한다. 후보값은 `POWER_WHEELCHAIR`, `MANUAL_WHEELCHAIR`, `OTHER_MOBILITY`다.
- 관리자 진입 판단은 일반 사용자 정보 응답이 아니라 `users.role=ADMIN`과 `GET /admin/me`를 기준으로 처리한다.
- 서버 사용자 정보에 앱 닉네임, 온보딩 완료 여부, 약관 동의 여부, TTS 설정, 경로 수집 설정은 포함하지 않는다.
- 푸시 알림, 진동 알림, TTS 설정은 앱 내부 설정으로 관리한다.

#### Success Example

```json
{
  "status": "S2000",
  "data": {
    "userId": "018f7f6c-2b7e-7c3a-9f4a-8b4e3b7c9a01",
    "socialProvider": "KAKAO",
    "selectedPrimaryUserType": "MOBILITY_IMPAIRED",
    "selectedMobilitySubtype": "MANUAL_WHEELCHAIR"
  },
  "message": "정상 처리되었습니다."
}
```

### Error

| HTTP Status | status | message | 발생 조건 |
| --- | --- | --- | --- |
| 401 | A4010 | 인증이 필요합니다. | 토큰 누락/만료 |
| 404 | U4040 | 사용자를 찾을 수 없습니다. | 사용자 정보가 존재하지 않음 |

## 3. 사용자 유형 수정

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P2 |
| Method | `PATCH` |
| URL | `/users/me/user-type` |
| 인증 | 필요 |
| 설명 | 마이페이지의 사용자 유형 변경에서 1차 사용자 유형과 조건부 보행약자 세부 유형을 수정한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |
| Content-Type | Y | `application/json` |

#### Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| selectedPrimaryUserType | string | Y | 1차 사용자 유형. `LOW_VISION`, `MOBILITY_IMPAIRED` 중 하나 |
| selectedMobilitySubtype | string | 조건부 | 보행약자 세부 유형. `selectedPrimaryUserType=MOBILITY_IMPAIRED`이면 필수, `LOW_VISION`이면 `null` 또는 미전송 |

#### Request Example

```json
{
  "selectedPrimaryUserType": "MOBILITY_IMPAIRED",
  "selectedMobilitySubtype": "MANUAL_WHEELCHAIR"
}
```

### Response

- HTTP Status: `200 OK`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| userId | string(UUID) | N | 사용자 ID |
| selectedPrimaryUserType | string | N | 저장된 1차 사용자 유형 |
| selectedMobilitySubtype | string | Y | 저장된 보행약자 세부 유형. 저시력자는 `null` |

#### Success Example

```json
{
  "status": "S2000",
  "data": {
    "userId": "018f7f6c-2b7e-7c3a-9f4a-8b4e3b7c9a01",
    "selectedPrimaryUserType": "MOBILITY_IMPAIRED",
    "selectedMobilitySubtype": "MANUAL_WHEELCHAIR"
  },
  "message": "정상 처리되었습니다."
}
```

### Error

| HTTP Status | status | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | U4000 | 사용자 요청값이 올바르지 않습니다. | 사용자 유형 조합 오류 또는 `selectedMobilitySubtype` 형식 오류 |
| 401 | A4010 | 인증이 필요합니다. | 토큰 누락/만료 |
| 404 | U4040 | 사용자를 찾을 수 없습니다. | 사용자 정보가 존재하지 않음 |
