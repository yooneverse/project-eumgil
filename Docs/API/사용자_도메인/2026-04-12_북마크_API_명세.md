# 📋 북마크 API 명세

> **작성일:** 2026-04-12  
> **작성자:** 김응서  
> **최종 수정일:** 2026-05-09

---

## 1. API 목록

| 우선순위 | API | 설명 |
| --- | --- | --- |
| P1 | `GET /bookmarks` | 저장한 장소 목록 조회 |
| P1 | `POST /bookmarks` | 내부 장소 또는 외부 장소 북마크 등록 |
| P1 | `DELETE /bookmarks/places/{placeId}` | 내부 장소 북마크 해제 |
| P1 | `DELETE /bookmarks/targets/{bookmarkTargetId}` | 내부/외부 공통 북마크 해제 |

> 북마크 서버 API는 로그인 사용자의 서버 저장 데이터만 처리한다. 운영 기준 비로그인 북마크 저장은 제공하지 않는다.

## 2. 저장한 장소 목록 조회

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `GET` |
| URL | `/bookmarks` |
| 인증 | 필요 |
| 설명 | 로그인 사용자가 서버에 저장한 장소 북마크 목록을 조회한다. 내부 장소는 canonical 데이터를 우선 사용하고, 내부 매칭되지 않은 외부 대상은 저장된 snapshot을 반환한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |

#### Query Parameter

| 변수명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| cursor | Long | N | 마지막으로 받은 `bookmarkId`. 없으면 첫 페이지부터 조회 |
| size | int | N | 페이지 크기 |

#### Body

> 없음

### Response

- HTTP Status: `200 OK`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| content | array | N | 북마크 장소 목록 |
| content[].bookmarkId | Long | N | 북마크 ID |
| content[].bookmarkTargetId | string | N | 북마크 대상 식별자. 삭제 API와 목록 diff 기준으로 사용 |
| content[].targetType | string | N | `INTERNAL_PLACE`, `EXTERNAL_POI`, `EXTERNAL_ADDRESS` 중 하나 |
| content[].placeId | Long | Y | 내부 장소 ID. 내부 매칭 실패 외부 대상은 `null` |
| content[].provider | string | Y | 장소 원천. 현재 외부 대상은 `KAKAO` |
| content[].providerPlaceId | string | Y | 카카오 장소 ID |
| content[].name | string | N | 장소명 |
| content[].category | string | Y | 내부 장소와 연결된 경우의 장소 카테고리. 장소 도메인의 `PlaceCategory`를 따른다. |
| content[].providerCategory | string | Y | 외부 제공자의 원본 카테고리 문자열 |
| content[].address | string | Y | 주소 |
| content[].point | object | N | 장소 좌표 |
| content[].accessibilityFeatures | array | N | 내부 장소인 경우 어떤 시설/접근성 항목인지 나타내는 목록 |
| content[].accessibilityFeatures[].featureType | string | N | `accessibleToilet`, `elevator`, `chargingStation` 등 |
| content[].accessibilityFeatures[].isAvailable | boolean | N | 현재 사용 가능 여부 |
| size | int | N | 페이지 크기 |
| nextCursor | Long | Y | 다음 조회에 사용할 마지막 `bookmarkId` |
| hasNext | boolean | N | 다음 페이지 존재 여부 |

#### Success Example

```json
{
  "status": "S2000",
  "data": {
    "content": [
      {
        "bookmarkId": 1,
        "bookmarkTargetId": "tgt_9d13f0b44d68abcd",
        "targetType": "EXTERNAL_POI",
        "placeId": null,
        "provider": "KAKAO",
        "providerPlaceId": "123456789",
        "name": "부산시민공원",
        "category": null,
        "providerCategory": "여행 > 관광,명소 > 공원",
        "address": "부산광역시 부산진구 시민공원로 73",
        "point": {
          "lat": 35.1686,
          "lng": 129.0576
        },
        "accessibilityFeatures": []
      }
    ],
    "size": 10,
    "nextCursor": null,
    "hasNext": false
  },
  "message": "정상 처리되었습니다."
}
```

#### 비고

- 목록 정렬은 최신 북마크순을 기본으로 한다.

### Error

| HTTP Status | status | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | BM4000 | 북마크 요청값이 올바르지 않습니다. | `cursor`, `size` 형식 또는 범위 오류 |
| 401 | A4010 | 인증이 필요합니다. | 토큰 누락/만료 |

## 3. 장소 북마크 등록

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `POST` |
| URL | `/bookmarks` |
| 인증 | 필요 |
| 설명 | 로그인 사용자의 장소를 서버 북마크로 등록한다. 내부 장소는 canonical link로 저장하고, 내부 매칭되지 않은 외부 대상은 사용자 snapshot으로 저장한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |
| Content-Type | Y | `application/json` |

#### Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| placeId | Long | N | 내부 장소 ID. 있으면 내부 장소 북마크로 처리 |
| provider | string | N | 외부 제공자. 현재는 `KAKAO`만 허용 |
| providerPlaceId | string | N | 외부 제공자 장소 ID. POI 외부 북마크인 경우 전달 권장 |
| name | string | N | 외부 장소명 또는 주소 제목. `placeId`가 없으면 필수 |
| providerCategory | string | N | 외부 제공자의 원본 카테고리 문자열 |
| address | string | N | 외부 표시 주소 |
| point | object | N | 외부 대상 좌표. `placeId`가 없으면 필수 |
| point.lat | double | N | 외부 대상 위도 |
| point.lng | double | N | 외부 대상 경도 |

#### Request Example

```json
{
  "provider": "KAKAO",
  "providerPlaceId": "123456789",
  "name": "부산시민공원",
  "providerCategory": "여행 > 관광,명소 > 공원",
  "address": "부산광역시 부산진구 범전동 200",
  "point": {
    "lat": 35.1686,
    "lng": 129.0576
  }
}
```

### Response

- HTTP Status: `201 Created`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| bookmarkId | Long | N | 북마크 ID |
| bookmarkTargetId | string | N | 북마크 대상 식별자 |
| targetType | string | N | `INTERNAL_PLACE`, `EXTERNAL_POI`, `EXTERNAL_ADDRESS` 중 하나 |
| placeId | Long | Y | 내부 장소 ID. 외부 snapshot만 저장된 경우 `null` |

#### Success Example

```json
{
  "status": "S2010",
  "data": {
    "bookmarkId": 1,
    "bookmarkTargetId": "tgt_9d13f0b44d68abcd",
    "targetType": "EXTERNAL_POI",
    "placeId": null
  },
  "message": "생성되었습니다."
}
```

### Error

| HTTP Status | status | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | BM4000 | 북마크 요청값이 올바르지 않습니다. | `placeId`와 외부 snapshot 필수값이 모두 없거나, body 형식/JSON이 올바르지 않거나, 지원하지 않는 provider를 보낸 경우 |
| 401 | A4010 | 인증이 필요합니다. | 토큰 누락/만료 |
| 404 | BM4040 | 장소를 찾을 수 없습니다. | `placeId`로 요청한 내부 장소가 존재하지 않음 |
| 409 | BM4090 | 이미 북마크한 장소입니다. | 이미 북마크한 장소 |

### 처리 규칙

- `placeId`가 있으면 내부 장소 북마크로 처리하고, 응답/목록에서는 내부 `places` canonical 데이터를 우선 사용한다.
- `placeId`가 없으면 외부 snapshot 북마크로 처리한다. 이때 `name`, `point`는 필수이며, `providerPlaceId`, `providerCategory`, `address`는 있으면 함께 저장한다.
- 외부 요청이라도 `providerPlaceId`가 내부 `places.providerPlaceId`와 매칭되면 백엔드는 내부 장소 link를 우선 생성할 수 있다.
- 백엔드는 `bookmarkTargetId`를 생성해 중복 여부와 삭제 기준으로 사용한다.
- 외부 snapshot은 전역 장소 마스터 데이터로 승격하지 않고 사용자 북마크 데이터로만 저장한다.
- `placeId`는 우리 DB의 내부 자동 증가 ID다. 카카오 장소 ID를 내부 PK로 사용하지 않는다.
- 조회 응답에서 내부 장소는 `accessibilityFeatures`를 같이 내려주므로, FE는 이를 이용해 장애인 화장실/엘리베이터/충전소 등 시설 성격을 표시할 수 있다.

## 4. 내부 장소 북마크 해제

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `DELETE` |
| URL | `/bookmarks/places/{placeId}` |
| 인증 | 필요 |
| 설명 | 로그인 사용자의 서버 북마크를 내부 장소 ID 기준으로 해제한다. 내부 마커 중심의 기존 호출 호환성을 위해 유지한다. 신규 구현은 `bookmarkTargetId` 기준 삭제를 우선 사용한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |

#### Path Variable

| 변수명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| placeId | Long | Y | 북마크 해제할 장소 ID |

#### Body

> 없음

### Response

- HTTP Status: `204 No Content`
- 응답 Body 없음

### Error

| HTTP Status | status | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | BM4001 | 북마크 해제 요청값이 올바르지 않습니다. | `placeId` 형식 오류 |
| 401 | A4010 | 인증이 필요합니다. | 토큰 누락/만료 |
| 404 | BM4041 | 북마크 내역을 찾을 수 없습니다. | 북마크 내역이 존재하지 않음 |

## 5. 공통 북마크 해제

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `DELETE` |
| URL | `/bookmarks/targets/{bookmarkTargetId}` |
| 인증 | 필요 |
| 설명 | 로그인 사용자의 서버 북마크를 `bookmarkTargetId` 기준으로 해제한다. 내부/외부 북마크 공통 삭제 경로다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | Bearer 액세스 토큰 |

#### Path Variable

| 변수명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| bookmarkTargetId | string | Y | 북마크 해제할 대상 식별자 |

> `bookmarkTargetId` 형식은 `tgt_[0-9a-f]{16}` 이다.

#### Body

> 없음

### Response

- HTTP Status: `204 No Content`
- 응답 Body 없음

### Error

| HTTP Status | status | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | BM4001 | 북마크 해제 요청값이 올바르지 않습니다. | `bookmarkTargetId` 형식 오류 |
| 401 | A4010 | 인증이 필요합니다. | 토큰 누락/만료 |
| 404 | BM4041 | 북마크 내역을 찾을 수 없습니다. | 북마크 내역이 존재하지 않음 |
