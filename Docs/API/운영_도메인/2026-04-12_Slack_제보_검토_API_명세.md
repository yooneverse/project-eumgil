> **상태:** 폐기된 초안
> 제보 승인/반려는 Slack 콜백으로 처리하지 않고, 관리자 페이지와 관리자 API에서 처리한다.

## 1. API 목록

| 우선순위 | API | 설명 |
| --- | --- | --- |
| HOLD | `POST /slack/hazard-reports/actions` | 폐기된 Slack 제보 승인/반려 콜백 초안 |

---

## 2. Slack 제보 승인/반려 콜백

| 항목 | 내용 |
| --- | --- |
| 우선순위 | HOLD |
| Method | `POST` |
| URL | `/slack/hazard-reports/actions` |
| 인증 | Slack 서명 검증 |
| 설명 | 폐기된 Slack 콜백 초안이다. `hazard_reports.status` 변경은 관리자 API에서 처리한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| X-Slack-Signature | Y | Slack 요청 서명 |
| X-Slack-Request-Timestamp | Y | Slack 요청 시각 |
| Content-Type | Y | `application/x-www-form-urlencoded` |

#### Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| payload | string | Y | Slack interactive payload JSON 문자열 |

#### Decoded Payload Fields

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| reportId | Long | Y | 제보 ID. Slack 버튼 `value` 또는 `private_metadata`에서 추출 |
| action | string | Y | 처리 액션. `APPROVE`, `REJECT` |
| slackUserId | string | Y | 버튼을 클릭한 Slack 사용자 ID |
| responseUrl | string | N | Slack 메시지 후속 응답 또는 업데이트에 사용할 URL |
| rejectReason | string | N | 반려 사유. `REJECT`일 때 사용 |

#### Request Example

Slack 요청 Body 예시:

```text
payload={...}
```

서버에서 해석한 payload 예시:

```json
{
  "reportId": 1,
  "action": "APPROVE",
  "slackUserId": "U123456",
  "responseUrl": "https://hooks.slack.com/actions/...",
  "rejectReason": null
}
```

### Response

- HTTP Status: `200 OK`
- Slack 콜백 API는 앱 클라이언트용 공통 응답 래퍼를 사용하지 않고, Slack 요청 확인용으로 빠르게 `200 OK`를 반환한다.
- Slack 메시지 변경이 필요하면 payload의 `response_url` 또는 Slack Web API로 후속 처리한다.

#### Success Example

```text
HTTP/1.1 200 OK
```

### Error

| HTTP Status | status | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | OP4000 | 운영 콜백 요청값이 올바르지 않습니다. | payload 누락, action 값 오류, reportId 누락 |
| 401 | OP4010 | 운영 콜백 인증에 실패했습니다. | Slack 서명 검증 실패 |
| 404 | OP4040 | 운영 처리 대상을 찾을 수 없습니다. | 제보가 존재하지 않음 |
| 409 | OP4090 | 이미 처리된 요청입니다. | 이미 `APPROVED` 또는 `REJECTED` 처리된 제보 |

### 처리 규칙

- Slack 버튼 클릭 요청은 `X-Slack-Signature`, `X-Slack-Request-Timestamp`로 검증한다.
- `APPROVE` 액션이면 `hazard_reports.status`를 `APPROVED`로 변경한다.
- `REJECT` 액션이면 `hazard_reports.status`를 `REJECTED`로 변경한다.
- 이미 `APPROVED` 또는 `REJECTED` 상태인 제보는 중복 처리하지 않고 `409 CONFLICT`로 응답한다.
- Slack 타임아웃을 피하기 위해 요청 확인 응답은 빠르게 반환하고, 메시지 업데이트는 후속 처리할 수 있다.
- Slack 응답 메시지에는 제보자 개인정보를 포함하지 않는다.
- 사용자 앱에는 처리 상태를 노출하지 않는다. 상태값은 운영 검토와 지도/장소 반영 여부 판단에만 사용한다.
