# 저상버스 예약 외부 호출 명세

- 작성일: 2026-05-13
- 범위: 부산 BIMS 모바일 `low_request.asp` 직접 호출 계약
- 상태: 구현 전/구현 중 확인 명세

## 결론

저상버스 예약 요청은 부산 BIMS 모바일 페이지의 `low_request.asp` GET 호출로 처리된다.

이 요청은 조회가 아니라 운전자 단말 메시지 전송/예약 완료로 이어지는 외부 상태 변경 요청이므로, 앱은 사용자 명시 확인 이후에만 호출해야 한다.

## 호출 URL

```text
https://bus.busan.go.kr/busanBIMS/mobile/webApp/page/customer/low_request.asp
```

## 파라미터 계약

| 파라미터 | 예시 | 필수 | 설명 |
| --- | --- | --- | --- |
| `svc_yy` | `2026` | 예 | 요청 시점의 연도 |
| `svc_mm` | `5` | 예 | 요청 시점의 월. 공식 JS는 zero-padding하지 않음 |
| `svc_dd` | `13` | 예 | 요청 시점의 일. 공식 JS는 zero-padding하지 않음 |
| `svc_tm` | `141322` | 예 | 요청 시점의 `HHmmss` |
| `svc_wd` | `3` | 예 | 요청 시점의 요일. 일요일 `0`, 월요일 `1` |
| `bstop_nm` | `%BA%CE%BB%EA%BF%AA` | 예 | 정류소명. BIMS 모바일 페이지 흐름에서는 EUC-KR query encoding 사용 |
| `ars_no` | `70001` | 예 | 정류소 ARS 번호 |
| `line_no` | `7000` | 예 | 버스 노선 번호 |
| `car_no` | `1618` | 예 | 차량 번호 |
| `wait_tm` | `14` | 예 | 도착 예정 분 |
| `wait_dist` | `2` | 조건부 | 남은 정류장 수. 공식 JS는 값이 없으면 `1`로 대체 |

## 관측 샘플

```text
GET /busanBIMS/mobile/webApp/page/customer/low_request.asp?svc_yy=2026&svc_mm=5&svc_dd=13&svc_tm=141322&svc_wd=3&bstop_nm=%BA%CE%BB%EA%BF%AA&ars_no=70001&line_no=7000&car_no=1618&wait_tm=14&wait_dist=2
```

관측 흐름:

1. `trafficWeak.asp`에서 저상버스 예약하기 진입
2. `trafficWeakList.asp`에서 정류소명 검색
3. `trafficWeakResult.asp`에서 정류소/노선/차량 도착 목록 확인
4. 사용자가 차량의 예약 버튼을 누르면 confirm 후 `low_request.asp` GET 호출

## 실패 응답 처리 기준

공식 모바일 페이지의 프론트 스크립트는 `low_request.asp` 응답 본문을 파싱하지 않고 HTTP status `200`만 성공으로 취급한다. 별도 오류 코드/JSON 계약은 확인되지 않았다.

따라서 앱 구현은 다음 기준을 따른다.

- 사용자 confirm 이전에는 절대 호출하지 않는다.
- 앱 시작, 화면 진입, 카드 노출, prefetch, retry로 호출하지 않는다.
- HTTP `2xx`만 성공 toast로 처리한다.
- non-`2xx`, timeout, 네트워크 예외는 실패 toast로 처리한다.
- 실패 시 자동 재시도하지 않는다. 같은 요청 재전송은 중복 예약/중복 메시지 위험이 있으므로 사용자의 재확인이 필요하다.

## 구현 입력 조건

앱의 예약 CTA는 다음 값이 모두 있을 때만 노출한다.

- 저상버스 여부가 `true`
- 도착 예정 분
- 정류소명
- ARS 번호
- 노선 번호
- 차량 번호

차량 번호 또는 ARS 번호가 없으면 `low_request.asp` 호출 계약을 만족하지 못하므로 CTA를 숨긴다.
