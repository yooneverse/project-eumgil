## 3. STT 텍스트 의미 분석

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `POST` |
| URL | `/voice/analyze` |
| 인증 | 필요 (`Bearer accessToken`) |
| 설명 | FE가 STT로 변환한 텍스트를 AI 서버가 분석해 intent와 후속 실행에 필요한 파라미터를 추출한다. 장소 검색을 직접 수행하지 않으며, intent 해석 후 실제 검색/이동/화면 전환은 FE가 수행한다. |

### 3.1 Request Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Authorization | Y | `Bearer {accessToken}` |
| Content-Type | Y | `application/json` |

### 3.2 Request Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| text | string | Y | STT 변환 텍스트 |
| mode | string | Y | 사용자 흐름. `MOBILITY_IMPAIRED`, `LOW_VISION` |
| history | array | N | 이전 대화 목록. 두 모드 모두 사용. 기본값 `[]` |
| history[].role | string | Y | 발화 주체. `user`, `assistant` |
| history[].content | string | Y | 발화 내용. `assistant`는 이전 응답 `data`를 JSON 문자열로 직렬화한 값 |
| currentRoute | string | N | 현재 화면 route. 전역 음성 어시스턴트(`MOBILITY_IMPAIRED`)에서만 전달. 예: `navigation/guidance`, `map` |

### 3.3 Request 규칙

- `MOBILITY_IMPAIRED`도 `history`를 AI 서버에 그대로 전달한다.
- `LOW_VISION`은 멀티턴 확인 흐름을 위해 `history`를 계속 누적한다.
- `currentRoute`는 선택 필드이며, 없으면 `null`로 처리한다.
- 요청에 `model` 필드는 받지 않는다. AI 모델 선택은 BE/AI 서버 내부 설정으로 관리한다.

### 3.4 Request 예시

#### 이동약자 1턴 제보

```json
{
  "text": "제보할게요",
  "mode": "MOBILITY_IMPAIRED",
  "currentRoute": "navigation/guidance",
  "history": []
}
```

#### 이동약자 2턴 제보 유형

```json
{
  "text": "계단이요",
  "mode": "MOBILITY_IMPAIRED",
  "currentRoute": "navigation/guidance",
  "history": [
    { "role": "user", "content": "제보할게요" },
    { "role": "assistant", "content": "{\"intent\":\"ASK\",\"placeName\":null,\"category\":null,\"bookmarkAction\":null,\"departure\":null,\"destination\":null,\"reportType\":null,\"description\":null,\"confirmed\":null,\"confirmationMessage\":null}" }

```

#### 저시력자 장소 검색 1턴

```json

  "text": "부산역 어디야",
  "mode": "LOW_VISION",
  "history": []

```

#### 저시력자 장소 검색 2턴 긍정

## 4. Response

- HTTP Status: `200 OK`

### 4.1 Success Wrapper

```json
{
  "status": "S2000",
  "data": {},
  "message": "정상 처리되었습니다."
}
```

### 4.2 `data` 필드

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| intent | string | N | 추론된 의도 |
| placeName | string | Y | 장소명 |
| category | string | Y | 카테고리명 |
| bookmarkAction | string | Y | 북마크 액션. `add`, `delete` |
| departure | string | Y | 출발지. `NAVIGATE`에서 `null`이면 FE가 현재 GPS 사용 |
| destination | string | Y | 도착지 |
| reportType | string | Y | 제보 유형 |
| description | string | Y | 제보 설명 |
| confirmed | boolean | Y | 저시력자 확인 상태. `true`, `false`, `null` |
| confirmationMessage | string | Y | 저시력자 확인 질문 또는 안내 문구 |

### 4.3 Intent별 필수 필드

| intent | 필수 필드 | 설명 |
| --- | --- | --- |
| `PLACE_SEARCH` | `placeName` | 장소 검색 |
| `CATEGORY_SEARCH` | `category` | 카테고리 검색 |
| `BOOKMARK_ADD` | `placeName`, `bookmarkAction` | 북마크 추가 |
| `BOOKMARK_DELETE` | `placeName`, `bookmarkAction` | 북마크 삭제 |
| `NAVIGATE` | `destination` | 경로 안내. `departure`는 선택 |
| `REPORT` | `reportType` | 제보하기. `description`은 선택 |
| `SHOW_BOOKMARKS` | 없음 | 북마크 목록 이동 |
| `SHOW_FAVORITE_ROUTES` | 없음 | 저장 경로 목록 이동 |
| `LOGOUT` | 없음 | 로그아웃 |
| `NAVIGATION_END` | 없음 | 길 안내 종료 |
| `OPEN_MY_PAGE` | 없음 | 마이페이지 이동 |
| `OPEN_MAP` | 없음 | 지도 홈 이동 |
| `ASK` | 없음 | 추가 정보 수집 |
| `UNKNOWN` | 없음 | 의도 미인식 |

### 4.4 `reportType` 후보값

- `STAIRS_STEP`
- `BRAILLE_BLOCK`
- `SIDEWALK_MISSING`
- `RAMP`
- `SIDEWALK_WIDTH`
- `OTHER_OBSTACLE`

## 5. Intent 목록

| intent | 모드 | 설명 |
| --- | --- | --- |
| `PLACE_SEARCH` | 공통 | 장소 검색. FE는 `GET /places/search?keyword={placeName}` 호출 |
| `CATEGORY_SEARCH` | 공통 | 카테고리 검색. FE는 카테고리 기반 화면/조회 흐름으로 이동 |
| `BOOKMARK_ADD` | 공통 | 북마크 추가 |
| `BOOKMARK_DELETE` | 공통 | 북마크 삭제 |
| `NAVIGATE` | 공통 | 경로 안내 시작 |
| `SHOW_BOOKMARKS` | 공통 | 북마크 목록 이동 |
| `SHOW_FAVORITE_ROUTES` | 공통 | 저장 경로 목록 이동 |
| `LOGOUT` | 공통 | 로그아웃 |
| `REPORT` | 공통 | 제보 화면 이동 및 파라미터 전달 |
| `NAVIGATION_END` | 공통 | 길 안내 종료 |
| `OPEN_MY_PAGE` | 공통 | 마이페이지 이동 |
| `OPEN_MAP` | 공통 | 지도 홈 이동 |
| `ASK` | 공통 | 추가 정보 수집. `LOW_VISION`은 `confirmationMessage` TTS, `MOBILITY_IMPAIRED`는 FE 고정 멘트 사용 |
| `UNKNOWN` | 공통 | 의도 미인식 |

## 6. 모드별 동작

| 항목 | `LOW_VISION` | `MOBILITY_IMPAIRED` |
| --- | --- | --- |
| 대화 방식 | 멀티턴 (`history` 누적) | 멀티턴 (`history` 누적) |
| 확인 과정 | 있음 | 없음 |
| `confirmed` 사용 | `null / true / false` | 응답 계약상 `null` |
| `confirmationMessage` 사용 | 확인 질문/TTS 문구 | 응답 계약상 `null`, ASK 안내는 FE 고정 멘트 |
| `history` 전달 | FE -> BE -> AI 그대로 전달 | FE -> BE -> AI 그대로 전달 |
| `currentRoute` 전달 | 보통 미전달 | 전달 가능 |

## 7. Assistant History 직렬화 규칙

FE가 `history[].content`에 assistant 응답을 직렬화할 때는 아래 필드를 모두 포함한다. `null`이어도 생략하지 않는다.

- `intent`
- `placeName`
- `category`
- `bookmarkAction`
- `departure`
- `destination`
- `reportType`
- `description`
- `confirmed`
- `confirmationMessage`

## 8. 에러 응답

| HTTP Status | status | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | C4000 | 잘못된 입력입니다. | `text` 누락/공백, `mode` 누락/오류, `history` 형식 오류 |
| 401 | A4010 | 인증이 필요합니다. | 토큰 누락/만료 |
| 502 | V5020 | 음성 분석 AI 호출에 실패했습니다. | AI 서버 호출 실패 또는 AI 응답 계약 위반 |

### Error Example

```json
{
  "status": "C4000",
  "message": "text: 공백일 수 없습니다."
}
```

## 9. 책임 분리

| 주체 | 책임 |
| --- | --- |
| FE | 마이크 권한, STT, `history` 관리, TTS, intent별 화면 전환, `currentRoute` 전달 |
| BE | `/voice/analyze` 제공, AI 서버 호출, intent별 필수 필드 검증, 공통 응답 변환 |
| AI | intent와 파라미터 추출, 저시력자 확인 문장 생성, `currentRoute` 기반 문맥 해석 |
