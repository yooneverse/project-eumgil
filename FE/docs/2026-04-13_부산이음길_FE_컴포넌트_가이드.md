## 1. 컴포넌트 설계 원칙

1. 부산이음길은 일반 CRUD 앱이 아니라 지도 중심 서비스다. 따라서 카드보다 `지도 오버레이`, `검색`, `필터`, `바텀시트`, `권한 상태`가 더 중요하다.
2. 로그인 필수 정책이므로 미인증 사용자는 핵심 기능으로 진입하지 않는다. 다만 API 실패, 캐시 부재, 세션 만료 상태에서도 화면 구조가 무너지면 안 된다.
3. 디자인이 아직 변할 수 있으므로, 지금은 **토큰과 primitive는 안정적으로**, **도메인 composite는 얇게** 가져간다.
4. 접근성은 나중에 붙이는 옵션이 아니라 컴포넌트 정의 자체에 포함한다.
5. Light mode only로 간다. 다크 모드는 지금 논의하지 않는다.

## 2. 디자인 토큰 체계

### 2.1 토큰 계층

```text
Token
└─ Semantic Token
   └─ Primitive Component
      └─ Map Composite Component
         └─ Screen-only UI
```

#### 기본 규칙

- Figma에서 먼저 토큰 이름을 확정하고, 코드도 같은 의미 이름으로 맞춘다.
- raw hex, raw radius, raw spacing은 screen 코드에서 쓰지 않는다.
- 토큰 변경은 `token -> primitive -> composite -> screen` 순서로 반영한다.

### 2.2 Color 규칙

부산이음길은 **화이트 베이스 + Primary 600 블루 + 시맨틱 상태색**으로 간다.

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `bg/default` | `#FFFFFF` | 앱 기본 배경 |
| `bg/surface` | `#FFFFFF` | 카드, 시트, 입력 필드 |
| `bg/muted` | `#F3F4F6` | 비활성, 구분 영역 |
| `bg/map-overlay` | `rgba(17, 24, 39, 0.32)` | 지도 위 dim |
| `text/primary` | `#111827` | 제목, 본문 |
| `text/secondary` | `#374151` | 보조 설명 |
| `text/tertiary` | `#6B7280` | placeholder, 메타 |
| `border/default` | `#D1D5DB` | 기본 보더 |
| `border/strong` | `#93C5FD` | 강조 보더 |
| `action/primary` | `#2563EB` | 기본 CTA |
| `action/primary-support` | `#3B82F6` | 보조 강조, 링크, 강조 아이콘 |
| `action/selected-bg` | `#DBEAFE` | 선택 카드, tint 배경 |
| `status/success` | `#16A34A` | 완료 |
| `status/warning` | `#F59E0B` | 주의 |
| `status/danger` | `#EF4444` | 오류, 파손, 위험 |

#### 컬러 사용 원칙

- 지도 서비스 핵심 CTA는 `action/primary`를 기본으로 쓴다.
- `action/selected-bg`는 primary 대체가 아니라 선택 상태와 약한 강조 배경에만 쓴다.
- 경로 위험도, 오류, 권한 차단은 색 하나로만 표현하지 않는다. 아이콘, 텍스트, 상태 라벨을 같이 준다.
- 전체 화면 배경을 노란색, 파란색으로 채우지 않는다.

### 2.3 Typography 규칙

- 기본 폰트는 `Pretendard`를 쓴다.
- 본문 최소 기준은 기본 모드 `16sp`, 시각보조 모드 `18sp`다.
- 핵심 길안내 정보는 `label`이 아니라 `title` 이상 스케일을 쓴다.

| 토큰 | 기본 크기/줄간격 | 용도 |
| --- | --- | --- |
| `display/lg` | `32 / 40` | 온보딩 핵심 문구 |
| `headline/lg` | `28 / 36` | 화면 대표 제목 |
| `title/lg` | `22 / 30` | 카드/시트 제목 |
| `title/md` | `18 / 26` | 섹션 제목, 길안내 핵심 문구 |
| `body/lg` | `16 / 24` | 기본 본문 |
| `body/md` | `14 / 22` | 보조 본문 |
| `label/lg` | `14 / 20` | 버튼, 칩 |
| `label/md` | `12 / 18` | 메타, 상태 |

#### 시각보조 모드 규칙

- `body/lg` 미만 텍스트를 주요 액션 영역에 쓰지 않는다.
- 길안내 진행 카드, 권한 요청, 제보 상태, 바텀시트 주요 버튼은 `18sp` 이상으로 올린다.
- `maxLines = 1`은 하단 탭 라벨, 짧은 상태 배지 정도에만 허용한다.

### 2.4 Spacing 규칙

4dp 기반 그리드를 고정한다.

| 토큰 | 값 |
| --- | --- |
| `space/01` | `4dp` |
| `space/02` | `8dp` |
| `space/03` | `12dp` |
| `space/04` | `16dp` |
| `space/05` | `20dp` |
| `space/06` | `24dp` |
| `space/08` | `32dp` |

#### 기본 규칙

- 화면 좌우 기본 패딩은 `16dp`
- 카드/시트 내부 기본 패딩은 `16dp`
- FAB와 화면 가장자리 간격은 `16dp`
- 검색바와 지도 상단 safe area 간격은 `12dp` 이상

### 2.5 Radius 규칙

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `radius/xs` | `4dp` | 얇은 배지, 작은 보조 요소 |
| `radius/sm` | `8dp` | chip, 작은 입력 |
| `radius/md` | `12dp` | 버튼, 입력, 기본 카드 |
| `radius/lg` | `16dp` | 시트, 큰 카드 |
| `radius/xl` | `24dp` | 큰 선택 카드, 강조 surface |
| `radius/full` | `999dp` | pill, FAB |

### 2.6 Elevation 규칙

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `elevation/0` | `0dp` | 기본 surface |
| `elevation/1` | `2dp` | 기본 카드 |
| `elevation/2` | `6dp` | 지도 위 검색바, 시트 |
| `elevation/3` | `10dp` | modal dialog |

#### 규칙

- 같은 화면에서 elevation 단계는 최대 3개까지다.
- 지도 위 오버레이는 elevation으로만 띄우지 않는다. 배경, 보더, 그림자를 같이 쓴다.

### 2.7 Icon 규칙

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| `icon/sm` | `20dp` | chip, list meta |
| `icon/md` | `24dp` | 일반 icon button |
| `icon/lg` | `28dp` | 지도 액션, 상태 강조 |
| `icon/xl` | `32dp` | 시각보조/음성 중심 액션 |

#### 규칙

- 아이콘은 의미 그룹별로 나눈다.
  - 시스템 제어: back, close, chevron
  - 지도 제어: my location, route start, filter
  - 시설/마커: elevator, toilet, ramp, charger, tactile paving
- 시설 마커 아이콘은 Material 기본 아이콘으로 끝내지 않는다.
- 실제 Android 아이콘은 `FE/app/src/main/res/drawable/ic_*.xml`을 기준으로 한다.
- Compose 구현은 `painterResource(id = R.drawable.<resource_name>)`를 사용하고, 화면별 임의 SVG나 Material 기본 아이콘 대체를 섞지 않는다.
- 텍스트가 함께 있는 아이콘은 `contentDescription = null`로 두고, icon-only 액션은 호출부에서 구체적인 접근성 라벨을 넘긴다.

#### 리소스 그룹

| 그룹 | 리소스 prefix | 대표 리소스 | 컴포넌트 사용처 |
| --- | --- | --- | --- |
| 전역 탐색 | `ic_nav_*`, `ic_action_favorite` | `ic_nav_home`, `ic_action_favorite`, `ic_nav_report`, `ic_nav_mypage` | `EumTopLevelTabBar` |
| 시스템 액션 | `ic_action_*` | `ic_action_back`, `ic_action_close`, `ic_action_dropdown`, `ic_action_more` | `EumIconButton`, App Bar, Bottom Sheet |
| 권한/온보딩 | `ic_permission_*`, `ic_user_*` | `ic_permission_location`, `ic_permission_mic`, `ic_user_wheelchair` | 온보딩 카드, 권한 안내 카드 |
| 장소/시설 | `ic_place_*` | `ic_place_restroom`, `ic_place_charging`, `ic_place_welfare` | 지도 필터, 시설 카드, 마커 |
| 경로/접근성 | `ic_route_*`, `ic_direction_*` | `ic_route_slope`, `ic_route_elevator`, `ic_direction_turn_left` | 경로 요약, 단계별 안내, 접근성 칩 |
| 안내 제어 | `ic_control_*` | `ic_control_voice`, `ic_control_pause`, `ic_control_stop` | 길안내 제어 트레이, 시각 지원 흐름 |
| 제보/상태 | `ic_report_*`, `ic_status_*` | `ic_report_construction`, `ic_status_warning`, `ic_status_check` | 제보 유형 카드, 완료/오류/주의 상태 |

## 3. Light Mode 기준 원칙

- MVP는 Light mode only다.
- 흰 배경 위에 Primary 600 CTA, 어두운 텍스트, 명확한 보더를 기본으로 한다.
- 노란색 계열은 선택 카드 내부 아이콘 강조나 warning 성격 상태에 제한적으로 쓴다. 선택 상태 자체는 Primary 계열 토큰으로 전달한다.
- 지도 위 UI는 반투명 카드가 아니라 **불투명 흰 surface + 그림자 + 보더**를 기본으로 한다.
- 앱 전체를 베이지/크림/노랑 톤으로 칠하지 않는다.

## 4. 접근성 기준

### 4.1 최소 규칙

| 항목 | 기본 규칙 |
| --- | --- |
| 최소 터치 영역 | 일반 `48x48dp`, 핵심 CTA와 지도 액션 `56x56dp` |
| 대비 | 일반 텍스트 `4.5:1` 이상, 길안내/위험/권한 핵심 정보 `7:1` 이상 |
| 본문 텍스트 | 기본 `16sp` 이상, 시각보조 모드 `18sp` 이상 |
| 상태 표현 | 색상 단독 표현 금지 |
| TalkBack | 핵심 화면 구현 후가 아니라 컴포넌트 정의 시점부터 라벨 포함 |

### 4.2 Compose 기준 강제 규칙

#### `contentDescription`

- 텍스트가 이미 보이는 버튼/칩/탭에 같은 문장을 다시 넣지 않는다.
- icon-only 버튼은 반드시 라벨을 준다.
- 시설 카드처럼 여러 정보를 묶은 컴포넌트는 `clearAndSetSemantics` 또는 `mergeDescendants = true`를 쓴다.

```kotlin
Modifier.semantics(mergeDescendants = true) {
    role = Role.Button
    contentDescription = "부산시청, 휠체어 접근 가능, 길안내 열기"
}
```

#### `stateDescription`

- 선택 여부, 활성/비활성, 토글 상태는 `stateDescription`으로 보강한다.

```kotlin
Modifier.semantics {
    stateDescription = if (selected) "선택됨" else "선택 안 됨"
}
```

#### 포커스 순서

- 시각 순서와 TalkBack 순서가 다를 가능성이 있는 지도 오버레이에서는 `isTraversalGroup`, `traversalIndex`를 사용한다.
- 지도, 상단 검색바, 필터칩바, 상세 시트가 겹칠 때는 시트가 열린 순간 시트 내부가 우선 읽히도록 그룹을 나눈다.

#### live region

- 내비게이션 진행 안내 문구는 `LiveRegionMode.Polite` 이상으로 노출한다.
- 재탐색, 위치 미허용, 경로 탐색 실패 같은 긴급 상태는 snackbar만 띄우지 말고 화면 텍스트도 남긴다.

#### font scale 대응

- `fontScale = 1.3`, `1.5`에서 핵심 화면이 깨지지 않아야 한다.
- 지도 화면처럼 공간이 좁은 곳은 텍스트를 숨기지 말고 sheet나 card로 내린다.

### 4.3 지도 화면 접근성 추가 규칙

- 지도 마커 위치만으로 정보를 전달하지 않는다.
- 선택된 마커의 상세 정보는 항상 바텀시트나 리스트로 텍스트 대안을 제공한다.
- 필터는 아이콘만 두지 않는다. 텍스트 라벨을 같이 둔다.
- 현재 위치, 길안내 시작, 제보 버튼은 서로 충분히 떨어뜨린다.

## 5. 공통 레이아웃 템플릿 정의

### 5.1 `EumScreenScaffold`

기본 문서형/리스트형 화면에서 사용한다.

- 포함 슬롯: `topBar`, `bottomBar`, `snackbarHost`, `content`
- 기본 패딩: 좌우 `16dp`
- 상태 처리: loading / empty / error를 content 밖에서 감싸지 않는다. scaffold 안에서 처리한다.

### 5.2 `EumMapScreenScaffold`

지도 메인, 경로 미리보기, 내비 진행 계열에서 사용한다.

- 전체 배경은 지도다.
- 포함 슬롯:
  - `mapContent`
  - `topOverlay`
  - `chipBar`
  - `floatingActions`
  - `sheetContent`
  - `snackbarHost`
- 지도 위 오버레이 간 우선순위는 `topOverlay -> chipBar -> floatingActions -> sheetContent`다.

### 5.3 TopBar 규칙

- `EumTopBar`는 문서형 화면에서만 쓴다.
- 지도 메인은 일반 top bar 대신 `MapTopSearchBar`를 쓴다.
- 액션 아이콘은 최대 1개만 둔다.
- 제목은 짧고 명확하게 유지한다.

### 5.4 BottomBar 규칙

- MVP 하단 탭은 `지도`, `저장`, `MY` 3개만 둔다.
- `검색`은 탭 대신 지도 상단 검색 진입으로 우선 처리한다.
- 현재 탭 표시는 색상 외에 indicator 또는 weight 변화가 함께 있어야 한다.

### 5.5 BottomSheet 규칙

BottomSheet는 두 종류로 나눈다.

| 종류 | 용도 | 기본 규칙 |
| --- | --- | --- |
| `EumModalBottomSheet` | 범용 선택/상세/확인 | drag handle, 제목, CTA 영역 포함 |
| `MapOverlaySheet` | 시설 상세, 경로 요약, 위치 권한 상태 | 지도와 함께 열리고 닫히는 map 전용 시트 |

#### 규칙

- map 전용 시트를 성급하게 범용 공통 컴포넌트로 만들지 않는다.
- 대신 공통 container만 공유하고 내부 내용은 feature에서 조합한다.
- 바깥 탭 dismiss 금지는 권한/법적 동의처럼 명확한 경우에만 허용한다.

### 5.6 Dialog 규칙

- 확인/취소 구조를 기본으로 한다.
- 위치 권한/데이터 수집 동의처럼 시스템 행동 전 단계 설명이 필요한 경우에만 다이얼로그를 쓴다.
- full screen modal로 승격하는 기준은 설명이 길거나 OS 이동이 필요한 경우뿐이다.

### 5.7 Snackbar 규칙

- 길이: 최대 2줄
- 액션: 최대 1개
- 자동 사라짐에만 의존하지 않는다. 오류면 화면에도 남긴다.

## 6. 공통 컴포넌트 규칙

### 6.1 Button

코드 이름: `EumButton`

| Variant | 용도 | 색상 |
| --- | --- | --- |
| `Primary` | 주요 CTA | 딥블루 채움 |
| `Secondary` | 보조 CTA | 흰 배경 + 보더 |
| `Ghost` | 텍스트 액션 | 배경 없음 |
| `Danger` | 파괴적 액션 | 위험 컬러 텍스트 또는 보더 |

| Size | 높이 | 기본 용도 |
| --- | --- | --- |
| `Sm` | `32dp` | 보조/컴팩트 버튼 |
| `Md` | `40dp` | 일반 보조 버튼 |
| `Lg` | `48dp` | 핵심 CTA |
| `Xl` | `56dp` | 시각보조/음성 중심 액션 |

| State | 표현 |
| --- | --- |
| `Default` | 기본 컬러 |
| `Pressed` | 색상 + 그림자 약화 |
| `Disabled` | 비활성 컬러 + 클릭 불가 |
| `Loading` | 진행 indicator + 클릭 차단 |

#### 규칙

- 한 화면의 1차 CTA는 최대 1개다.
- 지도 상세 시트에서 `북마크`, `길안내 시작`처럼 두 개가 필요한 경우 primary와 secondary로 우선순위를 분명히 나눈다.
- 로그인은 필수 정책이므로 소셜 로그인 버튼은 `feature/auth` 내부의 필수 컴포넌트로 정의한다. 단, Kakao/Naver/Google 등 제공자별 브랜드 색과 문구는 공통 `EumButton` variant로 흡수하지 않고 `AuthSocialLoginButton`에서 별도 관리한다.

### 6.2 IconButton

코드 이름: `EumIconButton`

- 최소 터치 영역 `48dp`
- variant: `Filled`, `Outlined`, `Ghost`
- 지도 액션용 `Fab` 변형은 `56dp`를 기본으로 한다.
- `contentDescription` 필수
- `iconRes: Int`는 디자인 컨벤션의 리소스 매핑 표에 있는 `ic_*.xml`만 받는다.
- 텍스트 버튼 안에 들어가는 보조 아이콘은 `EumIconButton`이 아니라 해당 Button content slot에서 `Icon(contentDescription = null)`로 렌더링한다.
- toggle 성격의 icon-only 버튼은 `contentDescription`과 별도로 `stateDescription`으로 `선택됨/선택 안 됨`, `북마크됨/북마크 안 됨`, `재생 중/일시정지됨`을 전달한다.

#### 대표 icon-only 액션 라벨

| 사용 위치 | 리소스 | 접근성 라벨 예시 | 상태 전달 |
| --- | --- | --- | --- |
| App Bar 뒤로가기 | `ic_action_back` | `이전 화면으로 이동` | 없음 |
| 바텀시트 닫기 / 평가 닫기 | `ic_action_close` | `닫기` 또는 `평가 건너뛰기` | 없음 |
| 장소 북마크 | `ic_action_favorite` | `장소 북마크` | `북마크됨/북마크 안 됨` |
| 지도 현재 위치 / 권한 액션 | `ic_permission_location` 또는 `ic_status_refresh` | `현재 위치로 이동` 또는 `위치 다시 확인` | 권한/로딩 상태 문구 병행 |
| 음성 안내 제어 | `ic_control_voice`, `ic_control_play`, `ic_control_pause`, `ic_control_stop` | `음성 안내 켜기`, `브리핑 재생`, `브리핑 일시정지`, `안내 종료` | `재생 중/일시정지됨` |
| 경로 옵션 상세 | `ic_action_dropdown` 또는 `ic_control_next` | `경로 상세 보기` | 없음 |

### 6.3 TextField

코드 이름: `EumTextField`

| Type | 용도 |

| `Text` | 일반 입력 |
| `Search` | 검색 입력 |

| `TextArea` | 제보 설명 |

| State | 규칙 |

#### 규칙

- 제보 입력 필드는 멀티라인 높이를 고정치로 박지 않는다.

### 6.4 SearchBar

코드 이름: `EumSearchBar`, `MapTopSearchBar`

| Variant | 위치 |
| --- | --- |
| `MapTop` | 지도 상단 오버레이 |
| `FullScreen` | 검색 화면 상단 |

#### 규칙

- 지도 상단 검색바는 높이 `56dp`, 좌우 패딩 `16dp`
- trailing voice icon은 기능이 들어가기 전까지 숨기거나 disabled 처리한다. 빈 아이콘 자리만 남기지 않는다.
- 검색어가 선택된 뒤에는 placeholder가 아니라 선택 장소명을 보여준다.

### 6.5 Chip

코드 이름: `EumChip`

| Type | 용도 |
| --- | --- |
| `Filter` | 지도 시설 필터 |
| `Choice` | 경로 옵션 |
| `Status` | 위험/처리 상태 |

#### 규칙

- 필터 칩은 수평 스크롤을 허용한다.

### 6.6 SegmentedControl

코드 이름: `EumSegmentedControl`

- 용도: 경로 옵션(`안전한 길`, `최단거리`)과 이동수단 탭(`도보`, `대중교통`)
- 경로 옵션은 2개 카드 세로 나열을 기준으로 한다.
- 선택 상태는 indicator + weight + color 조합으로 표현

### 6.7 Card

코드 이름: `EumCard`

| Variant | 용도 |
| --- | --- |
| `Static` | 정보 요약 |
| `Interactive` | 탭 가능 카드 |
| `Highlighted` | 위험/추천 요약 |

#### 규칙

- 카드 배경은 항상 surface다.
- 강조는 배경 채움보다 좌측 bar, badge, title weight로 준다.
- 지형 위험/접근성 정보는 카드 안에서 label 형태로 보여 준다.

### 6.8 ListItem

코드 이름: `EumListItem`

| Variant | 용도 |
| --- | --- |
| `Navigation` | 설정, 마이페이지 메뉴 |
| `Action` | 북마크 장소/경로 아이템 |
| `Status` | 제보 상태, 최근 검색 |

#### 규칙

- trailing action 정렬을 통일한다.
- 제목 2줄, 보조 텍스트 2줄까지 허용한다.
- icon-only trailing action이면 터치 영역을 따로 확보한다.

### 6.9 Empty / Error / Loading State

코드 이름: `EumEmptyState`, `EumErrorState`, `EumLoadingState`

#### 규칙

- 검색 결과 없음, 북마크 없음, 위치 권한 없음, API 실패를 각기 다른 텍스트로 구분한다.
- 지도 화면의 loading은 전체 화면 spinner보다 skeleton/overlay 우선이다.
- `ErrorState`에는 반드시 재시도 액션 또는 대체 흐름을 준다.

## 7. 지도형 서비스 전용 컴포넌트 규칙

### 7.1 `MapTopSearchBar`

- 지도 최상단 고정
- leading: 검색 아이콘
- trailing: 음성 검색 또는 clear
- 높이 `56dp`
- 열릴 때 지도 컨트롤과 겹치지 않게 safe inset 고려

### 7.2 `MapFilterChipBar`

- 지도 검색바 아래
- 수평 스크롤
- 기본 카테고리: 음식점, 관광지, 화장실, 엘리베이터, 충전소, 점자블록
- selected 상태는 색상 + 체크/아이콘 변화 동시 적용

### 7.3 `PlaceDetailSheet`

- 마커 선택 시 뜨는 기본 시트
- 포함 정보:
  - 장소명
  - 카테고리
  - 주소
  - 접근성 태그
  - 운영시간
  - 북마크 버튼
  - 길안내 시작 버튼
- 접근성 태그는 `경사로`, `자동문`, `장애인 화장실`, `엘리베이터`처럼 도착지 접근 가능 정보 중심으로 배치한다.

### 7.4 `RouteOptionCard`

- 경로 탐색 결과 리스트에서 사용
- 필수 정보:
  - 경로 성격 라벨
  - 총 거리
  - 예상 시간
  - 위험 요약
  - 포함 편의시설
- "최단거리"보다 "안전 우선"이 먼저 보이게 한다.

### 7.5 `RouteSummaryCard`

- 내비게이션 시작 직전/진행 중 요약
- 필수 정보:
  - 다음 안내
  - 남은 거리
  - ETA
  - 위험 알림
- 내비 진행 화면에서는 정보 위계를 하나만 분명하게 둔다. 동시에 너무 많은 값을 띄우지 않는다.

### 7.6 `HazardReportForm`

- 제보 유형
- 위치 확인
- 사진 첨부
- 설명 입력
- 제출 CTA

#### 규칙

- 제보 유형은 radio/dropdown이 아니라 명확한 선택형 chip 또는 list를 우선한다.
- 사진 첨부는 선택 사항이지만 입력 영역 존재는 명확해야 한다.
- 위치 수정은 지도 drag 하나에만 의존하지 않는다. 주소 텍스트와 현재 위치 재설정 액션을 같이 준다.

### 7.7 `BookmarkPlaceItem`

- 장소명
- 주소
- 접근성 태그 또는 카테고리
- 길안내 버튼
- 삭제/해제 액션

### 7.8 `BookmarkRouteItem`

- 출발지
- 도착지
- 북마크 라벨
- 마지막 사용 시간 또는 옵션
- 바로 길안내 액션. 북마크 경로는 `NAV-01`로 즉시 진입한다.

### 7.9 `PermissionGuideCard`

- 위치 권한
- 마이크 권한: STT 기반 장소 검색 사용 시점에 요청
- 알림 권한: MVP 제외 또는 후순위
- 데이터 수집 동의

#### 규칙

- 권한 설명은 "왜 필요한지"와 "허용하지 않았을 때 제한되는 기능"을 같이 보여 준다.
- 권한 안내 화면에서 카드 선택 시 요청할지 하단 CTA에서 일괄 요청할지는 미정으로 둔다.
- 위치 권한 다이얼로그는 `현재 위치 표시`, `경로 안내`, `제보 위치 자동 입력` 제한을 명시한다.

### 7.10 `LocationPermissionEmptyState`

- 위치 권한 미허용 시 지도 위 빈 상태 또는 전체 상태 화면으로 사용
- 구성:
  - 제목
  - 제한 기능 설명
  - 권한 허용 버튼
  - 나중에 하기 또는 검색으로 계속

## 8. 화면 전용 일회성 컴포넌트와 공통 컴포넌트의 분리 기준

### 지금 공통으로 올리는 기준

- 2개 이상 feature에서 재사용된다.
- size / state / variant가 명확하다.
- 디자인 변경이 들어와도 screen 로직을 안 건드리고 교체할 수 있다.

### feature 내부에 남기는 기준

- 특정 화면 플로우에 강하게 묶여 있다.
- 문구와 상태가 그 화면에서만 의미가 있다.
- 추상화 비용이 재사용 이익보다 크다.

#### 공통으로 올리지 않는 예시

- 온보딩 사용자 유형 선택 카드
- 온보딩 권한 안내 카드
- 온보딩 서비스 소개 카드
- 약관 화면 전용 동의 묶음 섹션
- 시각 지원 확장 대화형 응답 카드

#### 공통으로 올리는 예시

- `EumButton`
- `EumTextField`
- `MapTopSearchBar`
- `PlaceDetailSheet`
- `RouteOptionCard`

## 11. 금지 패턴

1. 지도 화면에서 모든 액션을 FAB로만 몰아넣는 것
2. 시설 상세를 full screen page로 밀어 넣는 것
3. 선택 상태를 색상 하나로만 표현하는 것
4. icon-only 버튼에 라벨을 주지 않는 것
5. 버튼/칩/radius 값을 화면마다 제각각 쓰는 것
6. 검색바와 필터칩을 feature마다 따로 만드는 것
7. 하나의 거대한 `CommonMapSheet`에 장소 상세, 경로 요약, 권한 상태를 다 우겨 넣는 것
8. Figma 컴포넌트와 코드 컴포넌트 이름이 다르게 가는 것
9. raw hex / raw `dp` / raw `sp`를 screen에서 직접 쓰는 것
10. marker 위치만으로 정보를 전달하고 텍스트 대안을 주지 않는 것

## 13. 피그마 병행 작업을 위한 핸드오프 규칙

1. 새 화면보다 토큰과 primitive를 먼저 맞춘다.
2. 디자이너는 새 컴포넌트를 만들 때 `Type / Size / State / Mode` variant를 같이 정의한다.
3. 개발자는 새 컴포넌트를 구현할 때 Figma 이름과 동일한 코드 이름을 사용한다.
4. Figma에서 screen-only인지 shared인지 컴포넌트 설명에 표시한다.
5. 접근성 라벨이 필요한 icon-only 요소는 Figma 주석에 `a11y-label`을 같이 적는다.
6. 디자인에서 색상 변경 요청이 오면 screen CSS식 수정이 아니라 token 변경으로 반영한다.
7. map 전용 시트와 route card는 피그마에서 먼저 상태 표를 만들고 구현한다. 화면 캡처만 보고 개발하지 않는다.
