## 1. 문서 사용법

- 본 문서는 **시연 순서 기준**으로 박세홍 영역 박세홍 작업을 빠르게 한 번에 회귀 검증하기 위한 마스터 체크리스트다.
- 각 시나리오는 **개별 QA 체크리스트(선행 문서)에서 정제된 핵심 항목만 가져온 것**이며, 세부 자동 테스트 매핑·환경 이슈는 선행 문서를 참조한다.
- §3 시연 순서는 발표자 동선을 기준으로 묶었고, 시연 도중 마주칠 가능성이 가장 높은 순서로 정렬되어 있다.
- 각 항목의 **`출처`** 컬럼에 해당 변경을 도입한 Task 번호가 명시되어 있으니, 회귀 실패 시 그 Task의 PR/문서로 빠르게 추적 가능하다.

## 4. 시연 시 막힐 수 있는 알려진 한계 (미리 인지)

박세홍 영역 외이거나 후속 Task로 분리된 항목. 시연 도중 마주치면 당황하지 않도록 사전 인지.

| 항목 | 사유 | 회피·완화 |
| --- | --- | --- |
| **ARR-02 도착 화면 도달 불가** | navigation 도메인 `NavigationCompleteClicked → NavigateToArrival` UI 미연결 (이재호 영역). 결과적으로 **경로 북마크 자체 저장 흐름 시연 불가** | 흐름 C는 서버에 미리 저장된 데이터로만 시연 |
| **제보 사진 업로드 환경 확인 필요** | FE는 presigned upload 후 `imageObjectKeys`를 제출한다. 운영 S3/MinIO, CORS, URL 만료 설정이 맞지 않으면 사진 업로드가 실패할 수 있다. | 시연 전 presigned upload와 조회 이미지 로딩을 같은 계정으로 확인 |
| **마이페이지 제보 내역 서버 응답 계약 확인** | FE는 `GET /hazard-reports/me`의 `status`, `representativeImageUrl`을 읽어 처리 상태와 대표 사진을 표시한다. 운영 서버 응답 누락 시 표시가 제한될 수 있다. | E-3 시연 전 상태 badge와 대표 사진 회귀 확인 |
| **임시저장 후 탭 전환 시 화면 잔존** | navigation 도메인 영역. [인계 노트 §9](../debug-report/2026-05-07_빌드오류 및 작업 현황_수정필요.md) | 시연에서 임시저장 후 탭 전환 회피 |
| **임시저장 사진 복원 회귀** | 사진 outbox/Room migration 반영 후 기존 데이터와 신규 데이터 모두 최대 5장 복원이 유지되는지 확인 필요 | 시연 전 2장 이상 첨부 후 탭 전환/앱 재시작 확인 |
| **장소 북마크 API 신규 명세 미반영** | `S14P31E102-557/558`(2026-05-09) 페이징 cursor 전환, `bookmarkTargetId`/`accessibilityFeatures` 등 신규 필드 미반영. 시연 중 호환 가능성은 있으나 응답 일부 누락될 수 있음 | 시연 후 별도 hotfix Task 등록 권장 |
| **재시도 시 BE 중복 등록 가능성** | 멱등성 키 미정의. 네트워크 race 시 같은 제보 중복 등록 가능 | 시연 시 빠른 retry 연속 탭 회피 |
| **`Submitting` 상태 outbox 자동 cleanup 없음** | 앱이 제출 중 강제 종료되면 status=Submitting 영구 잔존 | 시연 중 앱 강제 종료 회피 |
| **401 자동 재로그인 흐름 없음** | 토큰 만료 시 메시지만 노출, 자동 navigate 없음 | 시연 직전 로그인 상태 확인 |
