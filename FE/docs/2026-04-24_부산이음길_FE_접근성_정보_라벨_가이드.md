# 부산이음길 FE 접근성 정보 라벨 가이드

- 작성일: 2026-04-24
- 대상: 검색 결과 카드, 시설 상세 바텀시트, 북마크 장소 카드, 경로 요약 내 시설 정보 영역
- 목적: FE에서 접근성 정보를 칩/라벨로 표기할 때 사용할 `고정 문구`, `표기 순서`, `노출 규칙`을 하나의 기준으로 관리한다.

## 1. 기본 원칙

- 접근성 정보는 아이콘만으로 전달하지 않고 `아이콘 + 라벨 텍스트`를 함께 사용한다.
- 동일 의미를 다른 문구로 바꾸지 않는다. 예를 들어 `엘리베이터 가능`, `엘리베이터 O`처럼 축약하거나 변형하지 않는다.
- 이번 목록은 `사용 가능 / 제공됨` 성격의 접근성 정보 라벨 기준이다.
- 주의/제한/위험 라벨은 별도 시맨틱 체계로 관리하되, 같은 행에 함께 노출될 수 있다.
- 2026-04-27 온보딩 디자인의 `보행약자`, `휠체어 사용자`, `기타 보행약자`, `무장애 경로 안내`, `음성 안내 (TTS)` 같은 사용자 유형/기능 소개 문구는 이 고정 라벨 목록의 적용 대상이 아니다. 온보딩 문구는 `FE/docs/archive/2026-04-27_온보딩_디자인_화면_구성_분석.md`를 우선한다.

## 2. 고정 라벨 목록

| 우선순위 | 라벨 | 의미 |
| --- | --- | --- |
| 1 | `출입 가능` | 출입구 기준으로 접근 가능한 상태 |
| 2 | `단차 없음` | 주요 출입/이동 구간에 단차가 없는 상태 |
| 3 | `경사로` | 휠체어/보행보조 기반 진입에 필요한 경사로 제공 |
| 4 | `엘리베이터` | 층간 이동을 위한 엘리베이터 제공 |
| 5 | `장애인 주차` | 장애인 전용 또는 접근 가능한 주차 공간 제공 |
| 6 | `장애인 화장실` | 접근 가능한 화장실 제공 |
| 7 | `안내시설` | 점자/안내판/유도 설비 등 안내 보조 시설 제공 |
| 8 | `장애인 객실` | 숙박/객실형 시설에서 접근 가능한 객실 제공 |

## 3. 표기 규칙

### 3.1 라벨 문구

- 라벨은 위 표의 문구를 그대로 사용한다.
- FE fixture, mock, seed, preview 데이터도 동일 문구를 사용한다.
- 한 화면 안에서 같은 정보를 `출입 가능`과 `접근 가능`처럼 혼용하지 않는다.

### 3.2 기본 노출 규칙

- 기본 노출은 `+n` overflow 칩으로 접지 않고 가능한 태그를 직접 표시한다.
- 검색 결과와 최근 목적지처럼 폭이 좁은 영역에서는 접근성 태그를 두 줄까지 자연스럽게 감싸서 보여준다.

### 3.3 정렬 규칙

- 여러 라벨이 함께 있을 때는 위 표의 우선순위를 따른다.
- 즉, 화면에 먼저 보여줄 후보는 `출입 가능 -> 단차 없음 -> 경사로 -> 엘리베이터` 순서로 우선한다.
- `장애인 주차`, `장애인 화장실`, `안내시설`, `장애인 객실`은 보조 편의 정보로 뒤에 배치한다.

## 4. 시각 규칙

| 항목 | 기준 |
| --- | --- |
| 표현 방식 | `아이콘 + 라벨 텍스트 + 시맨틱 칩` |
| 색상군 | 이번 목록은 기본적으로 `긍정 접근성 정보`이므로 블루 계열 칩을 사용한다. |
| 텍스트 | 축약형보다 전체 문구를 우선한다. |
| 경계 표현 | 칩 경계는 1px 내외의 선명한 보더 또는 충분한 톤 차이로 분리한다. |
| 아이콘 크기 | 칩 내부 보조 아이콘 기준 `16~20dp` 권장 |

## 5. 적용 화면

| 화면/영역 | 적용 방식 |
| --- | --- |
| 검색 결과 카드 | 장소명/주소 아래 접근성 라벨 칩 행으로 노출 |
| 지도 시설 상세 바텀시트 | 주소/메타 아래 접근성 정보 영역으로 노출 |
| 북마크 장소 카드 | 장소명/주소 아래 동일 라벨 체계 재사용 |
| 경로 요약/목적지 확인 | 필요한 경우 요약형으로 일부 라벨만 노출 |

## 6. 예시 조합

### 6.1 단일 행에 들어가는 경우

- `출입 가능`
- `출입 가능`, `단차 없음`
- `출입 가능`, `단차 없음`, `경사로`

### 6.2 두 줄로 감싸는 경우

- `출입 가능`, `단차 없음`, `경사로`, `엘리베이터`, `장애인 화장실`
- `출입 가능`, `엘리베이터`, `장애인 화장실`, `안내시설`

## 7. 구현 메모

- FE enum 또는 sealed model로 관리할 경우에도 사용자 노출 라벨은 이 문서의 표기명을 우선한다.
- TalkBack/스크린리더에서는 칩 단위 반복 낭독보다 카드 단위 병합 낭독을 우선 검토한다.
- 주의/제한 라벨과 함께 노출될 때도 접근성 정보 라벨의 고정 문구는 바꾸지 않는다.
