## 1. 문서 개요

본 문서는 신규 디자인 시안 `부산이음길 온보딩 / 사용자 유형 선택`과 온보딩 이후 `지도 홈 메인화면`을 기준으로 화면 구성, 정보 구조, 주요 컴포넌트, 화면 전환 흐름을 분석한 문서이다.

추후 기존 화면명세서를 디자인 기준으로 개정하거나 1차 FE 계약 문서를 갱신할 때, 온보딩 영역의 화면 ID, 컴포넌트, 문구, 액션 정의를 보정하기 위한 참고 자료로 사용한다.

### 1.1 분석 범위

| 구분 | 내용 |
| --- | --- |
| 대상 화면 | 온보딩 5단계 화면, 지도 홈 메인화면, 목적지 설정 4단계 화면, 경로 탐색/상세/길안내 3단계 화면, 도착 안내/경로 평가 화면, 북마크 화면, 장애물 제보 화면, 마이페이지 화면 |
| 플랫폼 | Android 모바일 앱 |
| 주요 사용자 | 저시력자, 보행약자 |
| 주요 목적 | 사용자 유형 1차 선택, 저시력자 전용 후속 화면, 보행약자 2차 선택, 약관/동의, OS 위치 권한 요청, 튜토리얼, 지도 기반 목적지 탐색, 목적지 검색/선택, 무장애 경로 비교와 길안내, 도착 후 경로 평가, 관심 장소/경로 관리, 현장 장애물 제보, 사용자 계정/설정 관리 |
| 디자인 톤 | 공공 서비스형 신뢰감, 접근성 중심, 파란색 기반의 명확한 CTA |

### 1.2 디자인에서 확인한 전체 흐름

```text
사용자 유형 1차 선택
  -> 저시력자 전용 후속 화면 또는 보행약자 2차 선택
  -> 약관/동의
  -> OS 위치 권한 시스템 팝업
  -> 튜토리얼
  -> 지도 홈 메인화면
  -> 목적지 설정
  -> 경로 탐색 결과
  -> 길안내
  -> 도착 안내 / 경로 평가
  -> 하단 탭 분기: 북마크 / 제보 / 마이페이지
```

### 1.3 로그인 필수 정책 반영 메모

2026-04-27 팀 회의에서 `로그인 필수`가 확정되었다. 현재 온보딩 디자인 시안에는 로그인 화면이 포함되어 있지 않으므로, 본 문서는 디자인에서 관찰 가능한 온보딩 5단계 구조를 유지하되 앱 전체 진입 기준에는 인증 gate가 추가된 것으로 본다.

```text
앱 실행
  -> 인증 세션 확인
  -> 로그인 여부 확인
  -> 온보딩 5단계 진행 또는 재개
  -> 메인 화면 진입
```

로그인 화면을 온보딩 5단계 앞에 둘지, 뒤에 둘지는 추가 디자인/기획 확인이 필요하다. FE 구현 영향은 `FE/docs/debug-report/2026-04-27_로그인_필수_전환_FE_정합성_및_구현_영향.md`를 참고하되, 최종 계약 변경은 1차 FE 계약 문서에 반영해 관리한다.

## 2. 공통 화면 구조

온보딩 화면은 모든 단계에서 동일한 모바일 프레임과 하단 고정 CTA 패턴을 사용한다. 사용자는 한 화면에서 하나의 주요 의사결정만 수행하도록 설계되어 있다.

| 영역 | 구성 | 설명 |
| --- | --- | --- |
| 상태바 | 시간, 네트워크, 배터리 | Android 기본 상태바 영역 |
| 진행 표시 | 상단 파란색 progress bar | 현재 온보딩 진행 상태를 시각적으로 안내 |
| 본문 영역 | 제목, 설명, 카드 또는 안내 콘텐츠 | 단계별 핵심 정보와 선택지를 배치 |
| 하단 CTA | Primary button | 다음 단계 이동 또는 시작 액션 제공 |
| 시스템 내비게이션 | Android navigation bar | 기기 기본 하단 내비게이션 |

### 2.1 공통 UI 규칙

| 항목 | 디자인 기준 |
| --- | --- |
| 기본 배경 | White |
| 주요 색상 | Blue 계열 Primary |
| CTA 형태 | 하단 고정, 전체 너비에 가까운 파란색 버튼 |
| 카드 형태 | 흰색 카드, 옅은 회색 border, 약한 radius |
| 선택 상태 | 진한 남색 또는 파란색 강조 배경 |
| 아이콘 | 서비스 정체성과 접근성 기능을 직관적으로 보여주는 라인/면 아이콘 |
| 텍스트 정렬 | 대부분 중앙 정렬, 카드 내부는 좌측 정보 정렬 |
| 정보 밀도 | 한 화면에 1개 핵심 목적만 배치 |

### 2.2 지도 홈 화면 구조 규칙

지도 홈은 온보딩과 달리 지도 전체 화면 위에 검색, 필터, 지도 컨트롤, 하단 정보 패널을 오버레이하는 구조이다. 핵심 행동은 목적지 검색과 최근 목적지 기반 길찾기이며, 하단 탭으로 주요 기능을 전환한다.

| 영역 | 구성 | 설명 |
| --- | --- | --- |
| 지도 영역 | 지도 타일, 장소/교통/주차 마커, 현재 위치 | 화면 대부분을 차지하며 탐색의 기본 배경이 된다. |
| 상단 검색 | 검색 입력 바, 음성 입력 아이콘 | 목적지 검색 또는 음성 검색 진입점이다. |
| 접근성 필터 | 장애인 화장실, 엘리베이터, 장애인 주차장, 더보기 | 지도 마커와 접근성 정보 노출 조건을 빠르게 조정한다. |
| 지도 컨트롤 | 확대/축소, 현재 위치 재중심 | 지도 조작을 우측 플로팅 버튼으로 제공한다. |
| 하단 패널 | 날씨/미세먼지, 시간대별 날씨, 최근 목적지 | 이동 전 필요한 환경 정보와 재방문 목적지를 제공한다. |
| 하단 탭 | 홈, 북마크, 제보, 마이페이지 | 메인 기능 간 이동을 담당한다. |

---

### 2.3 목적지 설정 화면 구조 규칙

목적지 설정은 지도 홈의 검색바에서 진입하며, 검색/최근 방문지, 검색 결과 목록, 지도 선택 바텀시트, 첫 사용 시 사용자 상태 설문으로 이어지는 흐름이다. 디자인상 목적지 설정 하위 화면에서도 하단 탭이 유지된다.

| 영역 | 구성 | 설명 |
| --- | --- | --- |
| 상단 앱바 | 뒤로가기, 화면 제목 또는 검색 입력 | 목적지 설정 흐름에서 이전 화면 복귀와 검색어 수정 제공 |
| 검색 입력 | 검색어, clear, 마이크 | 텍스트 검색과 음성 입력 진입점 |
| 목록 영역 | 최근 방문지 또는 검색 결과 카드 | 장소명, 주소, 거리, 접근성 라벨을 제공 |
| 지도 선택 | 지도, 선택 마커, 장소 바텀시트 | 지도에서 직접 선택한 목적지의 상세 정보를 제공 |
| 사용자 상태 설문 | 이동 특성 선택 카드 | 첫 사용 시 경로 추천 기준을 세분화하기 위한 선택 |
| 하단 탭 | 홈, 북마크, 제보, 마이페이지 | 목적지 설정 중에도 주요 탭 이동 유지 |

---

### 2.4 경로 탐색 및 길안내 화면 구조 규칙

경로 탐색 결과는 목적지 설정 완료 후 출발지/도착지를 확인하고, 여러 경로 옵션을 비교한 뒤 길안내를 시작하는 흐름이다. 길안내 시작 후에는 하단 탭을 숨기고 지도와 현재 안내 카드, 종료 CTA에 집중한다.

| 영역 | 구성 | 설명 |
| --- | --- | --- |
| 상단 앱바 | 뒤로가기, 화면 제목 | 경로 지도/상세/안내 시작 단계의 현재 위치를 명확히 표시 |
| 출발/도착 정보 | 출발지, 도착지, 주소, 전환 아이콘 | 경로 탐색 기준 정보를 확인하고 필요 시 출발/도착을 전환 |
| 이동수단 탭 | 도보, 대중교통 | 경로 탐색 방식을 전환한다. 대중교통 탭은 MVP 범위에 포함한다. |
| 지도 경로 | 지도, 경로선, 출발/도착 마커 | 선택 경로의 공간적 흐름과 현재 위치를 표시 |
| 경로 옵션 카드 | 안전한 길, 최단거리 | 소요 시간, 거리, 접근성/주의 라벨을 비교하며 바텀시트에서는 카드 2개를 세로로 나열한다. |
| 경로 상세 목록 | 단계별 이동 지시, 주의 구간, 접근성 시설 | 경로 내 상세 이동 정보와 위험 요소를 시간 단위로 제공한다. |
| 길안내 진행 | 현재 안내 카드, 지도, 남은 시간/거리, 상세 정보, 종료 버튼 | 실제 안내 중 필요한 정보만 크게 보여준다. |

---

### 2.5 도착 안내 및 경로 평가 화면 구조 규칙

도착 안내는 길안내가 완료된 후 목적지 도착을 명확히 알리고, 홈 복귀 또는 새 경로 탐색을 제공하는 종료 화면이다. 경로 평가는 도착 안내 위에 dim overlay와 바텀시트로 표시되며, 별점 평가와 경로 저장을 함께 제공한다.

| 영역 | 구성 | 설명 |
| --- | --- | --- |
| 브랜드 영역 | 부산이음길 로고, 부산 일러스트 | 이동 완료 후 서비스 정체성과 긍정적 종료감을 제공 |
| 완료 메시지 | 완료 문구, 보조 설명 | 안전한 이동 완료를 안내 |
| 후속 CTA | 홈으로 돌아가기, 새 경로 탐색하기 | 완료 후 사용자의 다음 행동을 제공 |
| 평가 바텀시트 | 별점, 만족도 라벨, 경로 저장, 평가 등록 | 경로 품질 피드백과 저장 액션을 수집 |
| dim overlay | 배경 흐림/어둡게 처리 | 평가 바텀시트에 사용자의 주의를 집중 |

### 2.6 북마크 화면 구조 규칙

북마크는 하단 탭의 독립 메인 화면이며, 북마크한 장소와 북마크한 경로를 segmented control로 전환해 관리한다. 각 항목은 접근성 라벨과 `길찾기` CTA를 함께 제공한다.

| 영역 | 구성 | 설명 |
| --- | --- | --- |
| 상단 앱바 | 뒤로가기, 화면 제목, 편집 | 북마크 화면 복귀/관리 진입 |
| 탭 전환 | 장소, 경로 | 북마크한 장소와 경로 목록 전환 |
| 장소 카드 | 장소 아이콘, 장소명, 주소, 접근성 라벨, 길찾기 | 북마크 장소를 확인하고 카드 본문 선택 시 상세로 이동 |
| 경로 카드 | 출발/도착 라인 아이콘, 경로명, 거리/예상 시간, 경로 특성 라벨, 길찾기 | 북마크 경로를 확인하고 카드 본문 선택 시 상세로 이동 |
| 하단 탭 | 홈, 북마크, 제보, 마이페이지 | 북마크 탭 활성 상태 |

---

### 2.7 장애물 제보 화면 구조 규칙

장애물 제보는 하단 탭의 `제보`에서 시작하는 단계형 입력 플로우이다. 유형 선택, 위치 확인, 상세 정보 입력, 완료 확인으로 구성되며, 제보 중에도 하단 탭은 유지되고 `제보` 탭이 활성화된다.

| 영역 | 구성 | 설명 |
| --- | --- | --- |
| 상단 앱바 | 뒤로가기, 단계별 화면 제목 | 제보 단계의 현재 위치와 이전 단계 복귀 제공 |
| 유형 선택 그리드 | 장애물 유형 카드 | 현장 문제 유형을 빠르게 선택 |
| 위치 확인 지도 | 지도, 현재 위치, 위치 바텀카드 | 제보 위치를 확인하거나 지도 이동으로 조정 |
| 상세 입력 | 설명 textarea, 사진 첨부 | 문제 상황을 텍스트와 이미지로 보강 |
| 완료 확인 | 성공 아이콘, 제보 요약 카드 | 제출 완료와 제보 내역 확인 진입을 안내한다. 처리 상태는 노출하지 않는다. |
| 하단 탭 | 홈, 북마크, 제보, 마이페이지 | 제보 탭 활성 상태 |

---

### 2.8 마이페이지 화면 구조 규칙

마이페이지는 로그인 필수 정책 이후 사용자 계정, 개인 설정, 제보 이력, 앱 정보/도움말을 관리하는 하단 탭 허브이다. 디자인 기준으로는 프로필 카드, 활동 배지, 메뉴 리스트, 계정 액션을 한 화면에서 제공한다. 알림 설정은 MVP 제외 또는 후순위로 둔다.

| 영역 | 구성 | 설명 |
| --- | --- | --- |
| 상단 앱바 | 화면 제목 또는 뒤로가기 + 화면 제목 | 마이페이지 홈과 하위 화면의 현재 위치 제공 |
| 프로필 카드 | 아바타, 사용자명, 사용자 유형, 사용자 유형 변경 | 로그인 사용자와 이동 특성 정보를 요약하고 변경 진입 제공 |
| 활동 배지 | 제보왕, 도움왕, 참여왕 | 사용자 참여/기여 지표를 시각적으로 표시 |

## 4. 화면명세서 반영 필요 사항

기존 화면명세서의 온보딩은 사용자 유형 선택, 권한 동의 등 기능 중심으로 정의되어 있으나, 신규 디자인 기준으로 다음 내용을 보강하는 것이 적절하다. 보행약자 공통 메인 홈은 `MAP-01`, 저시력자 전용 메인 홈은 `LV-HOM-01`로 분리해 반영한다.

| 구분 | 반영 내용 |
| --- | --- |
| 화면 수 | 온보딩을 5단계로 재정의한다. |
| 화면 ID | `ONB-001`부터 `ONB-005`까지 별도 ID 체계를 부여한다. |
| 사용자 유형 1차 선택 | 로그인 성공 후 최초 화면으로 `ONB-001`을 배치하고 `저시력자 / 보행약자` 분기를 정의한다. |
| 사용자 유형 | `저시력자 / 보행약자` 1차 선택과 `전동휠체어 사용자 / 수동휠체어 사용자 / 기타 보행약자` 2차 선택의 상태와 저장 동작을 정의한다. |
| OS 위치 권한 요청 | 약관/동의 화면의 `다음` 이후 시스템 팝업으로 위치 권한을 요청한다고 정의한다. |
| 약관 동의 | 필수 약관 동의 체크 여부와 노션 웹뷰 상세 진입에 따른 CTA 활성/비활성 상태를 정의한다. |
| 튜토리얼 | 위치 권한 응답 이후 노출되는 튜토리얼 단계의 카드/페이지 전환 동작을 명세한다. |
| 완료 처리 | 마지막 온보딩 완료 시 온보딩 완료 플래그를 저장하고, `저시력자`는 `LV-HOM-01`, `보행약자`는 `MAP-01`로 이동한다. |
| 지도 홈 | 보행약자 공통 홈 `MAP-01`은 검색바, 접근성 필터, 현재 위치, 지도 컨트롤, 최근 목적지, 하단 탭을 정의한다. |
| 저시력자 전용 홈 | `LV-HOM-01`은 `음성 입력`, `현재 위치`, 하단 `홈/북마크/카테고리/마이페이지`를 중심으로 정의한다. |
| 메인 탭 | 보행약자 공통 화면은 `홈`, `북마크`, `제보`, `마이페이지` 4탭을 사용하고, 저시력자 전용 UI는 별도 `LV-*` 화면군의 `홈`, `북마크`, `카테고리`, `마이페이지`를 사용한다. |
| 목적지 설정 | `DEST-01`~`DEST-04`로 검색, 검색 결과, 지도 선택, 첫 사용 사용자 상태 설문을 분리한다. |
| 검색 결과 | 보행약자 공통은 `DEST-02`, 저시력자 전용은 `LV-RES-01`로 분리하고, 저시력자 전용 결과는 2카드 시작 패턴과 `북마크/길찾기` CTA를 정의한다. |
| 지도 선택 | 선택 장소 바텀시트와 `도착지로 설정` CTA를 정의한다. |
| 사용자 상태 설문 | `전동휠체어 사용자`, `수동휠체어 사용자`, `기타 보행약자`는 보행약자 하위 이동 특성으로 매핑한다. 노출 조건은 추가 확인 필요로 둔다. |

## 7. 확정 반영 및 추가 확인 필요 사항

| 항목 | 최신 기준 |
| --- | --- |
| 사용자 유형 명칭 | 온보딩 1차 유형은 `저시력자 / 보행약자`, 보행약자 2차 유형은 `전동휠체어 사용자 / 수동휠체어 사용자 / 기타 보행약자`로 고정 |
| OS 위치 권한 요청 시점 | `ONB-004` 약관/동의 화면의 `다음` 버튼 이후 시스템 팝업으로 요청 |
| 마이크 권한 | 필수 아님. 온보딩에서 요청하지 않고 STT 기반 장소 검색 사용 시점에 요청 |
| 약관 상세 방식 | 약관 전문은 노션 웹뷰로 표시 |
| 튜토리얼 페이지 수 | 기존 서비스 소개 슬라이드는 튜토리얼 단계로 재배치한다. 페이지 수는 UI 기준 유지 가능 |
| 온보딩 재진입 | 마이페이지/설정에서 사용자 유형 변경이 필요하면 `ONB-001`부터 다시 진입 |
| 하단 탭 구조 | 기본 기준은 `홈/북마크/제보/마이페이지` 4탭이며, 저시력자 전용 UI에서는 `홈/북마크/카테고리/마이페이지` 예외 구성을 허용 |
| 더보기 필터 | 포함 항목 미정 |
| 음성 검색 | 지도 홈 검색바의 마이크 아이콘은 STT 기반 장소 검색 버튼 |
| 최근 목적지 전체보기 | 별도 전체 목록 화면 없이 북마크 화면으로 연결 |
| 날씨/미세먼지 상세 | 미세먼지 영역 chevron은 날씨 외부 정보 패널로 이동 |
| 목적지 설정 화면 ID | `DEST-01`~`DEST-04`를 별도 화면 ID로 유지 |
| 검색 결과 카드 선택 후 흐름 | 항상 `DEST-03` 지도 바텀시트 확인을 거침 |
| 사용자 상태 설문 노출 조건 | 정책 설명이 불완전하므로 추가 확인 필요 |
| 사용자 상태 용어 | 1차 유형은 `저시력자 / 보행약자`, 보행약자 세부 이동 특성은 `전동휠체어 사용자 / 수동휠체어 사용자 / 기타 보행약자` |

| 경로 옵션 | 기존 3번째 옵션은 삭제. `안전한 길`, `최단거리` 2개 카드 세로 나열 |

### 8.3 화면명세서 개정 최종 기준

| 기준 | 내용 |
| --- | --- |
| 온보딩 화면 수 | 5개 |
| 온보딩 ID | `ONB-001`~`ONB-005` |
| 화면 순서 | 사용자 유형 1차 선택 -> 저시력자 전용 후속 화면 또는 보행약자 2차 선택 -> 약관/동의 -> OS 위치 권한 시스템 팝업 -> 튜토리얼 -> 사용자 유형별 메인 홈 |
| 사용자 유형 | 1차 유형 `저시력자 / 보행약자`, 2차 유형 `전동휠체어 사용자 / 수동휠체어 사용자 / 기타 보행약자` |
| 권한 | OS 위치 권한은 `ONB-004` 다음 단계의 시스템 팝업으로 요청하고, 마이크 권한은 STT 사용 시점에 요청한다. |
| 약관 | `전체 동의`, `[필수] 서비스 이용약관 및 위치기반서비스 이용약관 동의`, `[필수] 민감정보 수집·이용 동의`, `[필수] 개인위치정보 수집·이용 동의`, `[필수] 만 14세 이상입니다`, `[확인] 개인정보 처리방침 확인` |
| 완료 조건 | 인증 완료, 1차 사용자 유형 선택, 분기 후속 단계 완료, 필수 약관 동의 완료, OS 위치 권한 응답 완료, 튜토리얼 완료 |
