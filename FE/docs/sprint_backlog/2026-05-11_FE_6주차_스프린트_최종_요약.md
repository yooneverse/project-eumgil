## 1. 스프린트 목적

6주차 FE 작업은 새 화면 확장보다 5주차 carry-over를 줄이고, 시연 가능한 흐름을 안정화하는 데 초점을 둔다. 핵심 축은 인증 cold-start, 마이페이지 계정 액션, 경로 재진입, 수동 QA, 일반모드 UI 정합성이다.

## 2. 핵심 정리

| 영역 | 최종 정리 |
| --- | --- |
| 인증/온보딩 | `/users/me` 재동기화와 start destination 보강을 최우선 리스크로 분류했다. |
| 마이페이지 | 외부 링크 fallback, 약관 URL 상수 정리, 로그아웃/회원탈퇴 회귀 QA를 주요 잔여 작업으로 좁혔다. |
| 경로/길안내 | 경로 북마크 상세 조회, route snapshot 재진입, `ROUTE -> NAV -> ARR -> BMK` 회귀 QA를 중심으로 재정리했다. |
| 지도/검색 | 신규 API 확장보다 현재 지도, 검색, 목적지 handoff, marker tap, empty/error/offline 상태의 수동 QA를 우선했다. |
| 공통 UI | 일반모드 radius, shadow, icon 잔여 정합성과 접근성 회귀 기준을 마감 범위로 정리했다. |

## 3. Jira 정리 방향

| 구분 | 처리 방향 |
| --- | --- |
| 기능이 이미 닫힌 검색/목적지 항목 | 종료 또는 수동 QA 항목으로 축소 |
| 범위가 넓은 경로/도착 Story | `경로 북마크 재진입 보강 및 도착 플로우 회귀 QA`로 제목 조정 |
| 일반모드 UI 항목 | radius, shadow, icon 잔여 정합성 작업으로 통합 |
| 문서성 QA 항목 | 수동 QA 체크리스트와 접근성 회귀 기준으로 재정리 |
| placeholder성 예비 작업 | 종료 또는 삭제 후보 |

## 4. 남긴 확인 사항

- 경로 북마크 상세 조회 응답의 route snapshot 구조와 필드 안정성은 BE 확인이 필요하다.
- OAuth provider 앱 키와 redirect 설정은 로컬 환경별 차이가 있을 수 있으므로 실기기 smoke 문서에 남긴다.
- 제보 이미지 업로드 정책이 변경되면 제보/북마크 후속 문서와 QA 기준을 다시 맞춘다.

## 5. 아카이브 기준

원본 6주차 백로그와 개인 업무 정리 문서는 계획/정리 원문으로 보존하되, 제출용 활성 문서 목록에서는 본 요약을 대표 문서로 사용한다.
