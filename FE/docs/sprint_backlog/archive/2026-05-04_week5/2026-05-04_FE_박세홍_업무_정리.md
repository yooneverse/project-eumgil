> **문서 성격:** 5주차 FE 공용 백로그에서 분리한 박세홍 개인 실행 문서

- 담당자: 박세홍
- 운영 기준: `Task 1개 = 브랜치 1개 = MR 1개`
- Jira 기준: Story와 Task는 `연결된 업무 항목`으로 연결하고, 서브태스크는 사용하지 않음
- 브랜치 기준: `fe/feat/{작업-slug}-{TaskNo}`. `{TaskNo}`에는 Jira 숫자만 넣고 브랜치명은 이슈 번호로 끝냄
- 5주차 목표: 북마크와 제보 흐름을 실제 API와 최신 디자인 기준으로 마감하고, 온보딩/마이페이지 약관 Notion 링크와 북마크/제보의 공통 상태, QA/QC, 접근성까지 정리해서 MVP 시연 시나리오에서 끊기지 않게 만든다.

## 1. 5주차 책임 범위

박세홍의 5주차 책임은 `북마크 API 실연동`, `제보 4단계 디자인 마감`, `제보 제출/완료 흐름 API 연결`, `온보딩/마이페이지 약관 Notion 링크 연결`, 그리고 그 과정에서 반드시 같이 터지는 `empty/error/loading 상태`, `접근성 라벨`, `QA/QC`, `재진입 handoff`를 닫는 것이다. 단순히 화면만 맞추는 것이 아니라, 하단 탭 `북마크`와 `제보`, 온보딩과 마이페이지의 약관 진입 동선이 실제 사용자 세션 기준으로 끝까지 이어지게 만드는 것이 목표다.

| 책임 흐름 | 5주차 종료 기준 |
| --- | --- |
| `BMK-01`, `BMK-02` | 장소/경로 북마크 목록, 삭제, 재진입, 길찾기 handoff가 실제 API 기준으로 동작 |
| `REPORT-01`~`REPORT-04` | 제보 4단계 디자인과 작성/제출/완료 흐름이 실제 API 및 상태 기준으로 마감 |
| `MYP-02`, `BMK`, `ROUTE` handoff | 제보 완료 후 후속 CTA와 북마크 재진입 흐름이 끊기지 않음 |
| `ONB`, `MYP` 약관 링크 | 온보딩과 마이페이지에서 동일한 Notion 약관 페이지로 안정적으로 이동 |
| 북마크/제보 공통 상태 UI | loading, empty, error, retry, offline 상태 문구와 UI가 정리됨 |
| 북마크/제보 접근성 및 QA/QC | TalkBack 라벨, 탭 노출 규칙, 버튼 설명, 화면별 QA/QC 점검 기준이 정리됨 |

## 5. 적당히 붙인 5주차 추가 일감

사용자가 먼저 준 `북마크 API`, `제보 디자인`, `제보 API`만으로는 5주차 마감 주간에 QA에서 다시 터질 가능성이 높다. 그래서 아래 추가 범위를 같이 박아두는 편이 맞다.

| 추가 일감 | 넣은 이유 | 대응 Task |
| --- | --- | --- |
| 북마크 재진입 handoff | 북마크는 목록만 붙여도 실제로는 상세/길찾기 진입에서 자주 끊긴다. | `북마크 상세/길찾기 재진입 handoff 정합성 마감` |
| 제보 pending/retry 상태 | 제출 API만 붙이면 실패/중복 제출/오프라인 상태가 빈약하게 남는다. | `REPORT 제출 API 연결 및 pending/retry 상태 정리` |
| 완료 후 CTA/후속 흐름 | 제보는 완료 화면만 나오고 그다음 행동이 비어 있으면 시연에서 어색하다. | `REPORT-04 완료 화면 디자인 마감 및 후속 CTA 정리` |
| 온보딩/마이페이지 약관 링크 | 약관 진입 동선이 빠지면 온보딩과 마이페이지가 서로 다른 정책 문서를 보거나 링크가 비게 된다. | `온보딩 화면 약관 Notion 링크 연결`, `마이페이지 화면 약관 Notion 링크 연결` |
| 공통 상태 UI | 북마크/제보는 empty/error/loading이 빠지면 QA 때 바로 다시 열린다. | `북마크/제보 empty/error/loading UI 정합성 조정` |
| 접근성 라벨 | 하단 탭, 카드, 제출 버튼은 TalkBack 점검 대상이라 미리 태스크로 빼두는 게 안전하다. | `북마크/제보 TalkBack 라벨·탭 노출 규칙 정리` |
| QA/QC 분리 점검 | MVP 마감 주간에는 기능 동작 QA뿐 아니라 문구, 외부 이동, 상태면 QC까지 같이 봐야 한다. | 각 Story별 `QA/QC` Task |
| 시연 회귀 QA/QC | MVP 마감 주간에는 기능 개발보다 시연 순서 기준 회귀 QA/QC가 더 중요하다. | `북마크/제보 시연 회귀 QA/QC` |

| 9 | `온보딩 화면 약관 Notion 링크 연결` | 온보딩 약관 진입은 사용자 첫 동선이라 별도로 안정화해야 한다. |

## 8. 협업 메모

- 북마크의 상세/길찾기 재진입 값은 이재호가 잡는 `FAC`, `ROUTE`, `NAV` 흐름과 맞춰야 한다.
- 제보 완료 후 `MYP-02` handoff는 마이페이지 쪽 상태 처리와 충돌하지 않게 연결 이슈 기준으로만 정리한다.
- 온보딩/마이페이지 약관 링크는 동일한 Notion 페이지를 사용하고, 링크 URL 변경 시 두 화면을 같이 수정한다.
- image upload 정책, 제보 제출 payload 세부값, 북마크 삭제 정책이 미확정이면 FE 임의 확정 대신 fallback 또는 `BE 확인 필요`로 남긴다.
- 북마크/제보 공통 상태 UI와 접근성 라벨은 시연 QA 직전이 아니라 기능 개발 중간부터 계속 같이 점검하는 편이 낫다.
