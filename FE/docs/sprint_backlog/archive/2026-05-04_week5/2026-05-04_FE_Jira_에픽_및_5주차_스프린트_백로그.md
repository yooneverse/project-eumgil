## 0. 문서 역할과 반영 기준

이 문서는 5주차 FE 공용 실행 백로그다. 4주차처럼 개인 문서를 별도로 만들지 않고, `현재 작업기준`을 팀 공통 작업 축으로만 정리한다. 5주차의 핵심은 새 기능 확장이 아니라 `MVP 마감`, `실제 서버 연동`, `UI 정합성`, `접근성/회귀 QA`, `시연 안정화`다.

개인별 Jira 분해안과 개인 실행 문서는 공용 백로그에 넣지 않는다. 이재호와 박세홍 개인 업무정리는 각각 `FE/docs/sprint_backlog/2026-05-04_FE_이재호_업무_정리.md`, `FE/docs/sprint_backlog/2026-05-04_FE_박세홍_업무_정리.md`에서 별도로 관리한다.

### 0.2 5주차 공통 원칙

1. 로그인은 필수이며 guest mode를 다시 만들지 않는다.
2. `AUTH -> ONB -> MAP -> DEST -> ROUTE -> NAV -> ARR`, `BMK`, `REPORT`, `MYP` 흐름이 실제 서버 기준으로 끊기지 않게 만드는 것이 우선이다.
3. API가 미확정인 항목은 FE에서 임의 확정하지 않고 `BE 확인 필요`, `추후 구현`, `fallback 유지`로 분리한다.
4. 보행약자 공통 탭은 `홈 / 북마크 / 제보 / 마이페이지` 4탭 기준을 유지한다.
5. 저시력자 전용 `LV-*` 화면군 전체 재설계는 이번 주 범위에 넣지 않고, 시작 분기와 세션 정합성만 맞춘다.
6. 일반모드 디자인 정합성은 `4 / 8 / 12 / 16 / 24 / full` radius scale과 icon/API 매핑 기준을 따른다.

## 1. 5주차 스프린트 목표

1. `AUTH-001`, 앱 시작 분기, `ONB-001`~`ONB-005`를 실제 인증/API 구조에 맞춰 마감한다.
2. 장소/검색/경로/북마크/제보를 mock 중심 상태에서 dev 서버 실연동 기준으로 전환한다.
3. `ROUTE-01`, `ROUTE-02`, `NAV-01`, `ARR-01/02`의 데이터 기반 상태와 위험 정보 정합성을 맞춘다.
4. `MYP`, `BMK`, `REPORT`의 사용자 데이터 흐름을 실제 인증 세션 기준으로 정리한다.
5. 일반모드 UI radius/shadow/icon 정합성과 핵심 접근성 라벨/읽기 순서를 마감한다.
6. `2026-05-09 ~ 2026-05-10`에는 신규 기능 추가를 멈추고 회귀 QA, 버그 수정, 시연 동선 안정화에 집중한다.

## 7. 완료 기준

1. FE 핵심 시나리오가 실제 서버 기준 또는 명시적 fallback 기준으로 끊기지 않는다.
2. `profileCompleted` 같은 구 기준이 앱 시작 분기 기준으로 남아 있지 않다.
3. 장소 카테고리, 접근성 feature, 제보 타입, 경로 옵션이 최신 문서/API 기준과 일치한다.
4. `REPORT`, `BMK`, `MYP`가 인증 세션 기준으로 동작한다.
5. 일반모드 UI에서 radius/elevation/icon 정합성이 최신 FE 문서 기준으로 맞는다.
6. TalkBack/TTS, 권한, empty/error/offline, 핵심 CTA 회귀 점검이 끝난다.
7. `2026-05-10` 이후에는 신규 기능이 아니라 발표용 안정화 백로그만 남는다.
