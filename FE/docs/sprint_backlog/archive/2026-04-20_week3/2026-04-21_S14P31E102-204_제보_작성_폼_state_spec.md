## A. 설계 요약

- 화면은 `feature/report/ReportContract.kt`, `ReportRoute.kt`, `ReportScreen.kt`, `ReportViewModel.kt` 패턴으로 구현한다.
- 상태는 `ReportUiState + ReportUiAction + ReportUiEvent` 단방향 흐름을 사용한다.
- MVP 제출 필수값은 `제보 유형`, `위치 좌표` 2개다. 설명과 사진은 선택값이지만 validation 대상이다.
- 사진은 현재 `ReportDraftEntity.photoUri`와 맞춰 선택 1장으로 제한한다.
- draft 저장 가능 조건은 "사용자가 의미 있는 값을 하나라도 입력함"이고, submit 가능 조건은 "필수값과 제약을 모두 통과함"이다.
- 오류는 색상만으로 표현하지 않고 label, helper text, error text, 접근성 announcement를 함께 사용한다.

## C. `ReportUiState` Kotlin data class 예시

실제 계약 파일은 `FE/app/src/main/java/com/ssafy/e102/eumgil/feature/report/ReportContract.kt`에 둔다.

```kotlin
data class ReportUiState(
    val screenState: ReportScreenState = ReportScreenState.Editing,
    val draftId: String? = null,
    val hasExistingDraft: Boolean = false,
    val reportType: ReportTypeInput = ReportTypeInput(),
    val location: ReportLocationInput = ReportLocationInput(),
    val photo: ReportPhotoInput = ReportPhotoInput(),
    val description: ReportDescriptionInput = ReportDescriptionInput(),
    val draftSaveState: ReportDraftSaveState = ReportDraftSaveState.Idle,
    val outboxState: ReportOutboxState = ReportOutboxState.NotSaved,
    val submitState: ReportSubmitState = ReportSubmitState.Idle,
) {
    val isDraftSavable: Boolean
        get() = reportType.value != null ||
            location.value != null ||
            photo.value != null ||
            description.value.isNotBlank()

    val isSubmitEnabled: Boolean
        get() = screenState == ReportScreenState.Editing &&
            submitState !is ReportSubmitState.Submitting &&
            reportType.value != null &&
            location.value != null &&
            reportType.error == null &&
            location.error == null &&
            photo.error == null &&
            description.error == null
}
```

필드별 입력 상태 기준은 다음과 같다.

| 상태 | 적용 필드 | 기준 |
| --- | --- | --- |
| `isTouched` | 유형, 위치, 사진, 설명 | blur, 선택 시도, 제출 시 true |
| `isDirty` | 유형, 위치, 사진, 설명 | 기본값과 다른 값으로 변경되면 true |
| `isValidating` | 일반 입력 필드 | 현재 단계에서는 불필요. 동기 validation으로 처리 |
| `isResolvingCurrentLocation` | 위치 | 현재 위치 요청 중 true |
| `error` | 유형, 위치, 사진, 설명 | validation 실패 시 enum으로 저장 |
| `isDraftSavable` | 전체 폼 | 입력값 하나라도 있으면 true |
| `isSubmitEnabled` | 전체 폼 | 필수값 존재 + 모든 error null + 제출 중 아님 |

## D. `ReportUiAction` / `ReportUiEvent` sealed interface 예시

```kotlin
sealed interface ReportUiAction {
    data object BackClicked : ReportUiAction
    data object DraftResumeClicked : ReportUiAction
    data object DraftDiscardClicked : ReportUiAction
    data class ReportTypeSelected(val type: ReportType) : ReportUiAction
    data object ReportTypeBlurred : ReportUiAction
    data object CurrentLocationResetClicked : ReportUiAction

    data class LocationSelected(
        val location: ReportLocation,
        val source: ReportLocationSource,
    ) : ReportUiAction
    data class AddressTextChanged(val address: String) : ReportUiAction
    data object LocationBlurred : ReportUiAction
    data object PhotoAddClicked : ReportUiAction
    data class PhotoSelected(val photo: ReportPhoto) : ReportUiAction
    data object PhotoRemoved : ReportUiAction
    data object PhotoBlurred : ReportUiAction
    data class DescriptionChanged(val description: String) : ReportUiAction
    data object DescriptionBlurred : ReportUiAction
    data object SaveDraftClicked : ReportUiAction
    data object SubmitClicked : ReportUiAction
    data object RetrySubmitClicked : ReportUiAction

sealed interface ReportUiEvent {
    data object NavigateBack : ReportUiEvent
    data object ShowDraftDiscardDialog : ReportUiEvent
    data object RequestLocationPermission : ReportUiEvent
    data object OpenLocationPicker : ReportUiEvent
    data object OpenPhotoPicker : ReportUiEvent

    data class ShowSnackbar(val message: String) : ReportUiEvent
    data class NavigateToReportComplete(
        val reportId: Long? = null,
        val outboxId: String? = null,
    ) : ReportUiEvent
}
```

## E. validation 규칙 표

| 항목 | 규칙 | 검증 시점 | 오류 노출 |
| --- | --- | --- | --- |
| 제보 유형 미선택 | `reportType.value == null`이면 `ReportTypeError.Required` | 유형 영역 blur, submit | submit 시 첫 오류로 scroll + TalkBack announcement |
| 위치 미선택 | `location.value == null`이면 `ReportLocationError.Required` | 위치 영역 blur, submit | 주소/helper 아래 error text 노출 |
| 위치 좌표 오류 | 위도/경도가 일반 범위를 벗어나면 `InvalidCoordinate` | 위치 선택 직후, submit | 지도 영역과 주소 영역 모두 오류 상태 |
| 현재 위치 실패 | 권한 거부는 `PermissionDenied`, 위치 불가는 `CurrentLocationUnavailable` | 현재 위치 재설정 결과 수신 시 | snackbar + 위치 helper error |
| 사진 첨부 | 선택값, 미첨부는 오류 아님 | 사진 선택 직후 | 미첨부 상태는 helper text 유지 |
| 사진 형식 | `image/*`가 아니면 `UnsupportedFormat` | 사진 선택 직후 | 사진 영역 error text |
| 사진 크기 | 10MB 초과 시 `TooLarge` | 사진 선택 직후 | 사진 영역 error text |
| 설명 길이 | 500자 초과 시 `TooLong` | 입력 중 즉시, blur, submit | 글자 수 카운터 + error text |
| 설명 공백-only | `trim()` 결과 빈 값이면 빈 설명으로 저장/제출 | draft 저장, submit 직전 | 오류 아님 |
| draft 저장 가능 | 유형, 위치, 사진, 설명 중 하나라도 입력 | 입력 변경 시 즉시 | 저장 버튼 활성화 또는 자동 저장 트리거 |
| submit 가능 | 유형 선택 + 위치 좌표 있음 + 사진/설명 제약 통과 | 입력 변경 시 즉시 | submit 버튼 활성화 |
| 서버 `INVALID_INPUT` | FE 통과 후 서버가 거부하면 `InvalidInput` failure | submit 응답 수신 시 | 폼 유지, 재시도 가능, 첫 필수 필드 재검증 |

## F. 상태 전이 표

| 상태 | `ReportUiState` 기준 | 진입 조건 | 다음 액션 |
| --- | --- | --- | --- |
| 초기 진입 | `screenState=InitialLoading` 또는 `Editing`, `hasExistingDraft=false` | 화면 최초 진입, draft 조회 전/후 | draft 있으면 resume 안내, 없으면 입력 대기 |
| 입력 중 | `screenState=Editing`, 일부 `isDirty=true` | 사용자가 필드 변경 | field validation, draft 가능 여부 갱신 |
| 필수값 누락 | `Editing`, 필수 필드 `error=Required` | blur 또는 submit 시 필수값 없음 | error text 노출, submit 비활성 |
| 일부만 입력된 draft 가능 상태 | `isDraftSavable=true`, `isSubmitEnabled=false` | 필수값 일부 누락 또는 제약 오류 있음 | `SaveDraftClicked` 또는 자동 저장 |
| 제출 가능 상태 | `isSubmitEnabled=true` | 유형 + 위치 valid, 사진/설명 valid | `SubmitClicked` |
| 제출 진행 중 | `screenState=Submitting`, `submitState=Submitting` | submit 시작 | 중복 입력/제출 방지, progress 표시 |
| outbox 저장 완료 | `outboxState=Saved(outboxId)` | 네트워크 전송 전 로컬 outbox 저장 성공 | 완료 피드백 또는 백그라운드 전송 대기 |
| 제출 완료 피드백 | `screenState=Completed`, `submitState=Success` | 서버 제출 성공 또는 outbox 수락 | `NavigateToReportComplete`, snackbar |
| 실패 후 재시도 가능 상태 | `screenState=Failure`, `submitState=Failed` 또는 `outboxState=Failed` | 서버/네트워크/로컬 저장 실패 | 폼 값 유지, `RetrySubmitClicked` 가능 |

## G. 205/206/207 연결 메모

- 205 화면 shell: chip/list 기반 `ReportType`, 위치 카드, 사진 첨부 영역, 설명 TextField를 `ReportUiState`만 받아 그린다. placeholder는 보조 역할만 하고 label은 별도 노출한다.
- 206 validation/state: `ReportUiAction`을 `ReportViewModel`에서 받아 touched/dirty/error를 갱신하고, `isSubmitEnabled` 기준으로 submit 버튼을 제어한다.
- 207 draft/outbox: `ReportDraftEntity` 매핑은 `reportType.apiValue`, `description.trim()`, `location.address`, `latitude`, `longitude`, `photo.localUri` 기준으로 한다. outbox 저장은 `ReportOutboxState`로 submit feedback과 분리한다.
