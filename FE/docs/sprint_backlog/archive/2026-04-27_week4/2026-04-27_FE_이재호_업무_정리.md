## 1. 4주차 사용자 흐름 책임

이재호의 4주차 책임은 MVP 시연의 첫 진입, 지도 탐색, 경로/길안내, 도착, 접근성 QA 기준을 한 흐름으로 닫는 것이다. 단순 화면 배치가 아니라 기능 동작, overlay, bottom sheet, card, CTA, 탭 표시/숨김, 상태 문구, 접근성 라벨까지 디자인 문서 기준으로 맞춘다.

| 책임 흐름 | 4주차 종료 기준 | 디자인 정합성 기준 |
| --- | --- | --- |
| 앱 실행 -> `AUTH-001` -> `ONB-001`~`ONB-005` -> `selectedPrimaryUserType` 홈 분기 | 세션 없음, 온보딩 미완료, 완료 상태별 startDestination이 동작 | 로그인 화면, 5단계 온보딩, progress/page indicator, 하단 CTA |

## 5.1 S14P31E102-313 AUTH-001 BE 확인 메모

- `AUTH-001`은 현재 로그인 gate, 로딩/오류/성공 이벤트, 다음 gate 재판단까지만 구현 범위로 둔다.

- Google/Naver/Kakao는 현재 화면 버튼과 테스트용 UI provider key이며, BE provider enum 또는 token 정책으로 확정하지 않는다.
- BE 확인 필요 항목은 `FE/docs/todo/2026-04-27_BE_연동_작업_요청사항.md`의 `1.0 AUTH-001 소셜 로그인 연동 확인 필요`를 기준으로 추적한다.
