## 1. 4주차 사용자 흐름 책임

박세홍의 4주차 책임은 제보와 북마크 프레임을 실제 MVP 기능으로 닫고, 하단 탭 `제보`/`북마크` 화면의 디자인 정합성을 맞추는 것이다.

| 책임 흐름 | 4주차 종료 기준 | 디자인 정합성 기준 |
| --- | --- | --- |
| `REPORT-01 -> REPORT-02 -> REPORT-03 -> REPORT-04 -> MYP-02` | 유형 선택, 위치 확인, 사진/설명, 제출 상태, 완료 후 내역 진입이 동작 | 유형 카드, 위치 지도, 사진 카드, textarea, 완료 요약 카드, 하단 `제보` 탭 활성 |
| `BMK-01/BMK-02 -> FAC-01/ROUTE-02/NAV-01` | 장소/경로 북마크 목록, 삭제, 상세, 길찾기 진입이 동작 | 장소/경로 segmented 탭, 카드 레이아웃, 접근성 라벨, 길찾기 CTA, empty/error 상태 |

- 이재호의 `FAC-01`, `ROUTE-02`, `NAV-01` handoff와 `BMK-01`/`BMK-02` 전달값이 충돌하지 않게 맞춘다.
- 김지윤의 `MYP-02` route/계정 정책이 확정되기 전까지 제보 내역 조회는 `BE 확인 필요`로 남긴다.
- 제보 imageUrl 선업로드, 제보 목록 조회, 북마크 삭제 단위, 즉시 길안내 payload는 BE 확인 전까지 임의 확정하지 않는다.

## 8. MR 작성 메모

- MR 설명에는 반영한 디자인 문서와 화면 ID를 적는다.
- 제보 MR은 단계별 기능 동작, 상태 문구, 완료 후 `MYP-02` 진입 여부를 남긴다.
