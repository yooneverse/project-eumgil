## 0. 현재 판단 기준

지금 남아 있는 Jira 항목은 실제 구현 상태보다 넓게 열려 있다. 따라서 이번 주에는 **현재 `FE/app` 코드와 이미 작성된 QA/debug 문서**를 기준으로 판단하고, 새 기능 추가보다 남은 리스크만 다시 묶는다.

| 기준 | 처리 방식 |
| --- | --- |
| 코드와 문서 산출물이 모두 있고 남은 리스크가 없음 | Jira 종료 |
| 구현은 들어갔지만 제목이 너무 넓어서 현재 남은 범위를 설명하지 못함 | 제목 변경 |
| 같은 성격의 작은 잔업이 둘 이상으로 갈라져 있음 | 1개로 합치고 나머지는 종료 |
| 예비 작업 / 지난 주 문서 태스크처럼 지금 주차와 맞지 않음 | 이번 주 문서 또는 QA 태스크로 재활용하거나 종료 |

## 3. 6주차 실제 업무 정리

이번 주에는 `새 화면 구현`보다 `이미 들어간 흐름을 시연 가능한 수준으로 닫는 것`이 중요하다. 우선순위는 아래 순서로 잡는다.

| 순서 | 작업 | 연결 Jira | 완료 기준 |
| ---: | --- | --- | --- |
| 1 | 앱 cold-start 시 `/users/me` 재동기화와 start destination 보강 | `445`와 별도 연결 또는 신규 task | 앱 재실행 시 사용자 유형과 시작 화면이 서버 기준으로 다시 맞춰진다. |
| 2 | 마이페이지 외부 링크 fallback 및 URL 상수 정리 | `449` | 브라우저 미설치/실행 실패 시 크래시 없이 복구되고, 온보딩과 마이페이지의 URL 기준이 하나로 정리된다. |
| 3 | 경로 북마크 상세 조회와 route snapshot 재진입 보강 | `445` | 북마크 경로에서 `ROUTE/NAV/ARR` 흐름이 끊기지 않는다. |
| 4 | `ROUTE -> NAV -> ARR -> 경로저장 -> BMK 재진입` 수동 QA | `452` | 저장/평가/재진입 시나리오가 실기기 또는 수동 QA 기준으로 검증된다. |
| 5 | `MAP / DEST / ROUTE` 수동 QA 체크리스트와 시연 smoke 작성 | `340` | 지도, 검색, 목적지 handoff, 경로 진입, empty/error/offline 상태가 문서로 정리된다. |
| 6 | 일반모드 radius / shadow / icon 잔여 정리와 접근성 회귀 점검 | `453`, `339` | 일반모드 핵심 화면 parity와 공통 UI/접근성 회귀 기준이 마감된다. |
| 7 | 이번 주 이재호 업무 정리 문서 마감 | `341` | Jira 정리안과 다음 주 실행 순서가 문서로 남는다. |

## 4. 이번 주에 다시 열지 않을 항목

| 항목 | 처리 기준 |
| --- | --- |
| 검색 기능 신규 확장 | 현재 검색은 구현이 닫혔으므로 신규 API/UX 확장은 열지 않는다. |
| 저시력 전용 화면 전체 polish | 일반모드 carry-over를 먼저 닫고 후순위로 둔다. |
| 고객센터 / 1:1 문의 실연결 | 준비중 상태를 유지한다. |
| 발표 자료 제작 | FE 구현/QA 범위 밖으로 둔다. |

## 5. BE 확인 필요 사항

- `route bookmark detail` 응답에 포함되는 `route snapshot` 구조와 필드 안정성은 다시 한 번 확인이 필요하다.
- OAuth provider 앱 키/redirect 설정이 로컬 환경마다 다르면 FE 실기기 smoke 문서에 환경 차이를 남겨야 한다.
- 제보 이미지 업로드 정책이 바뀌면 제보/북마크 후속 문서와 QA 기준도 다시 맞춰야 한다.

## 6. 한 줄 정리

이번 주 이재호 범위는 `완료된 구현을 Jira에서 정리`하고, 실제 남은 일만 `fallback`, `재진입`, `수동 QA`, `일반모드 parity`로 다시 좁혀 잡는 것이 맞다.
