## 1. 6주차 스프린트 목표

1. 5주차에 기능 코드는 들어갔지만 운영/시연 기준으로 덜 닫힌 항목만 추려 마감한다.
2. 인증, 마이페이지, 길찾기처럼 **실서버 세션과 외부 이동**이 걸린 경로를 우선 안정화한다.
3. 기능 추가보다 **carry-over 제거, 수동 QA 체크리스트, 시연 smoke**를 우선한다.
4. 공통 UI 정합성은 화면 추가가 아니라 `일반모드 회귀 안정화` 범위에서만 마감한다.

## 4. 이번 주 우선순위

| 순서 | Story / Task | 이유 |
| ---: | --- | --- |
| 1 | `/users/me` 재동기화와 start destination 보강 | cold-start 오동작은 인증/마이페이지/길찾기 전체를 흔든다. |
| 2 | 마이페이지 외부 링크 fallback | 현재 시연에서 가장 눈에 띄는 크래시 리스크다. |
| 3 | 경로 북마크 상세 조회 / 재진입 보강 | 5주차에서 기능은 닫혔지만 저장 후 재진입 품질이 아직 약하다. |
| 4 | ARR/NAV end-to-end QA | 서버 세션과 routeId/sessionId가 실제로 이어지는지 확인해야 한다. |
| 5 | 지도/검색/길찾기 수동 QA 체크리스트 | 기능 코드를 다시 열기보다 현재 동선을 고정하는 편이 효율적이다. |
| 6 | 일반모드 UI 정합성 마감 | 기능 동선이 고정된 뒤 visual polish를 정리한다. |

## 5. 이번 주에 다시 열지 않을 항목

| 항목 | 처리 기준 |
| --- | --- |
| 장소/검색 API 신규 확장 | 5주차 대비 새 API 추가는 하지 않고 현재 연동 경로의 QA만 수행한다. |
| 저시력 전용 UI 전체 polish | 일반모드 carry-over 해소 이후 후순위로 둔다. |
| 고객센터 / 1:1 문의 실제 연결 | `준비중인 기능입니다.` 수준을 유지한다. |
| 발표 자료 / 녹화 편집 | 본 스프린트 백로그 범위에서 제외한다. |

## 6. BE 확인 필요 / Cross-Lane Handoff

- 현재 문서 기준 **즉시 필요한 신규 BE 요청은 없다.**
- 다만 아래 항목은 FE 후속 구현 시 다시 확인이 필요하다.
  - 경로 북마크 상세 조회 응답에 포함되는 route snapshot 구조와 필드 안정성
  - 제보 image upload 정책 변경 시 report/backlog 재조정
  - 인증 dev 환경에서 provider 앱 키/redirect 설정이 로컬별로 다른 경우 환경 문서 보강
