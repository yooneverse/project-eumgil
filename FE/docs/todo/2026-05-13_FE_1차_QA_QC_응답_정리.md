# FE 1차 QA/QC 응답 정리
> 문서일: 2026-05-13
> Work Lane: FE
> 상태: 1차 QA/QC 답변 기준 재분류 완료, 수정완료 항목 반영
> 기준: 1차 QA/QC 30건 + FE 응답 메모

## 1. 목적

- 1차 QA/QC 항목을 FE 기준으로 추적 가능한 작업 목록으로 재정리한다.
- 상태를 `수정완료 / 추가 작업 없음`, `팀결정 필요`, `TODO`로 통일한다.
- FE 단독 수정 가능 여부, 담당자, 의존성, 백엔드 요청사항을 함께 남긴다.

## 2. 정리 원칙

- 이 문서는 실제 코드 수정 여부만이 아니라, 원인 파악 완료 또는 FE 추가 작업 불필요 여부까지 포함해 상태를 정리한다.
- `수정완료 / 추가 작업 없음` 섹션에는 코드 수정 완료, 동작 확인 완료, FE 이슈 아님 판정 항목을 함께 넣었다.
- `팀결정 필요` 섹션에는 UI/UX 정책 또는 기능 유지 여부에 대한 합의가 필요한 항목을 넣었다.
- `TODO` 섹션에는 수정 중, 재현 확인 필요, POC 필요, 타 담당자 확인 필요 항목을 넣었다.
- 백엔드 수정이 필요한 항목은 FE 문서에 남기되, 실제 처리 요청은 별도 handoff로 정리한다.

## 3. 상태 요약

- `수정완료 / 추가 작업 없음`: 9건 (`2`, `3`, `4`, `11`, `17`, `19`, `20`, `24`, `30`)
- `팀결정 필요`: 5건 (`1`, `6`, `7`, `9`, `14`)
- `TODO`: 16건 (`5`, `8`, `10`, `12`, `13`, `15`, `16`, `18`, `21`, `22`, `23`, `25`, `26`, `27`, `28`, `29`)
- `FE 단독 수정 불가 또는 외부 협업 필요`: `12`, `30`

- `4번`은 FE 범위 수정이 완료되어 별도 BE handoff 대상에서 제외한다.
- `12번`, `14번`은 각각 지윤 담당 확인 또는 협업이 필요하다.
- `30번`은 FE가 아니라 BE의 대표 장소명 정규화로 추적해야 한다.

## 8. 파트별 질문 정리

### 8.1 가입 / 권한 / 온보딩

- 위치 권한 동의는 어느 시점에 요청하는 것이 가장 자연스러운가?
- 재가입 시 튜토리얼 미노출 현상의 정확한 재현 조건은 무엇인가?
- 저시력자 음성 입력 timeout은 몇 초가 적절한가?

### 8.2 지도 / POI / 상세
- 접근성 카테고리 검색 결과는 현재 위치 기준으로 제한할지, 지도 범위 기준으로 확장할지?

### 8.3 북마크 / 최근 검색 / 최근 목적지
- 최근 검색의 mock 카테고리는 제거할지, 실데이터 기반 추천으로 대체할지?
- 최근 목적지는 어떤 행동을 기준으로 기록해야 하는가?
- 최근 목적지 접힘 상태를 사용자가 다시 펼칠 수 있는 명시적 액션이 필요한가?
- 검색 결과를 거리순 정렬로 바꿀 경우, 거리 표시도 함께 보여줄 것인가?

### 8.4 마이페이지 / 제보

- 제보 탭 실구현 완료 후 어떤 항목을 재검증해야 하는가?

### 8.5 경로 안내 / 배포

- 경로 안내 지도 확대 수준을 어느 정도까지 공격적으로 가져갈 것인가?
- 경로 저장 완료, 경로 이탈, 평가 완료 화면의 CTA 정렬 기준을 공통화할 것인가?
- FE prod 환경 전환 시 smoke test 범위는 어디까지로 잡을 것인가?

## 9. Cross-Lane Handoff
