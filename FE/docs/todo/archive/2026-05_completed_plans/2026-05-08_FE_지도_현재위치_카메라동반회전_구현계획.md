**Goal:** 지도 화면에서 현재위치 버튼을 두 번 눌렀을 때 `현재위치 추적 + 카메라 방향 회전` 모드에 진입하고, 기기 방향 변화에 맞춰 지도 카메라와 현재위치 puck이 함께 회전하도록 구현한다.

**Architecture:** 위치 추적과 별도로 `heading` 수집 계층을 추가하고, `MapViewModel`이 `IDLE -> FOLLOW -> FOLLOW_WITH_HEADING` 추적 모드를 관리한다. 카메라 회전은 `MapCameraTarget.rotationDegrees`를 통해 단일 상태로 흘려보내고, Kakao Map SDK는 `CameraPosition.from(..., rotationAngle, ...)` 기반으로 회전한다.

**Tech Stack:** Kotlin, StateFlow, Android `SensorManager`, Kakao Vector Map SDK 2.13.1, Jetpack Compose, JUnit4, `kotlinx-coroutines-test`

## Current Implementation Fact

- 현재 위치 모델은 `latitude`, `longitude`, `accuracyMeters`, `recordedAtEpochMillis`만 가지며 방향값이 없다.
- 일반 지도 화면은 `cameraTarget.center`, `cameraTarget.zoomLevel`만 관리하고 회전 상태는 없다.
- 현재위치 버튼 액션은 `MapUiAction.LocationActionClicked` 한 종류이며, 명시적 탭 시 카메라만 현재위치로 재중심 이동한다.
- 일반 지도 `overlayState`는 시설 마커와 `camera-focus`만 만들고, 현재위치 방향 puck은 아직 없다.
- Kakao SDK 바인딩은 `CameraUpdateFactory.newCenterPosition(...)`만 사용한다.
- 프로젝트 내부에 센서 기반 방향 추적 계층은 아직 없다.

## Target UX Contract

- 지도 최초 진입 또는 자동 위치 동기화만으로는 추적 모드가 켜지지 않는다.
- 현재위치 버튼 첫 탭: `FOLLOW` 모드 진입, 현재위치로 재중심 이동, 북쪽 고정 유지.
- 현재위치 버튼 두 번째 탭: `FOLLOW_WITH_HEADING` 모드 진입, 현재위치로 재중심 이동, 기기 heading에 맞춰 카메라와 puck 회전.
- `FOLLOW_WITH_HEADING` 중 기기를 회전하면 5도 이상 변화 시 카메라 bearing을 갱신한다.
- 지도에서 `Pan`, `Zoom`, `Rotate`, `Tilt`, `LongTapAndDrag`, `RotateZoom`, `OneFingerZoom` 제스처가 시작되면 추적 모드를 해제하고 센서 수집도 중단한다.
- `FOLLOW_WITH_HEADING` 중 현재위치 버튼을 다시 누르면 모드를 유지한 채 즉시 재중심 이동한다.
- 방향 센서가 없거나 정확도가 낮으면 `FOLLOW`까지만 동작하고, 카메라는 북쪽 고정을 유지한다.

```kotlin
data class HeadingSnapshot(
    val azimuthDegrees: Double,
    val accuracy: Int?,
    val recordedAtEpochMillis: Long,
)

interface CurrentHeadingManager {
    val latestHeading: StateFlow<HeadingSnapshot?>

    fun startHeadingUpdates()

    fun stopHeadingUpdates()
}

internal fun normalizeDegrees(raw: Double): Double =
    ((raw % 360.0) + 360.0) % 360.0

internal fun shortestAngleDelta(from: Double, to: Double): Double {
    val normalized = normalizeDegrees(to) - normalizeDegrees(from)
    return when {
        normalized > 180.0 -> normalized - 360.0
        normalized < -180.0 -> normalized + 360.0
        else -> normalized
    }
}

internal fun smoothAngle(previous: Double, next: Double, alpha: Double): Double =
    normalizeDegrees(previous + (shortestAngleDelta(previous, next) * alpha))

internal fun shouldApplyHeadingUpdate(previous: Double?, next: Double, minDelta: Double): Boolean =
    previous == null || kotlin.math.abs(shortestAngleDelta(previous, next)) >= minDelta
```

```kotlin
class AndroidCurrentHeadingManager(
    context: Context,
) : CurrentHeadingManager, SensorEventListener {
    private val sensorManager =
        context.applicationContext.getSystemService(Context.SENSOR_SERVICE) as SensorManager

    private val rotationVectorSensor =
        sensorManager.getDefaultSensor(Sensor.TYPE_ROTATION_VECTOR)

    private val mutableLatestHeading = MutableStateFlow<HeadingSnapshot?>(null)
    private var lastEmittedDegrees: Double? = null
    private var isTracking = false

    override val latestHeading: StateFlow<HeadingSnapshot?> = mutableLatestHeading.asStateFlow()
}
```

- [ ] **Step 2: start/stop 구현**

```kotlin
override fun startHeadingUpdates() {
    if (isTracking || rotationVectorSensor == null) return
    sensorManager.registerListener(
        this,
        rotationVectorSensor,
        SensorManager.SENSOR_DELAY_GAME,
    )
    isTracking = true
}

override fun stopHeadingUpdates() {
    if (!isTracking) return
    sensorManager.unregisterListener(this)
    isTracking = false
}
```

```kotlin
override fun onSensorChanged(event: SensorEvent) {
    if (event.sensor.type != Sensor.TYPE_ROTATION_VECTOR) return

    val rotationMatrix = FloatArray(9)
    val orientationAngles = FloatArray(3)
    SensorManager.getRotationMatrixFromVector(rotationMatrix, event.values)
    SensorManager.getOrientation(rotationMatrix, orientationAngles)

    val rawDegrees = Math.toDegrees(orientationAngles[0].toDouble())
    val normalized = normalizeDegrees(rawDegrees)
    val smoothed = smoothAngle(lastEmittedDegrees ?: normalized, normalized, alpha = 0.25)
    if (!shouldApplyHeadingUpdate(lastEmittedDegrees, smoothed, minDelta = 5.0)) return

    lastEmittedDegrees = smoothed
    mutableLatestHeading.value =
        HeadingSnapshot(
            azimuthDegrees = smoothed,
            accuracy = event.accuracy,
            recordedAtEpochMillis = System.currentTimeMillis(),
        )
}
```

```kotlin
private var latestHeading: HeadingSnapshot? = currentHeadingManager.latestHeading.value

private fun handleLocationAction() {
    val snapshot = latestLocation ?: return retryLocationResolution()

    when (mutableUiState.value.locationTrackingMode) {
        MapLocationTrackingMode.IDLE -> {
            currentHeadingManager.stopHeadingUpdates()
            syncCameraToCurrentLocation(snapshot = snapshot, incrementRequestId = true, rotationDegrees = 0.0)
            mutableUiState.update { it.copy(locationTrackingMode = MapLocationTrackingMode.FOLLOW) }
        }
        MapLocationTrackingMode.FOLLOW -> {
            currentHeadingManager.startHeadingUpdates()
            syncCameraToCurrentLocation(
                snapshot = snapshot,
                incrementRequestId = true,
                rotationDegrees = latestHeading?.azimuthDegrees ?: 0.0,
            )
            mutableUiState.update { it.copy(locationTrackingMode = MapLocationTrackingMode.FOLLOW_WITH_HEADING) }
        }
        MapLocationTrackingMode.FOLLOW_WITH_HEADING -> {
            syncCameraToCurrentLocation(
                snapshot = snapshot,
                incrementRequestId = true,
                rotationDegrees = latestHeading?.azimuthDegrees ?: mutableUiState.value.cameraTarget.rotationDegrees,
            )
        }
    }
}

private fun handleViewportGestureStarted() {
    currentHeadingManager.stopHeadingUpdates()
    mutableUiState.update { it.copy(locationTrackingMode = MapLocationTrackingMode.IDLE) }
}

private fun syncCameraToCurrentLocation(
    snapshot: LocationSnapshot,
    incrementRequestId: Boolean,
    rotationDegrees: Double,
) {
    val coordinate = snapshot.toMapCoordinate()
    mutableUiState.update { state ->
        state.copy(
            cameraTarget =
                MapCameraTarget(
                    center = coordinate,
                    source = MapCameraSource.CURRENT_LOCATION,
                    requestId = if (incrementRequestId) state.cameraTarget.requestId + 1L else state.cameraTarget.requestId,
                    zoomLevel = if (incrementRequestId) MapCameraSource.CURRENT_LOCATION.defaultZoomLevel() else state.cameraTarget.zoomLevel,
                    rotationDegrees = rotationDegrees,
                ),
        )
    }
}
```

```kotlin
private class FakeCurrentHeadingManager(
```

## Risks

- 센서 heading은 실내 자기장 간섭에 취약하므로 smoothing만으로 모든 흔들림을 제거할 수 없다.
- 카메라 bearing을 매 heading 이벤트마다 갱신하면 `moveCamera` 빈도가 과해질 수 있다.
- Kakao `OnCameraMoveStartListener`가 프로그램적 이동에도 호출되므로 `isDispatchingProgrammaticCameraMove` 가드가 없으면 추적 모드가 즉시 꺼질 수 있다.
- 현재위치 puck를 일반 지도에도 노출하면 시설 마커와 겹칠 수 있으므로 `includeInProjection = false`와 높은 `zIndex`가 필요하다.

## Acceptance Criteria

- FE 코드만 수정해서 구현 가능해야 한다.
- 지도 현재위치 버튼의 두 번째 명시적 탭에서 heading 기반 카메라 회전이 시작되어야 한다.
- manual gesture 시작 시 follow/heading 모드가 해제되어야 한다.
- 회전값은 `MapCameraTarget` 단일 상태를 통해 SDK와 fallback overlay 양쪽에 동일하게 전달되어야 한다.
- heading 연산 로직과 추적 모드 전이는 단위 테스트로 재현 가능해야 한다.
