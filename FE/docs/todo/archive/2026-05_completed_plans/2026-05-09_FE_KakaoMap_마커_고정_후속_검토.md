# FE KakaoMap 마커 고정 후속 검토

> 작성일: 2026-05-09  
> Work Lane: FE  
> 대상 화면: MAP-01 지도 메인, FAC-01 시설 상세 진입 전 지도 마커  
> 관련 파일: `KakaoMapViewport.kt`, `KakaoMapViewportBindings.kt`, `ic_map_current_location.xml`

## 1. 배경

- Kakao Vector Map SDK 위에 현재 위치 마커, 선택 목적지 마커, 지도 탭 선택 핀을 표시하는 과정에서 지도 이동 중 마커가 흔들리거나, 이동 중 지도를 따라왔다가 놓으면 원래 좌표로 돌아가는 현상이 발생했다.
- 시설 마커는 Kakao SDK `LabelLayer` 기반으로 표시하고 있고, 현재 위치/선택 핀 계열은 Compose 오버레이 기반으로 화면 좌표를 계산해 표시하는 구조였다.
- Compose 오버레이는 Android `SurfaceView`/지도 렌더러와 별도 렌더 파이프라인이라 지도 카메라 프레임과 Compose 재배치 프레임을 완전히 동기화하기 어렵다.

## 2. 현재까지 검토한 방식

### 2.1 Compose 오버레이 + 이동 종료 시 좌표 재계산

- 방식: Kakao SDK `toScreenPoint()`로 위경도를 화면 좌표로 변환하고 Compose `Image`에 `Modifier.offset`을 적용한다.
- 장점: Compose 접근성/리소스/레이아웃 제어가 쉽다.
- 문제: 지도 이동 중에는 화면 좌표가 갱신되지 않아 마커가 지도와 함께 고정되어 보이지 않는다. 이동이 끝난 뒤 계산된 위치로 돌아가는 것처럼 보인다.
- 결론: 지도 좌표에 붙는 마커를 구현하기에는 부족하다.

### 2.2 Compose 오버레이 + 이동 중 매 프레임 좌표 재계산

- 방식: `setOnCameraMoveStartListener` 이후 `MapView.postOnAnimation`으로 매 프레임 `toScreenPoint()`를 호출하고 Compose 오버레이 좌표를 갱신한다.
- 장점: 이동 중에도 마커가 지도 좌표를 어느 정도 따라간다.
- 문제: Kakao 지도 렌더링 프레임과 Compose recomposition/layout 프레임이 어긋나 마커가 흔들린다.
- 결론: 흔들림을 완전히 제거하기 어렵다.

### 2.3 현재 위치/선택 핀을 Kakao SDK Label로 합치기

- 방식: 현재 위치/선택 목적지/선택 맵 핀을 시설 마커와 같은 `LabelLayer`에 넣는다.
- 장점: 지도 엔진 내부에서 좌표 기반 렌더링이 이루어지므로, 지도 이동 중 흔들림을 구조적으로 줄일 수 있다.
- 실패 결과: 현재 위치 마커와 선택 핀이 보이지 않는 문제가 반복됐다.
- 추정 원인: Kakao `LabelStyle.from(context, drawableRes)`가 vector drawable, 리소스 bounds, anchor, dp scale, rank/z-order 조합에 민감하게 반응했을 가능성이 있다.
- 결론: 방향 자체는 정석이지만, 현재 vector drawable 그대로 적용하는 방식은 재시도하지 않는다.

## 3. 후속 구현 후보

### 후보 A. 선택 핀을 화면 중앙 고정 핀으로 전환

- 방식: 지도 좌표에 마커를 붙이지 않고, 지도 중앙에 Compose 또는 일반 View 핀을 고정한다. 지도를 움직인 뒤 카메라 중심 좌표를 선택 좌표로 사용한다.
- 적용 대상: 지도에서 위치를 직접 찍는 선택 핀.
- 장점: 흔들림이 없다. SDK label 문제를 피할 수 있다.
- 단점: 특정 위경도에 붙어 움직이는 마커 UX가 아니라, 배달앱식 "지도 중앙 핀" UX가 된다.
- 권장도: 높음. 선택 핀 UX가 허용된다면 가장 안정적이다.

### 후보 B. 이동 중 Compose 마커 숨김, 이동 종료 후 재표시

- 방식: 카메라 이동 시작 시 현재 위치/선택 핀 오버레이를 숨기고, 카메라 이동 종료 시 `toScreenPoint()`로 재계산해 다시 표시한다.
- 장점: 흔들림과 따라왔다가 튀는 느낌을 줄일 수 있다.
- 단점: 이동 중 마커가 보이지 않는다.
- 권장도: 중간. 빠른 안정화용 workaround로만 사용한다.

### 후보 C. PNG 기반 Kakao SDK Label 재시도

- 방식: 현재 위치/선택 핀을 vector drawable이 아니라 투명 배경 PNG로 만들고, Kakao SDK `LabelStyle` 기준에 맞춰 anchor/rank/layer를 다시 잡는다.
- 현재 위치 마커 권장 이미지: `44x44px` 또는 `48x48px`, 투명 배경, 흰 원 + 파란 점.
- 선택 핀 권장 이미지: `64x64px`, 투명 배경, 핀 하단 끝이 실제 좌표를 가리키는 형태.
- 현재 위치 anchor: `(0.5, 0.5)`.
- 선택 핀 anchor: `(0.5, 1.0)`.
- rank/z-order: 시설 마커보다 높게 잡는다.
- 주의: 이전 실패가 있었으므로 기존 vector drawable을 그대로 Label에 넣는 재시도는 하지 않는다.
- 권장도: 높음. 지도 좌표에 붙어 이동해야 한다면 이 방향이 정석이다.

## 4. 권장 작업 순서

1. 선택 핀 UX가 중앙 고정 핀으로 바뀌어도 되는지 기획/디자인 확인.
2. 허용되면 선택 핀은 중앙 고정 방식으로 전환하고, 현재 위치는 기존 Compose 오버레이 유지 또는 별도 처리한다.
3. 지도 좌표 고정이 반드시 필요하면 현재 위치/선택 핀 PNG 리소스를 먼저 만든다.
4. PNG 리소스로 Kakao `LabelStyle` 적용을 최소 범위에서 검증한다.
5. 검증 시 기존 시설 마커 레이어를 건드리지 말고, 현재 위치 마커 하나만 별도 실험한다.
6. 마커가 보이는 것을 확인한 뒤 선택 핀, 목적지 핀 순서로 확장한다.

## 5. 완료 기준

- 지도 이동 중 현재 위치/선택 핀이 흔들리지 않는다.
- 지도 이동 중 마커가 사라지지 않는다.
- 지도 이동 종료 후 마커가 잘못된 화면 좌표로 튀지 않는다.
- 시설 마커 클릭/표시 동작이 유지된다.
- `:app:compileDebugKotlin`이 통과한다.
