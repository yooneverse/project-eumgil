# FE 백엔드/기획 정합성 수정 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 최신 백엔드 API 명세와 기획 문서 기준으로 FE의 인증, 장소/검색, 제보, 경로, 북마크 구현과 테스트를 정합하게 맞춘다.

**Architecture:** 원격 API 계약은 `data/remote` DTO와 DataSource에서 수용하고, Repository에서 기존 도메인 모델로 매핑한다. 화면/Contract/ViewModel은 서버 계약을 직접 노출하지 않고, 필요한 UI 상태와 이벤트만 유지한다. 이 계획에서는 기존 검토 이슈 2번인 저시력 온보딩 라우팅/OS 권한/튜토리얼 플로우 수정은 제외한다.

**Tech Stack:** Kotlin, Android Jetpack Compose, Navigation Compose, Retrofit, kotlinx.serialization, Room, Coroutine, JUnit.

**Document Order:** `Docs/PRD/2026-04-14_기능명세서.md` -> `Docs/기획/2026-04-09_화면명세서.md` -> 도메인별 API 명세 -> `FE/docs/*` 보조 문서 순으로 판단한다. 이 계획 문서는 구현 작업용이며 기준 문서 자체를 대체하지 않는다.

## 수용 기준

- 인증은 신규 사용자에게 `signupToken`을 보존하고, 기존 사용자에게만 서비스 토큰 세션을 만든다.
- 서버 명세에 없는 `profileCompleted`가 로그인 완료/시작 목적지 판단 기준으로 쓰이지 않는다.
- 장소 검색은 `placeId = null`인 외부 검색 결과를 처리한다.
- 장소 카테고리는 서버 `ServiceCategory` 7종과 일치하고, 접근성 featureType은 카테고리와 분리된다.
- 제보 유형은 서버 `ReportType` 6종과 일치한다.
- 제보 생성 요청은 `reportPoint`와 `imageUrls`를 사용하며 사용자 화면에 처리 상태를 노출하지 않는다.
- 경로 검색 요청/응답은 `destinationPlaceId`, `warningLabels`, `usedConditionalStairs`, `slopePercent`, `widthState`를 보존한다.
- 경로 옵션은 MVP 기준 `SAFE`, `SHORTEST`만 사용한다.
- 북마크는 서버 API 기준으로 장소 북마크와 자주 가는 길을 분리한다.
- `./gradlew.bat :app:testDebugUnitTest`와 `./gradlew.bat :app:compileDebugUnitTestKotlin`가 통과한다.

## 확인 필요 사항

- 이미지 업로드 API가 별도 제공되는지, 아니면 제보 생성 시 이미 업로드된 `imageUrls`만 받는지 확인이 필요하다.
- mock fallback 정책을 API 연동 이후에도 유지할지, 개발 빌드에서만 허용할지 결정이 필요하다.
- 저시력 온보딩 상세 라우팅 이슈는 이번 계획에서 제외했으므로 별도 작업 번호와 계획이 필요하다.
