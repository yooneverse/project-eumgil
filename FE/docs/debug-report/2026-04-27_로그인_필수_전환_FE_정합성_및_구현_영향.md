# 로그인 필수 전환 FE 정합성 및 구현 영향

> **작성일** 2026-04-27  
> **기준 정책** 팀 회의 결정: 로그인은 선택이 아니라 필수  
> **최종 수정일** 2026-04-29
> **작성 범위** FE 문서 정합성, 현재 FE 구현 변경 필요 항목 정리
> **문서 성격** 팀 결정과 FE 영향 범위를 정리한 2차 정책 메모. 최종 계약 변경은 1차 FE 계약 문서에 반영된 내용을 우선한다.

---

## 1. 결론

로그인 필수 정책은 유지한다. 다만 현재 FE 온보딩 기준선에서 기존 회원 앱 진입 gate는 `authSession -> selectedPrimaryUserType 홈 분기`이며, 신규 사용자는 소셜 인증 후 `signupToken`으로 `ONB-001 사용자 유형 1차 선택`에 진입한다. `저시력자`는 `LV-HOM-01`, `보행약자`는 `MAP-01`로 연결한다.

현재 기준선에는 별도 프로필 완료 전용 화면이나 프로필 완료 여부를 끼운 이중 gate를 포함하지 않는다.

```text
앱 실행
  -> authSession 확인
  -> 미로그인: AUTH-001 로그인
  -> 기존 회원 로그인 완료: selectedPrimaryUserType에 따라 LV-HOM-01 또는 MAP-01
  -> 신규 사용자 signupToken 발급: ONB-001 또는 미완료 온보딩 단계

ONB-001 사용자 유형 1차 선택
  -> 저시력자: ONB-002 저시력자 전용 후속 화면
  -> 보행약자: ONB-003 보행약자 2차 선택
  -> ONB-004 약관/동의
  -> 다음 버튼 이후 OS 위치 권한 시스템 팝업
  -> ONB-005 튜토리얼
  -> MAP-01
```

## 3. FE 문서 정합성 반영

| 항목 | 기존 충돌 | 현재 기준 |
| --- | --- | --- |
| 앱 진입 gate | 기존 프로필 완료 전용 이중 gate | 기존 회원 `authSession -> selectedPrimaryUserType 홈 분기`, 신규 사용자 `signupToken -> 온보딩 -> POST /auth/signup` |
| 인증 화면 | 기존 로그인 화면 + 프로필 완료 전용 화면 구조 | `AUTH-001`만 현재 기준선에 포함 |
| 로그인 후 첫 화면 | 로그인 후 프로필 설정 또는 가입 완료 화면 | 로그인 후 최초 화면은 `ONB-001 사용자 유형 1차 선택` |
| 저시력자 처리 | `VIS-*` 후속 확장 또는 별도 메인 플로우 | `저시력자`는 `ONB-001`의 1차 분기이며 `ONB-002`로 연결 |
| 보행약자 세부 선택 | 기존 단일 상태키에 1차/2차 선택을 혼합 저장 | `selectedPrimaryUserType`와 조건부 `selectedMobilitySubtype`를 구분 |
| 약관/권한 | 약관 동의와 위치 권한 요청이 섞여 기술됨 | `개인위치정보 수집·이용 동의`는 법적 동의, `OS 위치 권한`은 시스템 팝업으로 분리 |
| 마이페이지 재진입 | 사용자 유형 변경 시 `ONB-002` 재사용 | `MYP-01` 사용자 유형 변경은 `ONB-001`부터 재진입 |

## 4. 구현 영향 메모

| 영역 | 현재 기준에 따른 메모 |
| --- | --- |
| Start destination | 앱 시작 분기는 기존 회원은 `authSession`과 `selectedPrimaryUserType`, 신규 사용자는 `signupToken`으로 판단한다. |
| Onboarding nav graph | `ONB-001 -> ONB-002/003 -> ONB-004 -> OS 위치 권한 시스템 팝업 -> ONB-005` 구조를 사용한다. |
| 상태 키 | `selectedPrimaryUserType`, `lowVisionFollowUpCompleted`, 조건부 `selectedMobilitySubtype`, 필수 약관 동의 상태, `locationPermissionStatus`, `signupToken` 구분이 필요하다. |
| 권한 요청 시점 | `ONB-004`에서 사용자가 `다음`을 누른 뒤에만 `OS 위치 권한 시스템 팝업`을 띄운다. |
| 튜토리얼 완료 | 위치 권한 응답 이후 `ONB-005`에서 완료 처리 후 `selectedPrimaryUserType`에 따라 `LV-HOM-01` 또는 `MAP-01`로 이동한다. |
| 프로필 완료 gate | 현재 FE 온보딩 기준선에는 포함하지 않는다. 추후 별도 계정/프로필 과제로 분리 검토한다. |

## 5. 오픈 이슈

| 항목 | 내용 |
| --- | --- |
| 저시력자 후속 화면 상세 | `ONB-002`의 세부 입력 항목은 문서 근거가 더 필요하다. 현재는 존재와 연결만 확정한다. |
| 서버 필드 매핑 | 사용자 유형과 약관 동의 저장을 기존 서버 필드와 어떻게 매핑할지는 `BE 확인 필요`다. |
| 약관 버전/저장 위치 | 약관 버전 관리, 저장 위치, 서버 동기화 범위는 `BE 확인 필요`다. |

## 6. 화면명세서 개정 시 최종 반영 기준

- 로그인은 선택 기능이 아니라 필수 진입 조건이다.
- 로그인 성공 후 최초 화면은 `ONB-001 사용자 유형 1차 선택`이다.
- `저시력자`는 `ONB-002 저시력자 전용 후속 화면`으로 이어진다.
- `보행약자`는 `ONB-003 보행약자 2차 선택`으로 이어지며, 선택지는 `전동휠체어 사용자`, `수동휠체어 사용자`, `기타 보행약자`다.
- `ONB-004 약관/동의`의 `개인위치정보 수집·이용 동의`와 다음 단계의 `OS 위치 권한 시스템 팝업`을 절대 한 단계로 합치지 않는다.
- `MYP-01`의 사용자 유형 변경은 `ONB-001`부터 다시 시작한다.
