# FE 북마크 화면 해결완료 디버깅 종결 리포트

> 작성일: 2026-05-12
> Work Lane: FE
> 상태: Resolved / Closed
> 범위: 북마크가 1개 이상 존재할 때 북마크 화면이 깨져 보이던 FE 이슈 종결 기록

## 1. 최종 결론

- 최종 원인은 `SavedRouteScreen`의 리스트 카드 액션 버튼 레이아웃이었다.
- `NoRippleSavedRouteNavigationButton()` 내부 content row가 항상 `fillMaxWidth()`를 사용하고 있었고, 이 버튼이 북마크 리스트 카드 안에서도 그대로 재사용되고 있었다.
- 그 결과 북마크가 0개일 때는 문제가 드러나지 않았지만, 북마크가 1개 이상 생겨 `CONTENT` 상태의 리스트 카드가 렌더되면 버튼이 카드 내부 가로폭을 과하게 점유하면서 제목/주소 컬럼 레이아웃이 눌리고 화면이 깨져 보였다.

## 2. 증상 정리

- 빈 상태에서는 정상
- 북마크를 등록한 뒤, 북마크가 1개 이상 보이는 상태에서만 화면이 깨짐
- 즉 `EMPTY` 상태 문제가 아니라 `CONTENT` 상태의 카드 렌더링 문제였음

## 3. 디버깅 과정 요약

- 1차 가설
  - `develop` 대비 네비게이션 복원 정책 변경이 원인일 수 있다고 판단
  - 북마크 경로에 한정한 navigation rollback을 적용했지만 증상은 해소되지 않음

- 2차 가설
  - 북마크 저장 직후 이동 경로에서 예외가 올라오는지 의심
  - `NavigationViewModel`, `LowVisionNavigationCompleteRoute`에 저장 예외 가드를 추가
  - 이 수정은 크래시 방어 관점에서는 의미가 있었지만, 사용자가 말한 "화면 깨짐" 자체의 직접 원인은 아니었음

- 3차 가설
  - "북마크가 있을 때만 깨진다"는 조건에 맞춰 리스트 카드 자체를 재검토
  - `SavedBookmarkPrimaryActionButton()`이 사용하는 커스텀 no-ripple 버튼의 폭 정책이 리스트 카드와 맞지 않는 점을 확인
  - 이 지점 수정 후 문제 해결

## 4. 최종 수정 내용

- 수정 파일
  - `FE/app/src/main/java/com/ssafy/e102/eumgil/feature/savedroute/SavedRouteScreen.kt`

- 수정 핵심
  - `NoRippleSavedRouteNavigationButton()`에 `fullWidthContent` 플래그 추가
  - 상태 카드 CTA (`SavedBookmarkStateActions`)만 `fullWidthContent = true` 사용
  - 리스트 카드 액션 버튼 (`SavedBookmarkPrimaryActionButton`)은 기본값 `false`를 사용해 버튼 content가 wrap 되도록 분리

- 효과
  - 빈/에러 상태의 넓은 CTA 레이아웃은 유지
  - 북마크 리스트 카드 안에서는 버튼이 필요한 크기만 사용
  - 카드 본문 텍스트 영역이 버튼 때문에 눌리지 않음

## 5. 함께 반영된 방어 조치

- 다음 파일들은 직접 원인 수정은 아니지만, 북마크 저장 실패 시 화면 전환이 그대로 이어지지 않도록 방어 조치를 유지한다.
  - `FE/app/src/main/java/com/ssafy/e102/eumgil/feature/navigation/NavigationViewModel.kt`
  - `FE/app/src/main/java/com/ssafy/e102/eumgil/feature/lowvision/LowVisionNavigationCompleteRoute.kt`

- 의미
  - 저장 실패가 발생하더라도 저장 후 북마크 화면으로 강제 이동하는 경로를 줄여 디버깅 난이도와 사용자 체감 리스크를 낮춤

## 6. 검증 결과

- 코드 검증
  - `./gradlew.bat :app:compileDebugUnitTestKotlin` 통과

- 정적 정책 검증
  - `FE/app/src/test/java/com/ssafy/e102/eumgil/feature/savedroute/SavedRouteScreenPolicyTest.kt`
  - 상태 카드 CTA는 전체 폭 사용
  - 리스트 카드 액션 버튼은 전체 폭 content를 강제하지 않음

- 사용자 검증
  - 사용자 기준 해결 완료 확인

- 제한 사항
  - `:app:testDebugUnitTest` 전체는 기존 테스트 런타임의 `ClassNotFoundException` / `initializationError` 이슈가 있어 이번 건의 성공/실패 판정 기준으로 쓰지 않음

## 7. 재발 방지 포인트

- 공통 컴포넌트를 상태 카드와 리스트 카드에 동시에 재사용할 때는 폭 정책을 분리해야 한다.
- 특히 `fillMaxWidth()`는 상위가 `weight()`를 주는 CTA 영역에서는 자연스럽지만, 카드 내부 trailing action에서는 레이아웃 붕괴를 만들 수 있다.
- 이후 no-ripple 커스텀 버튼을 재사용할 때는
  - 전체 폭 컨테이너인지
  - 콘텐츠 크기 기반 trailing action인지
  먼저 구분해서 사용해야 한다.
