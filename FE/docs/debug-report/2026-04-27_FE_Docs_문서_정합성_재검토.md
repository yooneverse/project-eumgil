## 1. 목적

2026-04-27 시점 FE 문서군에서 어긋났던 온보딩, 약관, 로그인 필수 전환 관련 표현을 한 번 모아 정리한 기록이다. 현재는 상위 기획 문서와 API 문서를 기준으로 보고, 이 문서는 FE 보조 문서가 어느 지점에서 어긋났는지 확인하는 색인과 전환 메모로만 사용한다.

## 1.1 문서 사용 원칙

| 이 문서가 다루는 것 | 상위 문서가 결정하는 것 |
| --- | --- |
| 2026-04-27 시점 FE 문서 간 충돌 기록 | 최신 화면 ID, 최종 사용자 흐름 |

## 3. 2026-04-27 시점 정합성 결론

아래 결론은 2026-04-27에 FE 문서를 맞추며 정리한 결과다. 현재 정책 확인 시에는 상위 문서와 실제 `FE/app` 구현을 먼저 보고, 이 표는 전환 배경과 당시 의사결정을 이해하는 용도로만 사용한다.

| 항목 | 2026-04-27 시점 결론 |
| --- | --- |
| 앱 진입 gate | 기존 회원 `authSession -> selectedPrimaryUserType 홈 분기`, 신규 사용자 `signupToken -> 온보딩 -> POST /auth/signup` |
| 인증 화면 | `AUTH-001` 로그인만 사용한다. 당시 정리 기준으로는 별도 프로필 완료 전용 gate를 현재 온보딩 기준선에서 제거하는 방향이었다. |
| 온보딩 1단계 | `ONB-001 사용자 유형 1차 선택` |
| 1차 분기 | `저시력자` / `보행약자` |
| 저시력자 분기 | `ONB-002 저시력자 전용 후속 화면`으로 이동한 뒤 `ONB-004`로 연결한다. |
| 보행약자 분기 | `ONB-003 보행약자 2차 선택`으로 이동한다. 선택지는 `전동휠체어 사용자`, `수동휠체어 사용자`, `기타 보행약자`다. |
| 약관/동의 | `ONB-004 약관/동의`에서 법적 동의 항목을 받는다. |
| OS 권한 | `ONB-004` 이후 다음 단계에서 `OS 위치 권한 시스템 팝업`을 띄우는 구조를 유지한다. |
| 튜토리얼 | 위치 권한 응답 이후 `ONB-005 튜토리얼`로 이동하고, 완료 후 `selectedPrimaryUserType`에 따라 `LV-HOM-01` 또는 `MAP-01`로 진입한다. |
| 마이페이지 재진입 | `MYP-01`의 사용자 유형 변경은 `ONB-001`부터 다시 시작한다. |
| VIS 레거시 | `VIS-*`는 현재 MVP 메인 라우트가 아니며, 저시력자 메인 분기를 대체하지 않는다. |

## 4. FE 문서 반영 메모

이 섹션은 당시 어떤 FE 문서 표현을 어느 방향으로 맞춰야 하는지 정리한 메모다. 현재 문서를 수정할 때도 참고는 가능하지만, 최종 표현은 상위 문서와 실제 코드 상태를 먼저 확인한다.

| 문서 | 반영 메모 |
| --- | --- |
| `2026-04-27_온보딩_디자인_화면_구성_분석.md` | `ONB-001~005`를 당시 기준선으로 유지했다. |
| `2026-04-22_부산이음길_FE_화면_인벤토리_및_라우트_맵.md` | 라우트 맵과 화면 inventory를 같은 기준선으로 맞추도록 봤다. |

| 기타 FE 문서 | 구형 저시력자 대체 용어, `VIS` 메인 플로우, `권한 안내 -> 서비스 소개`, 위치 동의와 시스템 권한을 같은 단계처럼 쓰는 표현을 쓰지 않도록 정리했다. |

## 6. 사용 시 주의 사항

- `개인위치정보 수집·이용 동의`와 `OS 위치 권한 허용`은 같은 의미가 아니다.
- 문서에서 두 개념을 같은 문장이나 같은 단계 의미로 합쳐 쓰지 않는다.
- 이 문서의 2026-04-27 결론과 현재 `FE/app` 구현이 다르면, 현재 구현 차이를 별도 메모로 남기고 상위 기준 문서를 다시 확인한다.
- 제품 정책, 로그인 방식, 온보딩 단계, 데이터 책임이 바뀌면 이 문서를 먼저 고치지 말고 `화면명세서`, `기능명세서`, `API 문서`를 먼저 수정한다.
