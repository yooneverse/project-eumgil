## 1. 핵심 정리

1. 현재 보행약자 공통 하단 탭은 `홈 / 북마크 / 제보 / 마이페이지` 4종 아이콘 세트를 사용한다.
2. 지도 화면의 버튼, 칩, 최근 목적지 목록은 drawable 아이콘을 쓰지만, 지도 마커 본체는 아직 drawable 아이콘이 아니라 텍스트 약어(`R`, `WC`, `EV` 등)를 사용한다.
3. 제보 화면은 현재 FE 코드 기준으로 8개 legacy 타입 아이콘을 쓰고 있고, 서버/API 계약은 6개 타입 기준이므로 별도 정리가 필요하다.
4. 서버 장소 카테고리 7종과 `AccessibilityFeatureType` 7종은 FE 코드에 아직 완전히 반영되지 않았지만, 아이콘 리소스 관점에서는 대부분 매핑 가능하다.
5. `accessibleEntrance`, `accessibleRoom`, `guidanceFacility`는 현재 전용 drawable이 없어서 임시 매핑 또는 신규 아이콘 추가 판단이 필요하다.

## 6. 문서 사용 원칙

1. 화면에 이미 텍스트가 같이 보이는 아이콘은 `contentDescription = null`을 유지하고, 카드/버튼 container에서 의미를 전달한다.
2. icon-only 버튼은 반드시 접근성 라벨과 필요 시 `stateDescription`을 같이 문서화한다.
3. 서버/API 값이 아직 FE enum으로 내려오지 않았더라도, 리소스명과 목표 라벨은 이 문서를 기준으로 먼저 고정한다.
4. 장소 카테고리와 접근성 속성은 같은 아이콘을 재사용하더라도 문서상 의미를 분리해서 기록한다.
5. 지도 마커처럼 drawable이 아니라 텍스트 약어를 쓰는 요소는 “아이콘 미적용” 상태로 명시한다.
6. 화면 전용 보조선이나 배치 구조는 문서에 별도로 남기되, 재사용 가능한 본체 아이콘은 가능한 한 `drawable` 리소스로 등록하고 화면에서는 tint/배치만 조합한다.
