# 2026-05-12 FE 실제 길찾기 최근 목적지 저장 추가 디버깅

## 1. 배경

- 기존 디버깅으로 북마크 장소 클릭 시 recent destination 저장 누락은 보강했다.
- 하지만 사용자 기대치는 더 넓다.
- 검색, 지도 선택, 저장된 경로, 저장된 장소 등 어떤 유입이든 실제로 길찾기를 시작했다면 최근 목적지에 남아야 한다.

## 2. 추가 원인

- 기존 recent 저장 경로는 일부 진입 액션에만 연결되어 있었다.
- `RouteSetting`에서 실제 길찾기 시작이 확정되는 공용 경로에는 recent 저장이 없었다.
- 따라서 아래 경로는 최근 목적지에 남지 않을 수 있었다.
  - 지도에서 목적지를 선택하고 길찾기 시작
  - 저장된 경로를 열고 길찾기 시작
  - 기타 `RouteSetting`을 거쳐 실제 네비게이션을 시작하는 경로

## 3. 수정 방향

- recent 저장 기준을 `RouteSettingViewModel.startNavigation()` 성공 경로로 올렸다.
- `routeRepository.selectRoute(...)` 성공 후 목적지를 recent destination으로 저장한다.
- recent 저장이 끝난 뒤 `StartNavigationRequested`를 emit 하도록 순서를 고정했다.
- recent 저장 실패는 길안내 시작 자체를 막지 않도록 `runCatching`으로 처리했다.

## 4. 반영 파일

- `FE/app/src/main/java/com/ssafy/e102/eumgil/feature/route/RouteSettingViewModel.kt`
  - `SearchRepository` 주입 추가

  - `startNavigation()` 성공 경로에 recent 저장 추가
- `FE/app/src/main/java/com/ssafy/e102/eumgil/feature/route/RouteSettingEntryRoute.kt`

- `FE/app/src/main/java/com/ssafy/e102/eumgil/core/model/SearchModels.kt`

- `FE/app/src/test/java/com/ssafy/e102/eumgil/feature/route/RouteSettingViewModelTest.kt`
  - recent 저장이 navigation event보다 먼저 일어나는 회귀 테스트 추가

## 6. 현재 판단

- FE 로직 기준으로는 이제 `RouteSetting`에서 실제 길찾기를 시작하는 모든 경로가 recent 저장 공용 경로를 탄다.
- 따라서 "실제로 길찾기를 썼는데 recent에 안 남는다"는 구조적 누락은 보강됐다.
- 남은 확인 포인트는 기능 로직보다도 현재 unit test runner의 클래스 로딩 실패다.
