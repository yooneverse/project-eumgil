# FE 제보/북마크/마이페이지 화면전환 리플 잔상 억제 디버그 리포트
> 작성일: 2026-05-11
> Work Lane: FE
> 대상 화면: 제보, 북마크, 마이페이지 계열의 즉시 화면 전환 CTA

## 1. 증상

- 사용자가 CTA를 누른 직후 다른 화면으로 이동할 때, 출발 화면에 눌림 ripple이 짧게 남아 `잔상`처럼 보였다.
- 이번 이슈는 지도 홈탭 복귀 잔상 정리 이후에도 제보, 북마크, 마이페이지 흐름에서 따로 남아 있던 케이스다.
- 사용자가 체감한 문제는 `탭 하이라이트가 아니라`, `이동 직전 CTA 눌림 표시가 화면 전환과 겹쳐 보이는 현상`에 가깝다.

## 2. 이번 작업의 판단 기준

- 같은 화면 안에서 상태만 바꾸는 CTA는 기존 ripple을 유지한다.
  - 예: 단계 진행, 입력 완료, 토글, 재시도 등
- 누르는 즉시 다른 화면으로 이동하거나 외부 링크를 여는 CTA는 ripple을 제거한다.
  - 예: 지도 복귀, 길안내 시작, 제보 내역 확인, 약관/가이드 링크 이동
- scrim, overlay dismiss 계열에서 `indication = null`을 쓰는 기존 정책은 이번 작업 범위와 별개로 유지한다.

## 3. 이번 적용 내용

### 3.1 제보 화면

- `FE/app/src/main/java/com/ssafy/e102/eumgil/feature/report/ReportScreen.kt`
  - 완료 단계의 `제보 내역 확인하기`만 no-ripple CTA로 분기했다.
  - `LocationConfirm`, `DetailInput` 단계의 기본 액션은 같은 화면 내부 진행이므로 기존 ripple을 유지했다.
  - `shouldSuppressReportPrimaryActionRipple(...)` 정책 함수를 추가해 의도를 코드에 남겼다.

### 3.2 북마크 화면

- `FE/app/src/main/java/com/ssafy/e102/eumgil/feature/savedroute/SavedRouteScreen.kt`
  - 비어 있음/오류/완료 상태에서 바로 다른 화면으로 나가는 주요 CTA를 no-ripple 버튼으로 통일했다.
  - 대표적으로 `지도로 돌아가기`, `길안내 시작` 계열이 대상이다.
  - 저장 장소 리스트 행은 기존에도 화면 이동 성격이라 no-ripple 정책이 맞는 상태였고, 이번에 명시적으로 유지되도록 정리했다.

### 3.3 마이페이지 앱 정보 화면

- `FE/app/src/main/java/com/ssafy/e102/eumgil/feature/mypage/MyPageAppInfoScreen.kt`
  - `화면 가이드`, `개인정보처리방침`, `서비스 이용약관`처럼 화면 이동 또는 외부 링크 오픈이 일어나는 행만 no-ripple 처리했다.
  - 준비 중 토스트만 띄우는 항목이나 현재 화면 안에서 끝나는 동작은 기본 동작을 유지했다.
  - `suppressRipple` 플래그를 row model에 추가해, 어떤 행이 정책 대상인지 문서화된 형태로 남겼다.

### 3.4 마이페이지 제보 내역 화면

- `FE/app/src/main/java/com/ssafy/e102/eumgil/feature/mypage/MyPageReportHistoryScreen.kt`
  - 빈 상태, 오류 상태, 콘텐츠 상태에서 노출되는 `제보하기` CTA를 no-ripple로 분기했다.
  - `ReportHistoryStateCard`에 액션별 suppress 플래그를 받아 상태별 정책을 분리했다.

## 5. 현재 검증 상태

### 5.1 확인 완료

- 소스 레벨에서 대상 CTA가 모두 no-ripple helper 또는 `indication = null + MutableInteractionSource()` 경로를 타는 것은 확인했다.
- 변경 시점 기준 `FE/./gradlew.bat :app:compileDebugUnitTestKotlin`은 통과했다.

### 5.2 현재 환경 한계

- `FE/./gradlew.bat :app:testDebugUnitTest`는 이번 변경 assertion 실패가 아니라, 현재 레포 전반의 unit-test runtime 문제로 끝까지 검증되지 않았다.
- 관측된 실패 형태는 여러 테스트 클래스에서 공통으로 발생하는 `initializationError`, `ClassNotFoundException`이다.
- 따라서 지금 단계의 신뢰 가능한 사실은 `코드 컴파일은 정상`, `런타임 unit-test 환경은 별도 정비 필요`다.

## 7. 후속 판단 기준

- 만약 잔상이 다시 보이면 먼저 `ripple 잔상`인지 `화면 자체 전환 애니메이션/overlay 잔상`인지 분리해서 봐야 한다.
- ripple 잔상이라면 해당 CTA가 실제로 no-ripple 경로를 타는지 다시 확인한다.
- ripple이 아니라면 이번 이슈와 별개로 화면 전환 레이어, overlay, 또는 지도/웹뷰 surface 교체 타이밍 문제일 가능성이 크다.
