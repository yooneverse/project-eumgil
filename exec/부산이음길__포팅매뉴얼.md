# 프로젝트 포팅 매뉴얼

## 1. 프로젝트 개요

부산이음길은 부산 지역 보행약자와 저시력자를 위한 배리어프리 지도 서비스다.

주요 구성은 다음과 같다.

| 영역 | 위치 | 역할 |
|---|---|---|
| Android 앱 | `FE/` | Kotlin, Jetpack Compose 기반 사용자 앱 |
| Backend API | `BE/` | Spring Boot API, 인증, 장소, 경로, 제보, 관리자 도메인 |
| AI 서버 | `AI/` | Flask/Gunicorn 기반 음성 분석 API |
| 관리자 웹 | `ADMIN/` | React/Vite 기반 운영 관리자 화면 |
| 인프라 자산 | `INF/` | Jenkins, GraphHopper, monitoring, Terraform, AWS 기준 파일 |
| 실행 스크립트 | `scripts/` | compose, DB 적재, GraphHopper build, deploy, smoke 자동화 |
| 문서 | `Docs/` | PRD, API, ERD, 인프라, 운영 가이드 |

현재 운영 기준은 `S1 dev/Jenkins/운영도구 + S2 primary prod` 구조다.

```text
사용자/관리자
  -> Route53 / DNS
  -> S2 Nginx 80/443
  -> prod backend / AI / ADMIN / GraphHopper blue-green
  -> RDS PostgreSQL / ElastiCache Redis / S3

개발/운영자
  -> S1 Nginx 80/443
  -> dev backend / dev AI / Jenkins / Grafana / SonarQube
  -> Portainer는 SSH tunnel 전용
```

Prod blue/green은 EC2 서버 단위가 아니라 S2 내부 GraphHopper runtime slot인 `graphhopper-blue`, `graphhopper-green` 전환을 의미한다.

### 3.3 환경 변수 원칙

환경 변수 파일은 루트에서 관리한다.

| 환경 | 파일 | 용도 |
|---|---|---|
| local | `.env.local` | 개인 PC compose 실행 |
| dev | `.env.dev` | S1 dev stack, Jenkins dev credential |
| prod | `.env.prod` | S2 prod deploy, Jenkins prod credential |

주의 사항:

- 실제 secret 값은 저장소 문서에 기록하지 않는다.
- `.env.*` 파일은 git에 올리지 않는다.
- key 목록과 실제 값은 팀의 별도 secret 저장소 또는 Jenkins credential 기준으로 관리한다.
- prod에서는 RDS, ElastiCache, S3 같은 managed resource endpoint를 사용한다.
- dev/local에서는 PostGIS, Redis, MinIO를 compose로 띄운다.

## 6. AI 음성 분석 서버 포팅

AI 서버는 Android STT 텍스트를 받아 intent와 실행 파라미터를 반환하는 음성 명령 추론 서버다. FE가 직접 AI 서버를 호출하지 않고, FE는 backend의 `POST /voice/analyze`를 호출하며 backend가 내부적으로 AI 서버의 `POST /voice/analyze`를 호출한다.

```text
Android FE
  -> Backend POST /voice/analyze
  -> AI server POST /voice/analyze
  -> GMS Gemini 2.5 Flash
  -> Backend response
  -> Android action / TTS / 화면 이동
```

| 위치 | 역할 | 포팅 기준 |

| `AI/Dockerfile` | dev/prod AI container build entrypoint | 운영 배포 기준 |
| `AI/llm_test/server/app.py` | Flask AI 서버 실제 entrypoint | 운영 배포 기준 |
| `AI/llm_test/server/config.py` | host, port, debug, default model 설정 | 운영 배포 기준 |
| `AI/llm_test/server/env_loader.py` | 루트 `.env.dev`, `.env.prod` 로딩 | 운영 배포 기준 |
| `AI/llm_test/server/providers/gemini_provider.py` | GMS Gemini provider, tool/function calling prompt | 운영 배포 기준 |
| `AI/llm_test/server/tests/` | API/error/voice flow 검증 스크립트 | 검증 기준 |
| `AI/llm_test/FE/` | Android AI PoC 앱 | 참고/실험용 |
| `AI/ondevice_test/` | sherpa-onnx on-device STT PoC | 참고/실험용 |
| `AI/voice_data/` | 음성 데이터 전처리/검증 파이프라인 | 참고/데이터 작업용 |

현재 Dockerfile은 아래 파일을 컨테이너에 복사하고 `gunicorn app:app`으로 실행한다.

```text

```

따라서 AI 포팅 시 `AI/app.py`가 아니라 `AI/llm_test/server/app.py`와 그 하위 provider/test를 기준으로 확인한다.

### 6.2 AI runtime 환경 변수

| key | 설명 | 기준 |
|---|---|---|
| `APP_ENV` | `dev`, `prod`, `local` 등을 지정 | `local`은 `dev`로 정규화 |
| `SPRING_PROFILES_ACTIVE` | `APP_ENV`가 없을 때 env 판정에 사용 | backend와 같은 env 파일 공유 가능 |
| `ENV_FILE` | 명시 env 파일 경로 | 지정 시 최우선 후보 |
| `GMS_KEY` | GMS API key | 필수 |
| `AI_HOST` 또는 `HOST` | Flask/Gunicorn bind host | 기본 `0.0.0.0` |
| `AI_PORT` 또는 `PORT` | Flask/Gunicorn port | 기본 `5000` |
| `DEBUG` | Flask debug 여부 | prod에서는 `false` 권장 |
| `DEFAULT_MODEL` | 기본 모델 key | 현재 기본 `gemini` |

환경 파일 로딩 우선순위:

1. 프로세스에 이미 주입된 runtime env
2. `ENV_FILE`이 지정한 파일
3. `APP_ENV` 또는 `SPRING_PROFILES_ACTIVE`에 맞는 루트 `.env.<env>`
4. `APP_ENV`가 없을 때 루트 `.env.dev`
5. legacy fallback `AI/llm_test/server/.env`

현재 배포 코드의 provider registry는 `gemini`만 로드한다. `AI/llm_test/README.md`에는 모델 비교 PoC 맥락으로 `claude`, `gpt_mini`가 언급되지만, 현재 `app.py` 기준 운영 provider는 `GeminiProvider`다.

### 6.3 AI API 계약

#### `GET /health`

응답 예시:

```json
{
  "status": "healthy",
  "providers": ["gemini"],
  "modes": ["MOBILITY_IMPAIRED", "LOW_VISION"],
  "endpoints": ["POST /voice/analyze"]
}
```

Jenkins dev/prod smoke는 `providers` 배열과 `POST /voice/analyze` endpoint 포함 여부를 확인한다.

#### `POST /voice/analyze`

요청:

```json
{
  "text": "부산역 어디야",
  "mode": "MOBILITY_IMPAIRED",
  "history": [],
  "currentRoute": "navigation/guidance"
}
```

| 필드 | 필수 | 설명 |
|---|---|---|
| `text` | Y | STT 결과 텍스트. 끝의 `.`, `。`, `、`, `·`는 후처리로 제거 |
| `mode` | Y | `MOBILITY_IMPAIRED` 또는 `LOW_VISION` |
| `history` | N | 멀티턴 대화 이력. 기본 `[]` |
| `currentRoute` | N | 현재 앱 route. 예: `navigation/guidance` |

정상 응답:

```json
{
  "status": "S2000",
  "data": {
    "intent": "PLACE_SEARCH",
    "placeName": "부산역",
    "category": null,
    "bookmarkAction": null,
    "departure": null,
    "destination": null,
    "reportType": null,
    "description": null,
    "confirmed": null,
    "confirmationMessage": null
  },
  "message": "정상 처리되었습니다."
}
```

입력 오류:

```json
{
  "status": "C4000",
  "data": null,
  "message": "잘못된 입력입니다."
}
```

AI provider 호출 실패:

```json
{
  "status": "V5020",
  "data": null,
  "message": "음성 분석 AI 호출에 실패했습니다."
}
```

지원 intent:

| intent | 주요 필드 |
|---|---|
| `PLACE_SEARCH` | `placeName` |
| `CATEGORY_SEARCH` | `category` |
| `BOOKMARK_ADD` | `placeName`, `bookmarkAction=add` |
| `BOOKMARK_DELETE` | `placeName`, `bookmarkAction=delete` |
| `NAVIGATE` | `destination`, optional `departure` |
| `SHOW_BOOKMARKS` | 없음 |
| `SHOW_FAVORITE_ROUTES` | 없음 |
| `LOGOUT` | 없음 |
| `REPORT` | `reportType`, optional `description` |
| `NAVIGATION_END` | `currentRoute=navigation/guidance` 맥락에서 사용 |
| `OPEN_MY_PAGE` | 없음 |
| `OPEN_MAP` | 없음 |
| `ASK` | `confirmationMessage` |
| `UNKNOWN` | `confirmationMessage` 가능 |

모드별 동작:

- `MOBILITY_IMPAIRED`: 확인 과정을 두지 않는다. `confirmed`는 항상 `null`이어야 한다.
- `LOW_VISION`: 확인 질문이 필요한 intent는 `confirmationMessage`를 반환하고, 사용자의 긍정 응답이 들어오면 `confirmed=true`를 반환한다.

### 6.4 Backend/FE 연동 포인트

Backend 설정:

```yaml
external:
  ai:
    base-url: ${AI_BASE_URL:http://localhost:5000}
```

Backend의 `AiVoiceAnalysisProperties.voiceAnalyzeUri()`는 `AI_BASE_URL` 뒤에 `/voice/analyze`를 붙인다. 따라서 `AI_BASE_URL`에는 path를 붙이지 말고 base URL만 넣는다.

| 환경 | 권장 `AI_BASE_URL` |
|---|---|
| local compose | `http://ai:5000` |
| local backend 단독 실행 | `http://localhost:5000` |
| dev compose | `http://ai:5000` |
| prod compose | `http://ai:5000` |
| public smoke | `https://ai.busaneumgil.com` |

FE는 backend base URL로 `POST /voice/analyze`를 호출한다. FE에서 AI server public URL을 직접 바라보게 구성하지 않는다.

### 7.1 dev compose 실행

```bash

```

종료:

```bash

```

S1 서버 기준은 repository에 없는 운영 override 파일을 함께 붙인다. 해당 파일은 S1의 `/opt/e102-server/docker-compose.s1.override.yml`에 있으며, Jenkins `e102-dev-deploy`는 이 파일을 workspace로 복사한 뒤 `docker-compose.dev.yml`과 함께 사용한다.

S1에서 직접 확인할 때는 아래처럼 실행한다.

```bash
cd /home/ubuntu/e102/repo
cp /opt/e102-server/docker-compose.s1.override.yml docker-compose.s1.override.yml

docker compose --env-file .env.dev \
  -f docker-compose.dev.yml \
  -f docker-compose.s1.override.yml \
  config --quiet

docker compose --env-file .env.dev \
  -f docker-compose.dev.yml \
  -f docker-compose.s1.override.yml \
  up -d
```

| 서비스 | 역할 |

| `ai` | dev AI server |
| `graphhopper` | dev GraphHopper runtime |
| `backend` | dev backend |

S1에서는 GraphHopper host 포트 충돌을 피하기 위해 `8998/8999`를 사용한다. dev backend의 `/health/graphhopper`는 Redis slot 또는 health URL 설정이 정렬되기 전까지 `DOWN`으로 보일 수 있으나, 직접 `http://127.0.0.1:8999/healthcheck`가 `200`이면 GraphHopper runtime 자체는 정상으로 본다.

## 9. GraphHopper 포팅

GraphHopper 운영 원천은 파일이 아니라 PostgreSQL/PostGIS의 보행 네트워크 테이블이다.

| 단계 | 내용 |
|---|---|

| build job | PostGIS -> temporary OSM XML -> graph-cache |
| runtime | 생성된 graph-cache serve only |
| overlay | `routing_segment_overrides` current-state |

### 9.1 build 명령


## 12. 보안과 secret 관리

- `.env.*`, OAuth secret, API key, SSH key, AWS key는 저장소에 커밋하지 않는다.
- Jenkins에서는 `e102-dev-env-file`, `e102-prod-env-file`, `gitlab-pat`, `e102-s2-ssh-key`, `e102-s2-host` credential을 사용한다.
- EC2 shell 접속은 SSH를 기본으로 하되, `22`는 관리자 고정 IP에서만 허용한다.
- RDS, ElastiCache 같은 private managed resource 접근은 SSM Session Manager port forwarding을 기준으로 한다.
- SSM Session Manager는 SSH 장애 시 복구 채널로 유지한다.
- Portainer는 외부 공개하지 않고 `make portainer-tunnel` 기반 SSH tunnel로만 접근한다.
- GraphHopper admin reload endpoint `/ieum/admin/overrides/reload`는 backend 내부 호출 전용이며, S1/S2 Nginx 공개 경로에서는 차단한다.

## 14. 자주 발생하는 문제

| 증상 | 확인할 것 |
|---|---|
| `git checkout develop` 실패 | 다른 linked worktree에서 `develop`을 사용 중인지 `git worktree list`로 확인 |
| `docker compose config` 실패 | `.env.*` 필수 key 누락 여부 확인 |
| backend가 DB에 연결하지 못함 | `DB_URL`, `DB_USERNAME`, `DB_PASSWORD`, network 기준 host 확인 |

| dev `/health/graphhopper`가 `DOWN` | S1 GraphHopper 직접 `/healthcheck`와 Redis slot/health URL 설정을 분리 확인 |
| ADMIN 화면이 API를 못 찾음 | `VITE_BACKEND_API_URL`, S2 Nginx host routing, CORS 확인 |
| AI health 실패 | `GMS_KEY`, `DEFAULT_MODEL`, `/voice/analyze` endpoint 포함 여부 확인 |
| prod rollback 실패 | `.deploy-state/previous-app-image` 존재 여부 확인 |
| S2에서 `git status`가 안 됨 | S2 prod 디렉터리는 Git worktree가 아니라 archive 배포 산출물임 |

7. 외부 ingress 문제면 S2 Nginx, Route53, certificate, upstream loopback port 확인
